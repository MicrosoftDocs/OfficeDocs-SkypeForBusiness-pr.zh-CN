# <a name="plan-my-service-management"></a>我的服务管理计划

这篇文章概括介绍了提供和维护高质量的 Microsoft 小组部署所需的要求。 您可以帮助通过在构想阶段，第一试验或生产部署之前规划服务管理功能和质量，确保成功的部署。

## <a name="service-management-for-teams"></a>小组的服务管理

服务管理是一个宽泛的话题，涵盖日常操作 Microsoft 小组服务部署并为用户启用之后。 团队服务包括 Microsoft Office 365 和基础结构的组件部署内部 （如连网）。

服务管理的概念很可能不是大多数组织的一个新概念。 您可能已经实现了流程和与现有服务相关联的任务。 话虽如此，但可能可以增加什么您有的今天以支持 Microsoft 小组将来计划服务管理。

服务管理包括的所有活动和进程所涉及的管理端到端的 Microsoft 小组。 某些组件服务管理 — — 包括 Office 365 提供服务本身的基础结构组件 — — 是 Microsoft 的责任，而客户是对其负责的用户都必须在管理团队、 网络上，各个方面和它们提供的终结点。
客户将负责团队服务管理和它与支持的用户体验质量的关键组件的完整讨论，请参阅[计划服务管理和质量](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide)。

![图的三个组件的质量 — Office 365 提供服务、 网络和端点 — 和服务管理与所有这三种的重叠。](media/plan-my-service-management-image1.png "图的三个组件的质量 — Office 365 提供服务、 网络和端点 — 和服务管理与所有这三种的重叠。")

## <a name="introduction-to-the-operations-guide"></a>操作指南简介 

**什么**，**谁**，以及**如何**将需要服务管理时回答的三个重要问题。

您可以使用[操作指南](https://docs.microsoft.com/MicrosoftTeams/1-drive-value-operate-my-service)来帮助您解决所有这三个问题。 指南 》 列出了要在每日、 每周、 每月和按需要的基础上进行的活动。 这些活动和任务对于维护高质量的团队部署至关重要。 确定谁将负责在服务管理中执行特定的活动是，您只需在构想阶段，以确保成功的部署的早期规划的一个重要方面。 已经确定的任务和活动后，他们需要被理解后, 跟的组或个人分配给它们。 操作指南提供的知识和如何执行各项任务，为指导和/或引用的外部内容。

## <a name="plan-for-operational-role-mapping"></a>计划的操作角色映射

早期规划服务管理是一个重要的里程碑，因为操作阶段开始时启用了第一个试验用户。 项目团队必须检查并达成任务和活动所需，确定负责执行每项操作任务，团队获得承诺并签署由每个相应的小组。

签字完毕后，负责小组必须启动留给这些角色和职责。 这可能包括培训和准备工作，正在更新人员配备模型，或确保外部合作伙伴已准备好提供。

在构想阶段的早期映射操作角色使所有团队过程的试验和运营量产开始其操作任务并确保一切都已准备好部署启动之后。

操作指南提供映射到应该是有效的在大多数情况下的典型角色的常见任务的列表。 您需要自定义这些责任为您的组织工作。

## <a name="the-quality-champion-role"></a>质量冠军角色

一组或个别需要负责在所有组织中的质量。
这是服务管理中最重要的角色。 质量冠军是客户角色分配给用户或组的致力于他们的用户体验。 此角色需要的技能，以了解环境和赞助来推动修正的其他小组工作中的趋势。
最适合进行质量冠军通常是客户服务所有者，谁 — — 这取决于组织的规模和复杂性 — — 可能是任何用户或组的是充满激情的用户体验。

质量冠军利用现有工具和有案可稽的流程，如呼叫质量仪表板 (CQD) 并监视用户体验质量的经验审查指南 》，确定质量趋势，并推动补救措施在需要的地方。 质量冠军与驱动器的补救动作，各自团队工作向指导委员会报告其进度和打开的问题上。

操作指南中记录的任务和活动与角色相关联。 在构想阶段的初期，应将分配此角色。 留给质量冠军的角色中的一个关键步骤获得所需的角色，确保各项前提条件均已到位，以提供有关任务的知识。 此角色的主要任务正在运行常规质量经验审查。

## <a name="introduction-to-the-quality-experience-review-guide"></a>质量的经验审查指南简介

质量经验审查指南有一套评估和提供补救措施指导对提高用户体验的影响最大，在下图中所示的关键领域中的活动。

![的关键领域，在质量检查体验评审： 音频、 可靠性和用户调查结果。](media/plan-my-service-management-image2.png "的关键领域，在质量检查体验评审： 音频、 可靠性和用户调查结果。")

通过不断地评估和纠正本文所述的区域，您可以减少他们可能会对用户体验产生负面影响。 在部署中遇到的大多数用户体验问题可以分为以下几类：

-   不完整的防火墙或代理服务器配置

-   较差的 Wi-fi 覆盖范围

-   没有足够的带宽

-   VPN

-   使用优化的还是内置的音频设备

-   有问题的子网或网络设备

使用的主要工具作为呼叫质量仪表板 (CQD) 联机报告并调查每个所述，音频，以最大限度地采用和影响为重点的区域重点质量经验审查指南 》 中提供的指导。 为提高音频体验到网络所做的任何优化将还直接翻译为视频和桌面共享的改进。

我们强烈建议您尽早提名质量冠军。 在被提名后他们应该开始熟悉[质量经验审查指南](https://github.com/MicrosoftDocs/OfficeDocs-SkypeForBusiness/blob/live/Teams/downloads/quality-of-experience-review-guide.docx?raw=true)中的内容。
