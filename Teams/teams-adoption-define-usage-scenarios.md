# <a name="define-usage-scenarios-for-microsoft-teams"></a>为 Microsoft Teams 定义使用方案

请确保了解业务项目 (方案) 将在实施的这一阶段范围内。 查看此示例方案列表，这些方案是早期采用者计划的优秀候选项。 你可以开始轻松获胜，例如：

- 个人工作效率
- 新式项目管理
- 新式会议

![三个核心方案的插图。](media/teams-adoption-modernizing-core-scenarios.png)

需要考虑的其他方案包括：

- 员工参与和沟通
- 更快地将市场活动推向市场
- 提高销售生产力和增加收入
- 简化业务评审

![每个方案的 Teams 用例图示。](media/teams-adoption-use-cases.png)

将这一方案现代化运动视为一个累积过程 - 让事情进入更基本的方案，以创造热情，熟悉和信誉与这种新的工作方式。 然后继续到更雄心勃勃的影响领域。 随着 Microsoft Teams 对业务同事的影响越来越大，他们中的更多人将参与进来，并且势头将会增强。 对于我们的许多客户来说，一旦他们完成此过程，就会发现业务利益干系人会在了解同事如何从 Microsoft Teams 获得价值后接近他们。

## <a name="interview-business-stakeholders"></a>采访业务利益干系人

若要确认选择这些早期项目，建议直接与你在此过程中前面确定的利益干系人会面。 此时，你的目标是倾听并了解有关其业务的其他信息。 请考虑以下问题来推动对话：

- 组织与沟通和协作相关的一些挑战或难题是什么？
- 组织希望改进哪些领域？
- 团队可以支持哪些组织的战略计划或当前转型项目？
- 组织通常比其他组织更能接受哪些通信和协作方法？
- 起草、分发和共享信息的过程是什么？

## <a name="map-and-prioritize-business-scenarios"></a>映射和确定业务方案的优先级

若要确保你拥有有关业务方案的正确信息，请考虑使用以下格式从完成工作的员工和流程的业务所有者的角度记录方案。 要制定一条成功的前进道路，需要两种视角。

![用于标识方案的框架的插图。](media/teams-adoption-identify-scenarios.png)

与一个或多个业务利益干系人交谈后，可以根据影响与困难确定方案的优先级。  (除了我们上面建议的方案之外，你的方案可能还适用。) 适合试验阶段的候选项应具有更高的影响和低到中等难度。 这样可以确保项目在能够显示工作价值之前不受范围爬行或技术困难的影响。 下面显示了一个示例图表。

![显示方案影响与难度的插图。](media/teams-adoption-impact-difficulty.png)
