# <a name="record-an-issue-ticket"></a>记录问题票证

对于任何类型的事件，MTR Pro 没有标记，或只是跟踪地铁或空间上的问题，客户或合作伙伴代表客户可以使用记录问题功能。

在“说明”字段中，尽可能地描述需要 Microsoft 帮助的问题。 可以添加一个或多个会议室。

检查  **事件** 是否导致服务或关键室的运行状况严重恶化。 若要帮助 Microsoft 正确确定请求的优先级，请详细描述影响很大的原因。

> [!NOTE]
> 不要将此字段用于一般问题、会议室和/或没有可用缓解措施的问题。

当一个房间 (或会议室) 选择为记录问题的一部分时，每个房间将被标记为 **“不正常”** ，直到记录的问题被关闭。

## <a name="closing-record-an-issue-tickets"></a>结束记录发行票证

客户或代表客户管理会议室的合作伙伴可以关闭记录的问题。

**关闭记录的发行票证**

1. 选择 **“关闭票证**”。

2. 通过从列表中选择一个类别来选择关闭的原因。

   确认关闭原因后，票证将关闭并移至 **“已解决**”。

票证的“详细信息概述”部分显示，托管会议室服务 (Microsoft) 或客户/合作伙伴的名称关闭了票证。  

## <a name="faq"></a>常见问题

**有人能关票吗？**

用户只能关闭客户报告的事件。 已分配合作伙伴以使用票证管理权限管理其房间的客户将能够关闭记录的发行票证。

**我可以限制谁可以关闭票证吗？**

目前不行。 拥有票证管理权限的所有用户都可以关闭记录的发行票证。

**当记录问题票证关闭时，是否收到通知？**

目前不行。
