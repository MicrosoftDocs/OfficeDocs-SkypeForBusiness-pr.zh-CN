# <a name="managing-room-software-stack"></a>管理会议室软件堆栈  
每个房间的软件堆栈的不同部分都协同工作以提供体验。 概括起来，可以概括如下： 

![软件堆栈摘要的屏幕截图](../media/update-management-006.jpg)

为了确保每个会议室中每天有积极的会议体验，必须标准化每个会议室上运行的每个软件，并基于配置在会议室中推出修补程序。 此外，通常还会看到客户设置自己的验证过程，通过验证它确实修复了他们可能遇到的问题或预览功能，使组织准备好进行更改，从而获得对任何挂起修复的置信度。  

托管服务Microsoft Teams 会议室上述所有操作，因此不必担心更新或排查更新问题。 在本部分，我们将介绍托管软件更新如何工作。  

## <a name="managing-to-a-good-state"></a>管理到良好状态 
我们的主要目标是让会议室保持运行并可供组织使用。 托管服务专家一直在代表你工作，以策展、验证更新，以及将更新部署到生产室。 在某些情况下，这可能意味着我们优先处理房间运行状况，而优先处理新功能和发布，然后再推出这些功能和版本进行验证。

最重要的是，这意味着不必担心花时间了解这些更新的详细信息并自己进行验证。 如果有任何特定更新和任何特定空间的问题，我们将与用户一起进行故障排除和解决。  

因此，推出更新的过程可以分为以下几部分：

- 托管服务团队会主动识别更新、验证更新，然后将其升级以发布到环境中。
- 将订购发送到你的聊天室的更新，以避免发生冲突和交互，然后计划以避免常规营业时间。
- 使用响铃系统，无需执行任何操作即可随时了解对房间应用的更新。
- 托管服务将监视更新，并自动提醒其聊天室运营中心跟进问题并解决问题。
