# <a name="managing-room-software-stack"></a>管理会议室软件堆栈  
在每个会议室中，软件堆栈的不同部分都协同工作，以提供体验。 在高级别上，可以按如下所示汇总它们： 

![软件堆栈摘要的屏幕截图](../media/update-management-006.jpg)

为了确保每个会议室每天都有积极的会议体验，必须标准化每个会议室上运行 **的每一个软件** ，并根据配置在会议室中推出修补程序。 同样常见的情况是，通过验证客户是否确实修复了可能遇到的问题或预览功能，以使组织准备好进行更改，从而设置自己的验证过程，以获得对任何挂起修复的信心。  

托管服务Microsoft Teams 会议室处理上述所有问题，因此无需担心更新或排查更新问题。 在本部分中，我们将介绍托管软件更新的工作原理。  

## <a name="managing-to-a-good-state"></a>管理到良好状态 
我们的主要目标是让会议室保持运行状态，并可供组织使用。 托管服务专家经常代表你来策划、验证更新并将其部署到生产室。 在某些情况下，这可能意味着我们将房间运行状况优先于要在推出之前验证的新功能和版本。

最重要的是，这意味着你不必担心投入时间来了解这些更新的详细信息并自行进行验证。 如果任何特定的更新和任何特定的会议室都存在问题，我们将与你合作进行故障排除并解决这些问题。  

因此，推出更新的过程可以细分为以下内容：

- 托管服务团队会主动识别更新、验证更新，然后将其升级为发布到环境。
- 汇报交付到您的房间将被命令，以避免冲突和交互，然后计划，以避免定期营业时间。
- 环形系统使你能够随时了解应用到房间的更新，而无需执行任何腿部工作。
- 托管服务将监视更新，并自动提醒其会议室运营中心跟进问题，并与你合作解决这些问题。
