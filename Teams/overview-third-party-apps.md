# <a name="overview-of-third-party-apps-in-microsoft-teams"></a>Microsoft Teams中的第三方应用概述

许多应用在Teams存储中提供应用。 这些应用由应用开发人员创建，Microsoft 将验证应用的基本功能、可用性和基本安全性。 只有在验证之后，Microsoft 才会在Teams存储中提供应用。 要使最终用户从丰富的应用生态系统中受益，管理员管理应用是关键步骤。 作为管理员，设置一个应用治理过程，以平衡最终用户的广泛应用要求以及内部 IT 策略、标准和风险配置文件的限制。

Microsoft Teams中的第三方应用不受 Microsoft 控制或拥有，不受 Microsoft Online 订阅协议的约束。 组织对每个第三方应用的使用受该应用的条款、条件和隐私声明的约束。 Teams管理中心允许你在允许其使用之前验证应用的所有权限、隐私声明和使用条款。 与往常一样，你可以允许对几个特定用户或所有用户使用应用。 对于第三方应用，对于自定义应用，Microsoft Teams允许管理员允许、禁止、治理和分析应用的使用。

除了确保管理员能够控制第三方应用外，Microsoft 还可确保应用商店中提供高质量的应用。 我们要求所有应用都经过 Microsoft 的强制验证。 我们鼓励应用成为可选合规计划的一部分，以演示对安全性和符合行业标准的额外关注。 各种程序包括：

1. [应用验证](overview-of-app-validation.md#app-validation-and-testing)
1. [Publisher验证](overview-of-app-validation.md#publisher-verification)
1. Microsoft 应用符合性计划：

   * [Publisher证明](overview-of-app-validation.md#publisher-attestation)
   * [Microsoft 365认证](overview-of-app-validation.md#microsoft-365-certification)
