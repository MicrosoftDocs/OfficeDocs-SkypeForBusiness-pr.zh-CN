# <a name="contact-center-integrations-for-microsoft-teams"></a>Microsoft Teams 的联系人中心集成

将常用联系中心解决方案与 Microsoft Teams 集成是部署 Teams 呼叫功能的客户常见的需求。  本文概述了如何将联系中心解决方案与 Microsoft Teams 集成，并提供有关参与 Microsoft Teams 连接联系中心认证计划的解决方案提供商的其他信息。

## <a name="what-is-a-contact-center-integration-for-microsoft-teams"></a>什么是 Microsoft Teams 的联系人中心集成？

如今的联系中心提供的不仅仅是支持 ， 它们充当交互和未经筛选的关于客户品牌体验的反馈的主要工具之一。 由于当今客户喜欢跨多种渠道（电话、电子邮件、文本、社交）参与，以及与当前购买流程相关的触摸点数量增加，许多组织实现了两个额外的挑战：

1. 组织的每一个成员都有可能直接与客户互动，因此需要配备适当的工具。

2. 这种扩展的客户交互范围需要有助于推动一致性、持续改进和缩放的工具。

Microsoft Teams 通过充当跨通信模式（包括聊天、视频会议和通话）的内部和外部客户连接的中心，支持客户交互工作流。 对于一些公司，Microsoft Teams[的云](./cloud-voice-landing-page.md)语音功能（包括[](./create-a-phone-system-call-queue.md)[自动](./what-are-phone-system-auto-attendants.md)助理和呼叫队列）提供功能和配置来满足其需求。

对于希望将解决方案与业务工具和工作流集成以驱动客户旅程的其他人，Microsoft Teams 还集成了一些行业领先的联系中心即服务 (CCaaS) 解决方案提供商。

### <a name="connect"></a>连接

此模型将 CCaaS 解决方案提供商与 Microsoft Teams 手机系统基础结构相连接，实现增强的路由、配置和系统见解。 

使用基于 Connect 模型的解决方案的代理可以收集信息&见解，并在必要时直接将呼叫转接给主题专家，使用 SME 在 Teams 中的存在来确保其可用性。

组织可以通过设置自动化虚拟助手和基于技能的路由队列，确保将调用路由到最佳代理。

**功能亮点：**

虽然以下不是此集成模型的功能综合列表，但重点包括：

  - 适用于代理的 Office 365 身份验证，允许代理从集成的 CCaaS 客户端连接到其 Microsoft 租户 

  - Teams 用户状态指示 

  - 支持 Teams 用户的转接和群组通话 

  - 用于与 Teams 集成的 Teams Graph API 和云通信 API 

  - 能够支持多租户 SIP 中继，以支持解决方案提供商 SBC 上的多个客户。  

  - 解决方案提供商使用 [<span class="underline">Microsoft 认证的会话边界控制器 (SBC) </span>](./direct-routing-border-controllers.md) 

### <a name="connect-and-extend"></a>连接和扩展

此模型通过使用 Microsoft Graph 中的 [Teams](/microsoftteams/platform/overview)客户端平台 [、Teams Graph](/graph/api/resources/teams-api-overview?view=graph-rest-1.0) API 和 [云通信 API](/graph/api/resources/communications-api-overview?view=graph-rest-1.0) 与 Teams 客户端集成，并针对所有联系中心呼叫和呼叫控制体验使用 Teams 电话系统，扩展了联系中心人员和代理体验。 在此模型中，联系中心解决方案提供商与 Microsoft 365 一起充当电话运营商。

通过使用基于连接和扩展的解决方案，代理可以在开始参与之前从多个系统关联数据的动态上下文笔记中获益，然后通过本机在 Teams 中为内部协作和外部通信工作来避免代价高昂的上下文切换。

组织可以设计工作流和高级路由配置（向下到个人）并测量其系统和交互的质量。

**功能亮点：**

虽然以下不是此集成模型的功能的综合列表，但它确实突出显示了主要重点领域：

  - 用于与 Teams 集成的 Teams Graph API 和云通信 API 

  - 用于代理体验的基于 Teams 的应用 

  - Teams 作为代理的主要调用终结点 

  - 用于所有呼叫控件的 Teams 客户端呼叫

  - 代理体验应用应该能够在 Teams Web 和移动客户端上工作

  - Teams 中 CCaaS 应用中的代理分析、工作流管理、基于角色的体验

  - 与 Teams 客户端集成的聊天和协作体验 

  - 在所有应用中保留 Teams 客户端体验的性能和质量  

### <a name="extend-and-power"></a>扩展和电源

此模型使解决方案提供商能够使用 Teams 呼叫基础结构和客户端平台创建基于本机 Azure 的语音应用程序，为协作客户和代理连接提供现代智能解决方案。 扩展和 Power 的目标是培养开发人员的创造力并提升客户工作效率。

通过直接在 Azure 上构建，解决方案提供商可以跨所有 Teams 地区和地理位置快速部署和预配其解决方案，同时利用 Azure 的存储、计算、分析&共享的全球通信网络。

借助扩展和 Power 集成模型，解决方案提供商可以为联系中心代理提供全通道通信体验，同时整合人工智能，以自定义参与者或其他服务参与在 Microsoft Graph 中应用云 [通信 API](/graph/api/resources/communications-api-overview?view=graph-rest-1.0)的呼叫方式和时间。

**功能亮点：**

虽然以下不是此集成模型的功能的综合列表，但除了连接和扩展模型提供的功能之外，这些功能还突出显示了这些方面。

  - 通过 Teams SDK 以本机方式为全渠道通信启用正式代理体验 

  - 使用 Teams 协作服务进行代理协作和客户交互  

  - 快速预配云服务，随时随地部署 

  - 在 Teams 对话期间直接进行对话控制和与用户交互
