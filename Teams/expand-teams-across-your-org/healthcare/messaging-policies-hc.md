<span data-ttu-id="0f5bc-104">消息传递策略用于控制 Microsoft Teams 中的用户可以使用哪些聊天和频道消息传递功能，并且它们是医疗保健组织（如医院、门诊部或医生诊所）的安全消息传递整体部署的一部分，其中，及时阅读和处理消息至关重要，同时了解关键消息何时得到阅读也是如此。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-104">Messaging policies are used to control which chat and channel messaging features are available to users in Microsoft Teams, and are part of the overall deployment of Secure Messaging for healthcare organizations like Hospitals, clinics, or doctor's offices, where having a message picked up and acted upon in a timely manner is crucial, as is knowing when crucial messages are read.</span></span>

<span data-ttu-id="0f5bc-105">你可以使用全局（组织范围的默认设置）策略，也可以为组织中的人员创建一个或多个自定义消息传递策略。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-105">You can use the global (Org-wide default) policy or create one or more custom messaging policies for people in your organization.</span></span> <span data-ttu-id="0f5bc-106">除非你创建并分配自定义策略，否则你组织中的用户将自动获取全局策略。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-106">Users in your organization will automatically get the global policy unless you create and assign a custom policy.</span></span> <span data-ttu-id="0f5bc-107">在创建自定义策略后，可为其分配组织中的一个或多个用户。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-107">After you create a custom policy, assign it a user or groups of users in your organization.</span></span> <span data-ttu-id="0f5bc-108">例如，你可以选择只允许某些工作角色使用这些功能（也许仅是医生和护士），而其他工作人员（如门卫或厨房工作人员）只能使用较为有限的一组功能。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-108">For example, you may choose to only allow certain job roles to use these features (perhaps doctors and nurses only) and other workers (like the janitorial or kitchen staff) to get a more limited set of features.</span></span> <span data-ttu-id="0f5bc-109">你可自行决定组织需要满足的需求，此处的指导最多只是建议。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-109">Decide for yourself what needs your organization has, the guidance here is at most a suggestion.</span></span>

## <a name="read-receipts"></a><span data-ttu-id="0f5bc-117">已读回执</span><span class="sxs-lookup"><span data-stu-id="0f5bc-117">Read receipts</span></span>

<span data-ttu-id="0f5bc-121">对于医疗保健应用程序，请选择“**用户控制**”或“**面向所有人打开**”。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-121">For Healthcare applications, choose either **User controlled** or **On for everyone**.</span></span> <span data-ttu-id="0f5bc-122">请注意，在使用“**面向所有人打开**”设置时，为整个租户设置回执的唯一方法是为整个租户仅设置一个消息传递策略（名为“全局（组织范围的默认设置）”的默认策略），或使租户中的所有消息传递策略都使用相同的回执设置。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-122">Be aware that when using the **On for everyone** setting, the only way to set receipts for the whole  tenant is either to have only one messaging policy for the whole tenant (the default policy named "Global (Org-wide Default)") or to have all messaging policies in the tenant use the same settings for receipts.</span></span> <span data-ttu-id="0f5bc-123">当已读回执功能启用为“**面向所有人打开**”时，该功能最有效。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-123">The read receipts feature is most effective when the feature is enabled to **On for everyone**.</span></span>

<span data-ttu-id="0f5bc-128">*有已读回执的使用示例：* 高危患者 Jakob Roth 被送进医院。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-128">*Usage example with read receipts:* Jakob Roth, a high risk patient, is admitted to the hospital.</span></span>  <span data-ttu-id="0f5bc-129">Sofia Krause 是一名护士，隶属医务工作者专业间团队 (IDT)（包括不同专家），她被指派为接洽此患者的初级护理协调员。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-129">Sofia Krause is a nurse working as part of the inter-disciplinary team (IDT) of medical workers, including different specialists, is assigned as the primary care coordinator in charge of this patient.</span></span>  <span data-ttu-id="0f5bc-130">Sofia 开始与一组医生和其他护士进行群聊，这些医生和护士将与患者合作协调护理并开始急诊分诊。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-130">Sofia starts a group chat with a set of doctors and other nurses who will be working with the patient to coordinate care and starts an emergency triage.</span></span>  <span data-ttu-id="0f5bc-131">在整个护理协调过程中，护士和医生们将就患者的护理计划进行沟通和协作。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-131">The nurses and doctors communicate and collaborate over the patient's care plan throughout the care coordination process.</span></span>  <span data-ttu-id="0f5bc-132">重要和紧急消息将通过一对一和群聊对话发送。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-132">Important and urgent messages are sent through 1:1 and group chat conversations.</span></span> <span data-ttu-id="0f5bc-133">Sofia 使用已读回执功能来确定为请求支援而发送的消息是否已传送并被目标医师或护士阅读。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-133">Sofia uses the read receipts functionality to determine if messages sent requesting support are delivered and read by the targeted physicians or nurses.</span></span> <span data-ttu-id="0f5bc-134">Jakob 的患者治疗效果几乎最佳，由于他的医疗团队沟通顺畅，他很快出院了。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-134">Jakob's patient outcomes are near-optimal and he goes home sooner because his health team communicates smoothly.</span></span>

<span data-ttu-id="0f5bc-139">管理员可以为分配了此策略的用户启用或禁用发送优先级通知的功能。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-139">An admin can enable or disable the ability for users assigned this policy to send priority notifications.</span></span> <span data-ttu-id="0f5bc-140">默认情况下，启用此功能。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-140">This feature is on by default.</span></span> <span data-ttu-id="0f5bc-141">优先级消息的收件人可能没有相同的消息传递策略，并且没有禁用接收优先级消息的选项。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-141">The recipient of the priority message might not have the same messaging policy, and will not have an option to disable receiving priority messages.</span></span> <span data-ttu-id="0f5bc-142">对于医疗保健应用程序，我们建议至少为某些用户启用该功能，但是你需要确定具体用户。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-142">For Healthcare applications, we recommend enabling the feature for at least some users, but you'll need to determine which ones.</span></span>

<span data-ttu-id="0f5bc-143">*使用示例：* Sofia Krause 再次负责接洽高危患者 Jakob Roth 入院。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-143">*Usage example:* Sofia Krause is readmitting a high-risk patient, Jakob Roth.</span></span> <span data-ttu-id="0f5bc-144">医生 Manuela Carstens 是该患者的初级保健医生。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-144">Manuela Carstens, a physician, is the primary care doctor for this patient.</span></span>  <span data-ttu-id="0f5bc-145">Sofia 使用优先级通知向 Manuela 发送了一条消息，要求立即对 Jakob 进行分诊。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-145">Sofia sends a message to Manuela using a priority notification asking for immediate help with triage of Jakob.</span></span>  <span data-ttu-id="0f5bc-146">Manuela 的电话收到了该消息，但 Manuela 并未感觉到电话振动，因此没有回复。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-146">Manuela's phone receives the message but Manuela didn't feel the phone vibration and does not reply.</span></span> <span data-ttu-id="0f5bc-147">Teams 将重新通知 Manuela，并且会持续不断地重新通知，直到她阅读了该消息。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-147">Teams re-notifies Manuela and will continue to persistently re-notify until she reads the message.</span></span> <span data-ttu-id="0f5bc-148">如果还启用了已读回执，则即使在 Manuela 决定如何响应之前，Sofia 就会知道 Manuela 已阅读该消息。</span><span class="sxs-lookup"><span data-stu-id="0f5bc-148">If read receipts are also enabled, Sofia can be aware that the message was read by Manuela, even before Manuela decides how to respond.</span></span>
