# <a name="microsoft-staffhub-to-be-retired"></a>Microsoft StaffHub 将停用

有效 2019 年 10 月 1，，Microsoft StaffHub 将要停用。 我们 StaffHub 功能构建到 Microsoft 团队。 如今，团队包括日程管理引进相关应用程序和其他功能将随着时间的推移推出。

这些更改是努力继续为提供强大功能与 Microsoft 365 每位员工的一部分。 现在，在工作组 Firstline 工作者的功能，您的组织中的每个员工都将能够使用团队优化其工作日，与同事和访问信息和专业技能，以帮助他们工作做到最好协作。

## <a name="frequently-asked-questions"></a>常见问题

### <a name="about-staffhub-retirement"></a>有关 StaffHub 退休

#### <a name="when-will-staffhub-be-retired"></a>何时停 StaffHub？

启动年 4 月 2019年中间 Microsoft StaffHub 将不再适用于新的租户。 当前用户仍然能够年 10 月之前使用服务 1,2019，但是没有其他功能将可供。

#### <a name="what-will-happen-once-staffhub-is-retired"></a>一旦 StaffHub 停用，则会发生什么情况？

Microsoft StaffHub 上 2019 年 10 月 1，将停止的所有用户的工作。 尝试打开应用程序的任何人都将显示一条消息，来下载团队。 如果用户具有的活动许可证包含团队和组织采取相应的步骤，以使团队，其数据，从 StaffHub 的核心功能将提供给他们时其过渡到团队。

2019 年 4 月和年 10 月 2019年，之间 StaffHub 用户将收到应用程序中通知鼓励他们使用团队。

#### <a name="will-teams-offer-all-the-functionality-currently-offered-in-staffhub"></a>将团队提供当前 StaffHub 中提供的所有功能？

年 1 月 2019年团队开始提供引进了以前在 Microsoft StaffHub 中可用。 此功能允许管理员规划和分发 shift 计划和团队能够查看其日程安排并进行更改的计划请求从其移动设备的成员。 若要了解详细信息，请参阅[管理您的组织在工作组引进相关应用程序](manage-the-shifts-app-for-your-organization-in-teams.md)。

我们承诺为提供出色的用户体验，以及将向团队的 StaffHub 各自的独特功能。 在其功能将成为团队中可用的详细信息，请参阅[很快就变成团队 Firstline 辅助功能](firstline-worker-features-coming-to-teams.md)。

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>需要执行的操作，此更改准备的什么？

若要了解如何从 StaffHub 到团队中的引进准备转换，转到[Prepare to 移动引进您 StaffHub 团队](move-staffhub-teams-to-shifts-in-teams.md#prepare)。

#### <a name="why-cant-users-in-my-organization-access-their-staffhub-team-from-shifts-in-teams"></a>为什么我的组织中的用户不能从引进团队中访问其 StaffHub 工作组？

仅可以查看和管理 StaffHub StaffHub 创建的团队。 后 StaffHub 团队移至团队，用户将能够查看其日程安排中引进。 我们建议您进行通信此更改早期和现有用户熟悉团队。

### <a name="about-shifts"></a>有关引进

#### <a name="what-is-shifts"></a>什么是引进？

团队中的班次应用程序是保留 Firstline 工作者连接和同步的计划管理工具。管理员创建、 更新和管理团队，shift 计划和工作组成员可以查看其日程安排，请参阅其他还有谁已安排在一天、 请求交换或提供 shift 键，并请求关闭所有从其移动设备的时间。  

#### <a name="which-plans-is-shifts-available-in"></a>该计划是引进中可用？

在下面的示例的工作组的一部分计划而无需任何其他成本是可用引进：

有关详细信息，请参阅[班次的可用性](manage-the-shifts-app-for-your-organization-in-teams.md#availability-of-shifts)。

如果您已在包含引进计划中，您可以开始使用引进今天团队的一部分。 签出[班次的开始](https://support.office.com/en-us/article/get-started-in-shifts-5f3e30d8-1821-4904-be26-c3cd25a497d6)。

#### <a name="which-devices-or-platforms-is-shifts-available-on"></a>哪些设备或平台位于引进上？

引进位于团队 web 客户端、 团队桌面客户端和团队移动客户端 （iOS 和 Android） 上。
 
#### <a name="are-there-differences-between-the-shifts-desktop-and-web-apps-and-the-mobile-ios-and-android-apps"></a>是否有引进桌面和 web 应用程序和移动 （iOS 和 Android） 应用程序之间的区别？

对于经理，优化的桌面和 web 应用程序创建团队、 添加成员、 生成和发布计划，并与团队共享内容。 经理通常使用移动应用程序审批请求、 向组发送快速消息、 呼叫团队成员和在协作。 

工作组成员移动应用程序经过优化，可查看其日程安排、 查看谁正在使用一天、 入和签出时钟、 设置可用性、 请求关闭的时间、 交换或提供引进和打开引进。 有用户可以在移动应用程序与桌面和 web 应用程序执行的操作的差异。 我们正在从事功能奇偶校验。

#### <a name="are-there-differences-between-the-shifts-ios-app-and-the-android-app"></a>是否有引进 iOS 应用程序和 Android 应用程序之间的区别？

核心功能是之间引进 iOS 和 Android 应用相同的。 有两个应用程序对齐到它们的推荐的设计模式之间的次要用户界面差异。

引进适用于所有浏览器的团队支持包括 Microsoft 边缘、 Internet Explorer、 Google Chrome 和 Mozilla Firefox 的最新版本。 有关详细信息，请参阅[获取团队的客户端](../../get-clients.md)。

#### <a name="how-do-i-move-my-organizations-staffhub-teams-to-teams"></a>如何将我的组织 StaffHub 团队移到团队？

默认情况下，所有组织启用团队。 团队许可证分配给用户，以控制对团队的单个访问。 有关详细信息，请参阅[您的组织中启用团队](../../Office-365-set-up.md)和[团队管理用户访问](../../user-access.md)。

团队 for Business 支持与 Skype 的共存情况。 有关详细信息，请参阅[了解团队和 Skype 的业务共存及互操作性](../../teams-and-skypeforbusiness-coexistence-and-interoperability.md)和[迁移和组织使用团队一起 for Business 的 Skype 的互操作性指南](../../migration-interop-guidance-for-teams-with-skype.md)。

#### <a name="does-each-user-need-to-have-an-office-365-account-or-azure-active-directory-azure-ad-account"></a>每个用户是否需要具有 Office 365 帐户或 Azure Active Directory (Azure AD) 帐户？

是，使用团队和引进团队中的每个管理器和团队成员必须具有 Office 365 帐户 (Azure AD)，并且必须分配有效的许可证。
 
#### <a name="in-staffhub-i-could-add-a-special-prefix-to-all-office-365-groups-that-were-created-by-staffhub-teams-can-i-do-the-same-in-teams"></a>在 StaffHub，无法向 StaffHub 团队创建的所有 Office 365 组添加特殊的前缀。 我如何在工作组中相同？

从 StaffHub 移至团队时，不会具有相同的功能。 如果您需要将名称一致性应用于所有 Office 365 组，则可以使用[Office 365 组命名策略](https://docs.microsoft.com/office365/admin/create-groups/groups-naming-policy)。

#### <a name="will-team-names-be-the-same-in-teams"></a>工作组名称将在工作组中相同？

则可能的工作组名称将有所不同。 团队中的工作组名称使用 Office 365 组名称。 如果 StaffHub 工作组名称不同的相应的 Office 365 组的名称，您将看到一个不同的名称，当切换到团队。

#### <a name="what-happens-to-team-membership-when-a-staffhub-team-is-moved-to-teams"></a>如果 StaffHub 团队移至团队到团队成员资格的怎么办？

当 StaffHub 团队移至团队，将保持团队成员资格。 小组成员的电话号码不会显示在引进。 如果 StaffHub 电话号码都将存储在 Azure AD 用户帐户的 Azure AD 中匹配的电话号码。 

我们建议您查看您现有的团队名单上 StaffHub 以及尽可能对齐电话号码是为了相关的 Azure AD 身份。
 
#### <a name="what-happens-to-staffhub-team-members-who-arent-provisioned-or-whose-status-is-inactive"></a>StaffHub 工作组成员人员未设置或其状态处于非活动状态会发生什么情况？

用户的计划将被移动到引进，但是，用户不是团队或 Office 365 组的成员。

对于不链接到的 Azure AD 用户帐户的非活动帐户，经理可以确保通过在 StaffHub 团队成员页上添加一个有效的 Office 365 电子邮件地址，以便用户链接到正确的 Azure AD 将用户添加团队的成员身份的帐户红外组织。

另一个选项是对于管理员运行**添加 StaffHubMember**和**删除 StaffHubUser** cmdlet 从 StaffHub 团队中移除的非设置的帐户和添加用户回其 UPN 与团队。

我们建议您确保您的组织中的所有用户都涵盖与相应的许可，以减少中断和访问服务。
 
#### <a name="what-happens-to-chats-in-staffhub"></a>聊天中 StaffHub 会发生什么情况？

为了帮助转换用户，我们计划迁移到团队中 StaffHub 数据聊天历史记录。 将在工作组中可用的聊天历史记录量取决于多久配置您的组织转换。 建议操作不晚于年 6 月 30，2019 90 天的数据通过 2019 年 10 月 1，提供足够的时间转换。

确保 Office 365 组创建已启用在 Microsoft 365 管理中心转换在开始之前，因为这是我们能够 StaffHub 聊天数据移到团队至关重要。 未能打开由 2019，年 6 月 30，此服务将导致更高版本的迁移日期或更少的通过复制的聊天数据。
