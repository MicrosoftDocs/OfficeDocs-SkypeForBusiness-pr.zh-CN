# <a name="plan-for-local-media-optimization-for-direct-routing"></a>规划用于直接路由的本地媒体优化

公共交换电话网络 (PSTN) 语音被视为业务关键型应用程序，对语音质量寄予厚望。 通过直接路由，你可以控制媒体流量流，以适应世界各地各种企业的众多网络拓扑和本地电话设置。

通过直接路由的本地媒体优化，可通过以下方法管理语音质量：

- 控制 Teams 客户端和客户会话边界控制器 (SBC) 之间的媒体流量流动方式。
- 将本地媒体保留在公司网络子网的边界内。
- 允许 Teams 客户端和 SBC 之间的媒体流，即使 SBC 位于具有专用 IP 且 Microsoft 不直接可见的公司防火墙后面。

本地媒体优化支持两种方案：

- 通过连接到主会话启动协议 (SIP) 中继的集中式 SBC 集中处理所有本地中继，为公司的所有本地分支机构提供电话服务。

- 构建 SBC 的虚拟网络拓扑，其中本地分支机构中的 SBC 通过其外部 IP 地址连接到一个集中代理 SBC，该 SBC 对 Microsoft Phone 系统可见并与之通信。 在虚拟网络拓扑中，下游 SBC 通过内部 IP 进行通信，对电话系统不直接可见。

本文介绍功能功能以及客户方案和解决方案。 有关配置的详细信息，请参阅 [配置本地媒体优化](direct-routing-media-optimization-configure.md)。

  > [!NOTE]
  > 如果要将本地媒体保留在 Intranet 的边界内，建议进行本地媒体优化。 如果已有“媒体旁路”且仅使用 SBC 的公共 IP 地址，则无需迁移到本地媒体优化。 可以继续使用媒体旁路。 有关详细信息，请参阅 [计划媒体旁路](direct-routing-plan-media-bypass.md)。

有关哪些 SBC 供应商支持本地媒体优化的信息，请 [参阅经过直接路由认证的会话边界控制器](direct-routing-border-controllers.md)。

## <a name="supported-customer-scenarios"></a>支持的客户方案

对于此讨论，假定 Contoso 在全球运营多个业务，如下所示。  (请注意，欧洲和 APAC 区域仅用作示例。 公司可能有几个不同区域具有类似的要求。) 

- **在欧洲**，Contoso 在大约 30 个国家/地区设有办事处。 每个办公室都有自己的专用分支机构 Exchange (PBX) 。

  Contoso 提供了一个选项，用于将所有 30 个欧洲办事处的中继集中到一个位置（阿姆斯特丹）。 Contoso 在阿姆斯特丹部署了 SBC，提供了足够的带宽来通过集中位置运行呼叫，将中央 SIP 中继连接到集中位置，并开始从阿姆斯特丹为所有欧洲位置提供服务。

- **在 APAC 区域**，Contoso 在不同国家/地区设有多个办事处。

  在许多国家，该公司仍有时间划分的多路复用 (TDM) 本地分支机构的中继。 在 APAC 区域中，TDM 中继的集中化不是一个选项，因此无法切换到 SIP。 假设整个 APAC 区域有 50 多个 Contoso 分支机构， (SBC) 数百个网关。 在这种情况下，由于缺少公共 IP 地址和/或本地 Internet 中断，无法将所有网关配对到直接路由接口。 此外，一些国家实施的监管要求，如果不建立本地 PSTN 网络连接，则无法满足这些要求。

Contoso 根据其业务要求实施了两个解决方案，其中包含用于直接路由的本地媒体优化：

- **在欧洲**，所有中继都是集中式的，媒体流在中央 SBC 和用户之间，基于用户位置。

  - 如果用户连接到公司网络的本地子网 (即，则用户是内部) ，媒体会在中央 SBC 的内部 IP 和用户的 Teams 客户端之间流动。

  - 如果用户不在公司网络的边界之外（例如，如果用户使用的是公共无线 Internet 连接），则该用户被视为外部用户。 在这种情况下，介质在中央 SBC 和 Teams 客户端的外部 IP 之间流动。

- **在 APAC 区域** 中，集中代理 SBC 与 Microsoft 直接路由配对，后者在直接路由接口与本地分支机构的下游 SBC 之间定向媒体。

  本地分支机构中的下游 SBC 对 APAC 中的直接路由不直接可见，但它们通过使用 Set-CSOnlinePSTNGateway cmdlet 在 Microsoft Phone 系统中创建虚拟网络拓扑进行配对。 媒体始终尽可能保持本地化。 外部用户的媒体在 Teams 客户端和代理 SBC 的公共 IP 之间流动。

## <a name="local-media-optimization-modes"></a>本地媒体优化模式

本地媒体优化支持两种模式：

- **模式 1：始终绕过**。 在这种情况下，如果用户是内部用户，则无论内部用户的实际位置如何，媒体都将流经本地下游 SBC 的内部 IP 地址：例如，在下游 SBC 所在的同一分支机构中，或者在其他一些分支机构中。

- **模式 2：仅适用于本地用户**。 在此模式下，只有当内部用户与下游 SBC 位于同一分支机构时，媒体才会直接流向本地下游 SBC 的内部 IP 地址。

为了区分本地媒体优化模式，租户管理员需要使用Set-CSonlinePSTNGateway cmdlet 将每个 SBC 的 -BypassMode 参数设置为“Always”或“OnlyForLocalUsers”。 有关详细信息，请参阅 [配置本地媒体优化](direct-routing-media-optimization-configure.md)。

> [!NOTE]
> 当用户是内部用户时， **需要** 用户与 SBC 之间通过内部 IP 地址建立媒体连接。 在这种情况下，媒体的公共交通中继没有回退，因为 SBC 将为媒体连接提供内部 IP。

如果本地分支机构之间存在不良连接，但每个本地分支机构与区域办事处之间连接良好，则建议的模式是“仅供本地用户使用”。

例如，在 APAC 区域中，假设 Contoso 在不同国家/地区设有多个办事处。 对于许多国家来说，无法切换到 SIP，因为该公司在许多本地分支机构中仍有 TDM 中继。 在 APAC 区域中，TDM 中继的集中化不是一个选项。 此外，APAC 区域有 50 多个 Contoso 分支机构， (SBC) 数百个网关。

为了构建一个解决方案，其中 PSTN 服务在 APAC 区域的所有本地分支机构中提供，其中 TDM 中继的集中化不是一个选项，Contoso 管理员将新加坡的一个区域 SBC 配对为直接路由服务的代理 SBC。 本地分支机构之间的直接连接并不好，但每个本地分支机构与新加坡的区域 SBC 之间有着良好的连接。 对于区域 SBC，管理员选择“始终绕过”模式，对于本地下游 SBC，管理员选择“仅供本地用户使用”模式。

下面介绍了两种方案：

- 方案 1。 用户与联机语音路由策略中定义的 SBC 位于同一位置

- 方案 2。 用户和网关位于不同的站点中
