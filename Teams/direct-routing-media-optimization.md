# <a name="local-media-optimization-for-direct-routing"></a>直接路由的本地媒体优化

公共电话交换 (PSTN) 语音被视为业务关键型应用程序，对语音质量有很高的期望。 直接路由允许控制媒体流量流，以适应世界各地各种企业的各种网络拓扑和本地电话设置。 

直接路由的本地媒体优化允许通过：

-   控制媒体流量在客户端Teams客户会话边界控制器之间流动 (SDC) 。
-   将媒体保留在本地企业网络子网的边界内。
-   允许媒体流Teams客户端和 SDC 之间，即使 SDC 位于具有专用 IP 的企业防火墙后面，并且无法直接对 Microsoft 可见。

本地媒体优化支持两种方案：

- 通过连接到主要会话启动协议 (SIP) 中继的集中式 SBC 集中所有本地中继 - 向公司的所有本地分支机构提供电话服务。

-   构建 SDC 的虚拟网络拓扑-其中，本地分支机构中的 SBC 连接到通过外部 IP 地址对 Microsoft 电话 System 可见并进行通信的集中式代理 SBC。 在虚拟网络拓扑中，下游 SDC 通过内部 IP 通信，并且无法直接电话系统。

本文介绍功能、客户方案和解决方案。 有关配置的详细信息，请参阅 [配置本地媒体优化](direct-routing-media-optimization-configure.md)。 

  > [!NOTE]
  > 如果要将媒体保留在本地 Intranet 的边界内，建议使用本地媒体优化。 如果已有媒体旁路，并且仅使用 SDC 的公共 IP 地址，则不强制移动到本地媒体优化。 可以继续使用媒体旁路。 有关详细信息，请参阅规划 [媒体绕过](direct-routing-plan-media-bypass.md)。


## <a name="supported-customer-scenarios"></a>支持的客户方案

对于此讨论，假设 Contoso 在全球运行多个业务，如下所示。  (注意，欧洲和 APAC 区域仅用作示例。 公司可能有几个具有类似要求的不同区域) 
 
- **在欧洲，Contoso** 在大约 30 个国家/地区设有办公室。 每个办公室都有自己的专用分支Exchange (PBX) 。 

  Contoso 提供了一个选项，用于集中所有 30 个欧洲办事处的一个位置（阿姆斯特丹）的中继。 Contoso 在阿姆斯特丹部署了 SBC，提供了足够的带宽来通过集中位置运行呼叫，将中心 SIP 中继连接到集中位置，并开始从阿姆斯特丹服务所有欧洲位置。 

- **在 APAC 区域，Contoso** 在不同的国家/地区有多个办公室。 

  在多个国家/地区，公司仍在本地分支机构 (TDM) 多路复用。 在 APAC 区域，TDM 中继的集中化不是一个选项，因此无法切换到 SIP。 假设 APAC 区域有 50 多个 Contoso 分支机构，数百个网关 (SDC) 。 在此方案中，由于缺少公共 IP 地址和/或本地 Internet 突围，无法将所有网关配对到直接路由接口。 此外，一些国家/地区施加了在没有本地 PSTN 网络连接的情况下无法满足的法规要求。

Contoso 根据业务要求，通过直接路由的本地媒体优化实现了两个解决方案：

- **在欧洲**，所有中继都是集中的，媒体流在中心 SBC 和用户之间，基于用户位置。 

  - 如果用户连接到企业网络本地子网 (即用户是内部) ，则媒体在中心 SBC 的内部 IP 与用户的 Teams 客户端之间流动。 
  
  - 如果用户位于企业网络边界之外（例如，如果用户使用公共无线 Internet 连接）则该用户被视为外部用户。 在这种情况下，媒体在中心 SBC 的外部 IP 与中心客户端Teams流。

- **在 APAC 区域**，集中式代理 SBC 与 Microsoft Direct Routing 配对，该路由在本地分支机构中的直接路由接口和下游 SBC 之间引导媒体。 

  本地分支机构中的下游 SDC 对 APAC 中的直接路由不可见，但它们是通过使用 Set-CSOnlinePSTNGateway cmdlet 在 Microsoft 电话 System 内创建虚拟网络拓扑进行配对的。 在可能的情况下，媒体始终保持本地。 外部用户的媒体在客户端Teams代理 SBC 的公共 IP 之间流动。

## <a name="local-media-optimization-modes"></a>本地媒体优化模式

本地媒体优化支持两种模式：

- **模式 1：始终绕过**。 在这种情况下，如果用户是内部用户，则媒体将流经本地下游 SBC 的内部 IP 地址，而不考虑内部用户的实际位置;例如，在下游 SBC 所在的同一分支机构内或其他一些分支机构中。  

- **模式 2：仅适用于本地用户**。 在此模式下，只有当位于下游 SBC 的同一分支机构中的内部用户生成时，媒体才直接流向本地下游 SBC 的内部 IP 地址。 

若要区分本地媒体优化模式，租户管理员需使用 Set-CSonlinePSTNGateway cmdlet 将每个 SBC 的 -BypassMode 参数设置为"Always"或"OnlyForLocalUsers"。 有关详细信息，请参阅 [配置本地媒体优化](direct-routing-media-optimization-configure.md)。  

> [!NOTE]
> 当用户是内部用户时，用户与 SBC 之间需要通过内部 IP 地址建立媒体 **连接**。 在这种情况下，媒体的公共传输中继没有回退，因为 SBC 将为媒体连接提供内部 IP。 

### <a name="mode-1-always-bypass"></a>模式 1：始终绕过

如果分支机构之间的连接良好，建议的模式始终绕过。
 
例如，假设一家公司在阿姆斯特丹有一个集中式 SIP 中继，该中继为 30 个国家/地区服务，并且在所有 30 个网站与本地用户之间具有良好的连接。 在德国还有一个分支，其中部署了本地 SBC。

可以"始终绕过"模式配置德国的 SBC。 无论用户位于什么位置，都将通过 SBC 服务的内部 IP 地址直接连接到 SBC， (例如，从法国到德国;请参阅下图，了解) 。

下面描述了两种方案：

- 方案 1. 用户与联机语音路由策略中定义的 SBC 位于同一位置。

- 方案 2. 用户和网关在不同的站点中。

### <a name="mode-2-only-for-local-users"></a>模式 2：仅适用于本地用户

如果本地分支机构之间的连接不佳，但每个本地分支机构与区域办事处之间的连接良好，则建议的模式为"仅适用于本地用户"。

例如，在 APAC 区域，假设 Contoso 在其他国家/地区有多个办公室。 对于多个国家/地区，无法切换到 SIP，因为公司在许多本地分支机构中仍有 TDM 中继。 在 APAC 区域，TDM 中继的集中化不是一个选项。 此外，APAC 区域有 50 多个 Contoso 分支机构，数百个网关 (SDC) 。 

要构建一个解决方案，其中 PSTN 服务在 APAC 区域的所有本地分支机构中提供，其中 TDM 中继的集中化不是一个选项，Contoso 管理员将新加坡的一个区域 SBC 配对为代理 SBC 到直接路由服务。 本地分支机构之间的直接连接并不良好，但每个本地分支机构与新加坡的区域 SBC 之间具有良好的连接。 对于区域 SBC，管理员选择"始终绕过"模式，对于本地下游 SBC，管理员选择"仅针对本地用户"模式。

下面描述了两种方案：

- 方案 1. 用户位于与联机语音路由策略中定义的 SBC 相同的位置

- 方案 2. 用户和网关在不同的站点中
