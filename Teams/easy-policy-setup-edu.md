# <a name="use-the-teams-for-education-policy-wizard-to-easily-apply-policies-for-a-safe-learning-environment"></a>使用 "团队 for 教育策略" 向导轻松地为安全的学习环境应用策略

[!INCLUDE [preview-feature](includes/preview-feature.md)]

## <a name="overview"></a>概述

Microsoft "适用于教育版的团队策略" 向导简化了对学生和教育版的管理策略。 使用它轻松快速地应用与创建安全和高效学习体验相关的最重要的策略集。

团队中的策略允许你控制团队在你的环境中的行为以及用户可以使用哪些功能。 例如，有一些通话策略、会议策略和邮件策略，可以命名一些策略区域，并且可以自定义每个策略区域以满足组织的需求。

若要维护安全且具有重点的学习环境，请务必设置策略来控制学生可以在团队中执行的操作。 例如，您可以使用策略控制谁可以使用私人聊天和私人通话，哪些人可以安排会议，以及可以共享哪些内容类型。 您还可以使用策略来打开丰富学习体验的团队功能。

必须为学生和教育人士调整策略，以确保学习体验的安全。 学生的政策需要更多的限制才能降低收到不当访问级别的风险。 教育版和员工需要一组单独的策略，这些策略可以更好地使其成功。 例如，允许教师安排会议和限制学生执行此操作。

:::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="向导的屏幕截图":::

本文将向您介绍如何运行向导。

> [!IMPORTANT]
> 向导应用的策略将满足教育客户的大多数团队的需求。 该向导将调整全球 (组织范围的默认) 定义一组核心策略，这些策略具有我们为学生安全建议的设置，并将其应用于学生。 该向导还为教师和员工创建并分配一组自定义策略。 大多数教育客户的团队在运行此向导之后不需要使用其他策略分配方法。 仅当你想要为学生、教育和员工手动创建和管理策略时， *才* 使用其他策略分配方法。

## <a name="teams-for-education-policy-wizard"></a>"团队教育策略" 向导

<a name="polwiz_intro"> </a>

该向导将一组核心策略定义应用于教师和员工，并将一组核心策略定义应用于教育版和员工，并具有适用于每个的设置。 下面是运行向导时发生的情况。

该向导根据教育机构类型 (**主要或次要** 或 **更高的教育**) 来设置策略。 选择 "机构类型"，向导将执行下列操作：

- **学生**：该向导调整全局 (组织范围的默认) 策略定义，该策略定义由向导覆盖的每个策略区域，具有适用于使学生安全的新默认设置。 这可确保当前学生和所有新学生获得最严格的一组策略。
- **教育版和员工**：向导为向导所涵盖的每个策略区域创建一组自定义策略定义，该策略定义的设置根据教育版和教职员工的需求而定制。 然后，它将策略定义分配给你选择的教师和员工组。 通过这种方式，教师和教职员工可以获得更好的策略集以使其成功。

您只需运行向导一次。 新学生将自动获得由向导应用的全局 (组织范围默认) 策略定义，并且添加到所选组的新职员将自动分配自定义策略。

> [!NOTE]
> 请参阅 [向导应用的策略](#policies-applied-by-the-wizard) ，以获取由向导应用的策略定义的详细列表。

现在，让我们开始吧！

## <a name="run-the-wizard"></a>运行向导

<a name="polwiz_run"> </a>

按照以下步骤运行向导。

1. 如果您不熟悉团队，向导将自动启动。 否则，你可以从仪表板随时启动向导。 在 Microsoft 团队管理中心的左侧导航中，转到 **仪表板**，然后在 **安全学习环境磁贴的简易策略设置** 中，选择 " **快速设置**"。

    :::image type="content" source="media/easy-policy-setup-quick-setup.png" alt-text="仪表板中的向导的屏幕截图":::

2. 选择教育机构类型 (**主要或辅助** 或 **更高版本的教育**) ，然后选择 " **下一步**"。

    :::image type="content" source="media/easy-policy-setup-institution-type.png" alt-text="向导中的页面的屏幕截图，用于选择机构类型":::

3. 搜索并选择包含教师和教职员工的组，然后选择 " **下一步**"。 如果尚未为教师和员工设置任何组，请 [创建一个组](https://docs.microsoft.com/microsoft-365/admin/create-groups/create-groups)，然后重新运行该向导。 <br/><br/>目前，您只能选择一个组。 将为你选择的组中的教育版和员工分配 [一组自定义策略](#policies-applied-by-the-wizard) ，以满足其需求。 请记住，此策略集与学生应用的策略不同。

    :::image type="content" source="media/easy-policy-setup-group.png" alt-text="向导中的页面的屏幕截图，用于选择 "教师和教职员工" 组":::

4. 查看您的选择。

    :::image type="content" source="media/easy-policy-setup-review-selections.png" alt-text="向导中页面的屏幕截图查看选择":::

5. 选择 " **应用** " 以应用所做的更改。 这可能需要几分钟才能完成。<br/><br/>全局 (组织范围的默认) 策略定义将立即应用于学生。 对于教师和员工，根据组的大小，将自定义策略分配给所选组的每个成员最多可能需要48小时。 在你成功完成此步骤后，在后台发生此情况。
6. 你正在进行操作，但尚未完成！ 还有一些其他事项需要考虑。 接下来，请查看本文的 " [运行向导后的操作"](#what-to-do-after-running-the-wizard) 部分中的步骤。

    :::image type="content" source="media/easy-policy-setup-on-way.png" alt-text="向导中第二步的屏幕截图":::

## <a name="what-to-do-after-running-the-wizard"></a>运行向导后的操作

<a name="polwiz_remove"> </a>

### <a name="step-1-remove-existing-policy-assignments-that-conflict-with-policies-applied-by-the-wizard"></a>步骤1：删除与向导应用的策略冲突的现有策略分配

> [!IMPORTANT]
> **只有在运行向导 *之前* 为学生或教育版和员工分配了现有策略，才能完成此步骤**。 如果你不熟悉团队，并且没有任何现有策略，而不是向导创建的策略，请跳过此操作并转到步骤2。

在团队中，对于给定的策略区域，可通过以下方式将策略应用于用户：

- 直接分配给用户
- 分配给用户是其成员的组
- 如果用户未直接分配策略或不是分配有策略的任何组的成员，则用户会自动获取全局 (组织范围的默认) 策略

如果一个用户存在这些策略分配中的多个，团队将使用以下顺序来确定哪个策略分配会生效。 有关详细信息，请参阅 [哪些策略优先？](assign-policies.md#which-policy-takes-precedence) 和 [优先规则](assign-policies.md#precedence-rules)。

|用户的策略分配|生效的策略 |
|---------|---------|
|分配给组的策略：否<br/>直接分配给用户的策略：否    |全局 (组织范围) 默认策略      |
|分配给组的策略：否<br/>已将策略直接分配给用户：是    |直接分配给用户的策略         |
|分配给组的策略：是<br/>已将策略直接分配给用户：是     |直接分配给用户的策略         |
|分配给组的策略：是<br/>直接分配给用户的策略：否     |分配给组的策略<br/><br/>如果用户是多个组的成员，并且每个组都分配有同一策略区域的策略，则具有最高 [组工作分配排名](assign-policies.md#group-assignment-ranking) 的策略将会生效。       |

由于此顺序，如果用户已有直接分配或组分配，则由向导创建的策略将不会生效。 这意味着您必须删除用户的现有策略分配，这样向导应用的策略才会生效。

对于向导应用的每个 [策略区域](#policies-applied-by-the-wizard)，请执行下列操作：

- 从学生中删除所有现有直接分配和组分配，以便向导应用的全局 (组织范围默认) 策略定义生效。
- 删除教师和员工的任何冲突直接分配，以使向导创建的自定义策略定义生效。 使用上表来确定适用于你的方案。 <br/><br/>请记住，向导使用 [组分配排名](assign-policies.md#group-assignment-ranking) 1，将策略分配给教师和教职员工组，这是最高排名。 如果您的 "教师和教职员工" 组已为其分配了相同策略区域的现有策略，则该现有策略将移到较低级别，并且向导分配的策略将生效。

[了解](batch-group-policy-assignment-edu.md#remove-a-policy-that-was-directly-assigned-to-users) 有关如何删除直接分配给用户的策略的详细信息。

例如，你将会议策略直接分配给教育机构，并且你的学生具有全局 (组织范围内的默认) 会议策略。 在此方案中，删除您直接分配给教育版的会议策略，以便向导创建的会议策略的自定义策略定义生效。 您无需对学生的现有全局 (组织范围默认) 会议策略执行任何操作，因为其他会议策略没有冲突。

<a name="polwiz_addmeasures"> </a>

### <a name="step-2-check-for-additional-measures-that-you-can-take-for-student-safety"></a>步骤2：检查是否可以为学生安全执行的其他措施

该向导将自动调整和应用 [这些策略](#policies-applied-by-the-wizard)。 还有一些其他措施，根据您的机构的需求保持安全，您可能需要执行这些措施。

请参阅 [让学生保持安全，同时使用团队进行距离学习](https://support.microsoft.com/office/keeping-students-safe-while-using-teams-for-distance-learning-f00fa399-0473-4d31-ab72-644c137e11c8#ID0EBBAAA) 以了解其他安全建议。

<a name="polwiz_mc"> </a>

### <a name="step-3-check-message-center-for-policy-updates"></a>步骤3：检查消息中心的策略更新

当前，向导在运行时应用我们推荐的策略。 请务必知道，随着新策略在团队中可用，"全局 (组织范围的默认) 设置" 学生安全不会由向导自动添加。 此功能将在未来版本中提供。

在此功能可用之前，请查看 Microsoft 365) 管理中心中的 " [消息中心](https://admin.microsoft.com/AdminPortal/Home?#/MessageCenter) " (，以了解团队中新策略和策略设置的最新信息。 新功能推出后，您可能需要手动更新策略，以确保您的学习环境安全。

## <a name="make-changes-in-the-wizard"></a>在向导中进行更改

<a name="polwiz_change"> </a>

如果在运行向导后需要进行更改，则可以重新运行它并更改所做的选择。

1. 在 Microsoft 团队管理中心的左侧导航中，转到 **仪表板**，然后在 **安全学习环境磁贴的简易策略设置** 中，选择 " **更改**"。
2. 在此处，继续浏览向导的每一页以进行更改。 你可以更改你的机构类型、要为其分配策略的教师和员工组，或两者。

下表总结了在向导中进行更改时会发生的情况。

|更改类型  |策略行为  |
|---------|---------|
|同时更改教育机构类型和教育版和教职员工组    |<ul><li>**学生**：根据新的教育机构类型，全球 (组织范围的默认) 策略定义将应用于学生。</li><li>**教育版和员工**：创建一组基于新教育机构类型的自定义策略定义，并将其分配给新的 "教育机构" 和 "教职员工" 组。 以前的自定义策略定义将从以前的 "教育版和教职员工" 组中删除。</li></ul>    |
|仅更改教育机构类型    |<ul><li>**学生**：根据新的教育机构类型，全球 (组织范围的默认) 策略定义将应用于学生。</li><li>**教育版和员工**：创建一组基于新教育机构类型的自定义策略定义，并将其分配给 "教育机构" 和 "教职员工" 组。 为以前的教育机构类型创建的自定义策略定义将从 "教育版和教职员工" 组中删除。</li></ul>         |
|仅更改 "教育版" 和 "教职员工" 组   |<ul><li>**学生**：不会更改应用于全球 (组织范围的默认) 策略定义。</li><li>**教育版和员工**：将自定义策略定义分配给新的 "教育版" 和 "教职员工" 组，并从以前的 "教育版和教职员工" 组中删除。</li></ul>         |
