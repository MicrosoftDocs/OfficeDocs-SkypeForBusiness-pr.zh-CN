# <a name="steering-committee"></a>指导委员会

## <a name="steering-committee-overview"></a>指导委员会概述

指导委员会是一个由主要利益干系人和项目负责人组成的管理小组，他们聚集在一起，指导项目或计划实现其定义的业务成果。 指导委员会不直接负责项目交付 *方式* ，而是项目向企业提供 *的内容* 。

每个项目都需要一个商定的愿景和宪章。 若要提供项目所需的结果，必须明确定义愿景，需要对其进行监视和维护。 这成为指导委员会的责任：推动决策、提供建议、提供战略监督、为组织的倡议充当倡导者，并在必要时删除阻止程序。

你的组织应该对指导委员会的组建进行重大考虑。 委员会必须确保该项目实现你为推动整个组织变革而定义的业务目标，定期开会讨论项目的当前脉搏，并帮助消除在旅程中遇到的任何障碍。

委员会应定义其宪章，以包括一些关键目标：

-   在项目团队与执行发起人或执行领导之间保持紧密一致。

-   向执行发起人或执行领导提供项目状态的见解。

-   允许执行发起人或执行领导团队通过调整项目计划、目标关键结果 (OKR) 和其他项目活动，为项目提供指导和投入，并确保它符合总体业务目标。

指导委员会在整个项目生存期内定期开会，以确保组织领导与项目团队保持一致。 此关键会议可确保项目的方向得到领导层的充分支持，并将领导提供的任何反馈纳入项目以推动成功。 委员会利用这些会议来深入了解项目状态，并执行以下操作：

-   就符合业务案例的业务成果达成一致，并确保项目正在推动这些成果的交付。

-   检查并批准项目，以获取业务案例的准确性和符合性。

-   查看和验证对业务案例所做的更改，这些更改可能会影响任何定义的结果。

-   就项目可交付结果的优先级做出战略决策，并批准临时可交付结果。

-   确定、管理和缓解差距、风险以及委员会需要额外影响的问题。

-   收集执行发起人或执行领导团队对需要升级、确定利益干系人业务单位之间任何冲突的优先级和解决问题的支持。 

-   根据需要，向执行领导、变更顾问委员会或其他业务和 IT 利益干系人提供正式反馈和建议。

## <a name="identify-members-of-the-steering-committee"></a>确定指导委员会成员

一个委员会只有其参与者具有影响力一样有效。 指导委员会需要保持业务和技术决策者以及任何为解决方案做出贡献的核心项目团队领导和合作伙伴团队的代表权。

每个指导委员会都需要强有力的领导，并明确界定角色和责任。 若要为委员会的表现及其成员的参与设定现实的期望，你必须记录并商定分配的角色和责任。 你绝对必须有一个响应式的执行发起人和业务领导在这个委员会，以提供有效的影响，整个组织，并确保一个多样化的视角。

> 根据组织和项目团队设计调整角色和责任。 在较小的组织中，这些角色可能重叠。

## <a name="schedule-steering-committee-meetings"></a>安排指导委员会会议 

尽管每个组织和每个项目团队的结构都略有不同，但指导委员会会议不应只是另一个项目待命会议。 考虑指导委员会会议的目标：你有机会从执行发起人或执行领导处获得支持和指导，使项目保持正常进行。 请确保每个人都参加此会议，以便整个项目团队与执行发起人或执行领导之间有直接消息传递，并且在传递消息时不会出现沟通不畅。

安排委员会会议时，请考虑项目和组织节奏。 根据你的执行发起人偏好和组织的步伐，尝试每周或每隔一周计划一次。

> [!TIP]
> 如果你是一个快速移动的环境，不要让太多的时间在会议之间流过，这可能会导致问题，本来可以避免更频繁的会议。 然而，在节奏较慢的环境中，每周举行一次会议可能会导致会议疲劳，这可能会降低指导委员会的有效性。 使用你的经验和判断来确定适当的节奏，并始终记住，你可以更改计划。 不要仅仅为了会议而开会。

### <a name="prepare-for-the-meeting"></a>为会议做准备

为了充分利用每个人的时间，你需要为会议做准备。 核心团队可以开始为指导委员会会议准备材料。 可以创建 PowerPoint 演示文稿来帮助为会议做准备。

应完成以下资产，并将其包含在会议的演示文稿中：

-   对之前指导委员会操作项的后续操作

-   指导委员会 PowerPoint，项目高级状态报告

    -   从每月/每季度/每年类型评审开始查看项目。

    -   确定关键项目里程碑：

        -   讨论里程碑的重要性和影响 (这些里程碑应在风险注册) 中捕获。

        -   根据准备情况（尤其是业务决策者）来决定即将进行的迁移。

    -   标注项目执行日期和影响很大的活动。

    -   关于确定对委员会很重要的 OKR 的报告;请务必包括使用情况、用户情绪、质量和可靠性指标。

-   指导委员会 PowerPoint，项目每周状态报告

    -   包含突出显示和低光、关键问题和操作项以及后续步骤。

    -   包含更多专注于日常问题的技术详细信息，而较少关注总体项目状态。

-   日志和[风险注册](1-envision-define-my-success-cloud-voice.md#define-okrs-ksis-and-risks)问题

    -   共享委员会正在跟踪的任何问题的状态。

        -   准备好提供有关任何和所有 bug 的更新。

    -   确定可能影响成功实现业务成果的任何新项目。

        -   确定它们对项目的影响。

        -   确定受影响的个人或业务单位。

        -   确定后续步骤以及委员会成员可能需要哪些援助。

-   质量概述：使用 Teams [网络准备](prepare-network.md)指南。

    -   准备好深入解释这些质量报告的结果，并包括修正所述任何问题的建议。

    -   调整每周的质量报告，重点关注以下方面：

        -   组织的总体质量趋势

        -   已知问题网站或遇到问题的站点的状态，以显示站点的方向，并且（理想情况下）报告修正工作的积极结果

        -   基于项目进度最近启用的站点的质量状态

接下来，安排一个电话，为指导委员会会议做准备。 此调用应仅包括核心项目团队中至关重要的团队成员，这可能包括来自网络、桌面、运营或其他团队的资源，这些团队的项目将在委员会会议期间得到解决。 核心项目团队成员不应被你计划在指导委员会通话期间涵盖的任何材料措手不及。

这次筹备电话是指导委员会开会前同步的好方法。 现在也是确保项目团队中任何公开的项目、分歧或不满被揭露的时候了。 这意在帮助促进公开沟通，防止任何人在会议期间感到不快。

与核心团队和合作伙伴利益干系人一起查看该周的报告，并解决任何担忧或问题。 团队内部的工作是根据讨论调整、添加或删除任何内容，或同意要求指导委员会进一步说明。 分配将呈现每个部分的团队成员，以帮助确保流程和连续性。

### <a name="hold-the-meeting"></a>召开会议 

会议本身应该是核心团队完成的准备工作的自然延续。 请记住，这些会议的目标是营造沟通和协作的气氛，以实现该项目的愿景。 使用以下建议来帮助确保会议成功：

1.  将会议安排为面对面。 确定一个会议室，其中有一个足够大的表格，供每个人围坐在一起。 确保会议室具有视频功能。 虽然最好让每个人都在会议室里，但这并不总是可行的，所以请确保会议邀请包括会议室和至少包含音频的在线会议信息，但理想情况下，是音频、视频和共享。 鼓励所有远程参与者在加入会议时使用视频。

2.  努力坚持议程，尽管毫无疑问，需要解决的干扰。 如果可能， (取决于它们) 的重要性，请将这些物品添加到“停车场”部分，并在稍后处理。 如果主题太重要，无法推迟，请同意继续讨论它，但意识到你需要将其他一些议程项目移到未来的会议。 同意留出一些时间 (5到10分钟) 解决任何需要委员会立即关注的紧迫项目。

3.  准备好讨论任何公开的支持案例。 请尽量保持本次会议的高层讨论：门票数量、门票类型、主要开放项目。 你应该准备好解决所有问题，即使它们是次要问题，也要了解对业务的影响和对项目的影响

4.  委托记事人。 当你演示时或在激烈的辩论中做笔记太难了。 确保详细信息记录在委员会中做出的任何决定中。 准备好在后续电子邮件中发送这些决策和备注。

5.  如果有) ，请始终尝试宣传有关人员工作努力和用户反馈 (的正面消息。 讲述和销售这个故事。 如果反馈完全是负面的，请准备好讨论它，并说明正在采取哪些措施来解决负面用户体验问题。

    a.  借此机会向各个团队成员授予荣誉，让他们完成他们所做的出色工作。 重要的是要认识到团队成员在完成出色、艰苦的工作时，尤其是在领导团队面前。

6.  请务必保存会议结束，以就任何后续步骤达成一致，并确定在会议期间商定的这些操作项目的所有者。

7.  向与会者发送包含你创建的演示文稿的后续电子邮件，并追加所做任何笔记。 请务必包括任何未出席的人员。

## <a name="next-steps"></a>后续步骤

在审查了指导委员会的概述后，现在是采取行动的时候了。项目团队应开始工作，将指导委员会落实到项目的日程表中，并准备定期指导委员会会议节奏。 理想情况下，委员会将在项目开始之前成立，并在所有阶段提供指导，但如果情况并非如此，请组成委员会，指导项目团队取得成功。

1.  确定指导委员会成员。 问问自己：

    -   我们是否拥有来自企业和领导层的适当代表？

    -   我们确定的成员是否能够影响其他单位的业务单位或对等方？

2.  创建要发送给指导委员会每个成员的通信：

    -   包括角色表。

    -   出售委员会的重要性。

3.  设置第一次会议，讨论角色和职责，定义委员会的章程，并验证或定义 OKR。 议程项目也可以包括：

    -   讨论会议节奏。

    -   验证特许和 OKR 是否正在推动有意义的业务成果。

    -   验证策略是否与 OKR 保持一致。

4.  在每次对话中开始提高驾驶质量：

    -   阐明了拥有高质量冠军的重要性。

    -   确定将担任质量冠军角色的人员或团队。

指导委员会的成功不是运气造成的。 选择具有最佳意图的适当成员仍会产生不良结果。 若要帮助推动指导委员会的成功，需要确保你拥有以下各项：

-   一个商定的愿景和范围的项目，具有明确定义，可衡量的业务成果，委员会将提倡。

-   一个组织良好的议程，推动有目的的会议。

-   开放沟通的气氛。

    -   开放沟通和协作将赋予委员会权力。 创建一个环境，在其中听到所有的声音，并尊重意见。 唯一的视角可能产生最大的影响。

    -   当领导人员采取行动促进成员协作和沟通时，可以启用开放环境。 下面是一些准则：

        -   所有委员会成员都应该是向组织介绍的变革的有力倡导者。

        -   将委员会的规模限制在那些绝对需要提供准确情况的人以及能够跨业务部门做出或影响决策的人。 当出现阻止程序时，了解组织的层次结构和每个成员的直接影响线至关重要。

        -   让项目经理和团队领导参与委员会的讨论和审议。 准备好邀请其他核心团队成员来澄清特定领域可能需要的深入讨论。

在解决方案的生命周期内，委员会的作用将发生变化。 随着设想和载入进入驱动器价值阶段，委员会可能需要进行一些调整：

-   调整会议的节奏。 查看组织的动态，并确定会议是否需要更低的频率进行。 随着组织变革开始减少，高层通信的频率可能会降低。 调查如何根据需要将会议更改为每月或季度会议节奏

-   调整会议中的代表。 当解决方案进入驱动器值阶段时，核心项目团队可能会被重新分配到新项目。 需要更新作为支持报告的团队，操作将成为解决方案状态的线索。

-   针对操作 OKR 的定义和报告。 关键结果和成功指标将在解决方案的生命周期内发生更改。 应继续跟踪和报告所需的业务成果。 在推动业务成果发展的背景下，继续报告语音解决方案的质量和可靠性。 其他指标和关键结果（例如，用户情绪、用户采用和支持指标）将成为成功的首要指标。

-   使用更大的指导委员会保护伞来涵盖多个解决方案的驱动器价值阶段。 寻找更高效地使用指导委员会的机会。 使用委员会跨更大的解决方案领域提供指导。
