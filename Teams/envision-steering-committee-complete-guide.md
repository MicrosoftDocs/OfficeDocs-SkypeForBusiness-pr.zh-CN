# <a name="steering-committee"></a>指导委员会

## <a name="steering-committee-overview"></a>指导委员会概述

筹划指导委员会是主要利益相关者以及项目主管人员已投入管理组在一起，指导项目群向其定义的业务成果。 指导委员会并不直接负责*如何*交付项目的但而*哪些*项目实现的业务。

每个项目需要商定的构想和章程。 为实现项目的预期的成效，必须清楚地定义的设想，它需要监视和维护。 这一点的指导委员会的责任： 驱动器的决定、 建议、 战略监督作为拥护者对项目的计划，组织和 — — 在必要的时候 — — 删除广告拦截器。

组织应对大量缜密筹划指导委员会成立。 该委员会必须确保项目达到业务目标，对于推动整个企业，满足定期来讨论该项目，当前脉冲的更改已定义并帮助删除过程中遇到的任何障碍之旅。

该委员会应定义其宗旨包括一些关键的目标：

-   让项目团队和上级主管或管理层领导力之间的强对齐。

-   对上级主管或管理层领导力提供洞察项目的状态。

-   允许的上级主管或行政领导团队，提供指导并输入到该项目并确保它对齐与总体业务目标，通过调整项目计划、 目标关键结果 (OKRs) 和其他项目活动。

以周期性间隔的整个生存期内的项目，以确保组织的领导层和项目团队之间的对齐方式可以满足筹划指导委员会。 此关键会议确保项目的方向具有领导地位的全面支持和集成了领导到驱动器成功项目所提供的任何反馈。 该委员会将使用这些会议来深入了解项目状态，以及：

-   在对齐到业务用例，并确保项目将朝这些成果的交付的业务成果达成一致。

-   检查和审核的准确性和法规遵从性的业务案例的项目。

-   复查并验证到业务用例可能会影响任何已定义的结果所做的更改。

-   使项目可交付结果的优先顺序有关的战略决策和批准中期可交付结果。

-   识别、 管理和缓解的间隙、 风险和其他影响所在需要从该委员会的问题。

-   从上级主管或行政领导团队的问题，需要升级，优先级并解决任何冲突的利益相关者业务部门之间争取支持。 

-   提供正式的反馈和建议管理层领导力、 更改顾问委员会，或其他业务和 IT 部门，（如果适用）。

## <a name="identify-members-of-the-steering-committee"></a>标识指导委员会的成员

一个委员会才那样有效，因为它的参与者都有影响力。 筹划委员会需要维护从业务和技术决策制定者，以及任何核心项目团队领导和合作伙伴团队参与到解决方案中的表示形式。

每个指导委员会必须拥有优秀的领导，并明确定义的角色和职责。 若要设置现实的预期值，为该委员会的性能和其成员的共同参与，必须记录并同意分配的角色和职责。 绝对必须响应上级主管和业务领导此委员会，以跨整个企业提供有效的影响，并确保各种不同的角度来看。

> 调整角色和责任，根据您的组织和项目团队设计。 在小组织中，这些角色可以重叠。

## <a name="schedule-steering-committee-meetings"></a>计划指导委员会会议 

虽然每个组织和每个项目团队的结构方式稍有不同，但指导委员会会议决不只是另一个项目站立会议。 请考虑指导委员会会议的目标： 它是您使项目保持在轨道上，通过收集支持和指导的上级主管或管理层领导力的机会。 确保所有人出席该会议，因此没有直接消息的整个项目团队和上级主管或管理层领导力之间并没有任何错误传达时邮件被中继。

如果排定的会议的一个委员会，考虑到项目和组织的节奏。 试着每周或每隔一周，取决于执行发起人的首选项并在您的组织的发展计划。

> [!TIP]
> 如果您是一个变化无常的环境，不要让会议时间间隔太长时间 — — 这可能会导致无法通过更频繁的会议避免的问题。 但是，在较慢速度的环境中，主持每周会议会导致会议疲劳，可能会降低指导委员会的有效性。 使用您的经验和判断来确定适当的节奏，并始终牢记您可以更改计划。 不只是为了会议能满足。

### <a name="prepare-for-the-meeting"></a>准备会议

要充分利用每个人的时间，您需要为会议做准备。 核心团队可以开始准备筹划指导委员会会议的资料。 [指导委员会模板 – 链接到示例的 PPT 模板] 指导委员会模板可以用作准备的起始点。

应完成和会议演示文稿幻灯片中包含以下资源：

-   在前期筹划指导委员会交办事项的后续工作

-   指导委员会 PowerPoint，项目高级状态报告

    -   从每月/季度/年度类型查看所有最多看项目。

    -   确定关键的项目里程碑：

        -   讨论的重要性和影响 （这些应该风险登记簿中捕获） 的里程碑。

        -   在进行/不进行准备工作，特别是与业务决策者根据即将迁移的决定。

    -   指明项目执行日期和影响最大的活动。

    -   报告在 OKRs 被视为重要的委员会;请务必包括使用情况、 用户主观意图、 质量和可靠性指标。

-   指导委员会 PowerPoint，项目每周状态报告

    -   包含突出显示和缺点，关键的问题，和交办事项和下一步行动。

    -   包含专注于日常的问题，而减少总体项目状态的更多技术详细信息。

-   问题日志和风险注册 [链接风险问题日志 （Web 内容）]

    -   共享状态的跟踪该委员会的任何问题。

        -   做好准备，以便在所有 bug 提供更新。

    -   标识实现业务成果的成功可能会影响任何新项目。

        -   确定它们对项目的影响。

        -   标识受影响的个人或业务部门。

        -   确定下一步，何种协助可能需要从委员会的成员。

-   质量概述： 使用报表从质量冠军 [链接到规划服务管理和质量 （Web 内容）]

    -   做好准备，以便解释结果的深入，这些质量报告，包括建议修正说明任何问题。

    -   调整质量报告每周集中在以下几个方面：

        -   该组织的整体质量趋势

        -   已知的问题或遇到的问题，以显示该站点的方向的网站的状态的发展方向和 — — 理想情况下 — — 报告正从补救工作的结果

        -   根据项目进度的最近启用站点的质量状态

下一步，计划邀请为指导委员会会议做准备。 此调用应只包括至关重要的核心项目团队的团队成员，这可能包括来自网络、 桌面、 操作或其他小组已将解决在委员会会议期间的项目资源。 核心项目团队成员不应捕获关闭保护计划指导委员会调用期间涵盖任何材料。

此预备的调用是同步筹划指导委员会达到之前的好方法。 还有时间来确保任何打开的项目、 意见分歧，否则或项目组之间的 grievances 进入光。 这是为了帮助促进开放式通信，并防止任何人从 unpleasantly 惊讶在会议期间。

检查该周的报告与核心团队和合作伙伴利益干系人和解决任何问题或疑问。 在调整、 添加或删除任何内容基础讨论时，小组内工作或同意要求其他澄清的筹划指导委员会。 指定将显示每个部分，以帮助确保流和连续性的团队成员。

### <a name="hold-the-meeting"></a>召开会议 

会议本身应该是自然的延续，核心团队已经完成了准备工作。 请记住这些会议的目的是要促进交流与合作，以实现项目的远景的氛围。 使用以下建议可帮助确保会议的成功：

1.  计划为面对面的会议。 标识的表足够大，以便让每个人都坐在周围的会议室。 确保会议室内有视频功能。 虽然会比较理想，要让每个人都在房间里，它并不总是可能的因此请确保会议邀请包括会议室和联机会议信息至少音频 — — 但是，理想情况下，音频、 视频和共享。 鼓励所有远程参与者使用视频，当他们加入会议。

2.  尽量保持的议程，尽管毫无疑问会出现干扰，需要解决。 如果可能的话 （根据其重要性），将这些项目添加到"停车场"部分和以后解决它们。 如果主题是非常重要的推迟，同意继续讨论，但是认识到需要将某些其他议程项目移到将来的会议。 留出一些时间 （五至十分钟） 来解决需要委员会立即注意的任何按项同意。

3.  准备讨论任何打开的支持的情况。 尽量使讨论作为该会议的可能为高层次 — — 票证数目、 票证，主要打开的项目的类型。 应准备好解决所有问题，即使它们是次要、 了解到的业务和项目的效果的影响

4.  委托的记录员。 它是太难做笔记，当您在演示时或在加热的辩论。 确保详细信息都记录在该委员会中所做的任何决定的。 做好后续跟踪电子邮件的一部分发送这些决策和注释掉。

5.  总是尝试提升有关人们的工作精力和用户反馈的正面新闻 （如果可用）。 判断和销售情景。 如果完全否定的反馈，一定要进行讨论，并说明正采取什么措施来解决负用户体验问题。

    a.  借此机会感谢给各个团队成员对他们所做的良好工作。 识别时的团队成员，特别是在其领导团队完成出色的、 硬的工作至关重要。

6.  总是保存会议达成任何下一步行动，并确定在会议期间达成的那些操作项的所有者的结束。

7.  将与会者发送后续跟踪电子邮件，其中包含演示文稿创建，并将追加拍摄的任何注释。 请务必包括未参与任何人。

## <a name="next-steps"></a>后续步骤

复查的筹划指导委员会概述，现在就采取行动。项目团队应开始他们的工作到项目的时间线实现筹划指导委员会和准备常规指导委员会会议节奏。 理想情况下，该委员会将形成之前项目开始和提供指导的各个阶段，但如果它不能出现这种情况，窗体的委员会，来指导项目团队成功的结果。

1.  标识指导委员会成员。 自问：

    -   我们有从商务和领导能力的适当表示吗？

    -   是我们发现的成员能够影响他们的业务部门或其他单位中的对等？

2.  创建一种通讯，发送到每个成员的筹划指导委员会：

    -   包括角色表。

    -   宣传委员会的重要性。

3.  建立了第一个会议，以讨论角色和职责、 定义的该委员会，宗旨和验证或定义 OKRs。 议程项目还包括：

    -   讨论会议节奏。

    -   验证的章程和 OKRs 将促使朝着有意义的业务结果。

    -   验证的策略是与 OKRs 保持协调。

4.  开始每个对话中促进质量：

    -   清楚地表述具有质量冠军的重要性。

    -   识别的人或团队，他们将采用质量拥护者角色。

指导委员会成功会发生由好运。 选择适当的成员，提供最佳的意图，可以仍然产生较差的结果。 为了推动成功的筹划指导委员会，您需要确保您具有下列：

-   商定的构想并具有定义完善的、 可衡量的业务成果，该委员会将提倡了项目的范围。

-   驱动器有目的的会议组织良好议程。

-   开放式交流的氛围。

    -   开放的交流与合作将使该委员会。 创建一个环境，其中所有的声音都能听到，会考虑的意见。 独特的观点会有大的影响。

    -   领导作用以促进成员协作和通信时，可以启用一个开放的环境。 下面是一些准则：

        -   委员会的所有成员都应强提倡组织所引入的更改。

        -   保留那些绝对需要提供准确和那些可以进行或影响跨业务部门的决策委员会大小有限制。 当拦截器出现，了解组织的层次结构和影响的每个成员的直拨电话将至关重要。

        -   委员会讨论和商议中参与的项目经理和团队领导者。 请准备邀请其他核心团队成员在特定区域可能需要进行的深入讨论将清晰度。

解决方案的生命周期将更改该委员会的作用。 构想和服务移动到驱动器值阶段若干调整可能需要向该委员会：

-   调整节奏的会议。 查看组织的动态并确定会议是否需要较少发生这种情况。 变革开始 dwindle 上级通信的频率可能将会降低。 调查会议改在适当情况下每月或每季度的会议节奏

-   调整在会议中的代表。 因为该解决方案将移动到的驱动器值阶段可能是核心项目团队将被重新分配到新项目。 它将需要更新报告为支持团队和操作将成为解决方案的状态上的潜在顾客。

-   定义和针对运营的 OKRs 报告。 主要结果和成功指标将更改解决方案的生命周期。 所期望的业务成果应继续跟踪和报告。 继续报告的质量和可靠性朝业务结果的上下文中的语音解决方案。 其他指标和密钥产生，如用户主观意图、 获得用户的认可和支持的指标将成为成功的顶部的指示灯。

-   使用较大的指导委员会，伞盖多个解决方案的驱动器值阶段。 寻找各种机会来更有效地使用筹划指导委员会。 该委员会用于在解决方案的一个大球体提供指导。
