# <a name="steering-committee"></a>指导委员会

## <a name="steering-committee-overview"></a>指导委员会概述

筹划指导委员会是一组关键利益干系人和项目负责人组，他们已将项目或项目群与其定义的业务成果结合在一起。 筹划指导委员会不直接负责项目的交付*方式*，而是项目交付给企业的*内容*。

每个项目都需要商定的愿景和章程。 若要提供项目的预期结果，远景必须明确定义，它需要监视和维护。 这将成为筹划指导委员会的责任：要推动决策，建议提供战略监管，以充当项目计划的组织的倡议，并在必要时删除阻止程序。

您的组织应将重要想法放入筹划指导委员会。 该委员会必须确保项目能够实现您为在整个组织中所做的更改而定义的业务目标，定期满足项目的当前脉冲，并帮助删除在旅行期间遇到的任何障碍。

委员会应定义其章程以包括一些关键目标：

-   在项目团队和总经理主办方或执行官领导之间保持紧密协调。

-   深入了解项目的状态与总经理发起人或管理层领导。

-   允许总经理主办方或执行官领导团队向项目提供指导和输入，并确保它与上级业务目标相协调，通过调整项目计划、客观的结果（OKRs）和其他项目活动。

在整个项目生命周期内，筹划指导委员会在定期间隔内达到一定的，以确保组织领导和项目团队之间的协调。 这项关键会议确保项目的发展方向具有领导层的完全支持，并将领导层提供的任何反馈纳入到项目中以推动成功。 该委员会使用这些会议深入了解项目状态，还可以：

-   同意与业务案例相符的业务成果，并确保项目能够推动这些成果的交付。

-   检查和批准项目以确保其准确性并遵守业务案例。

-   查看和验证对业务案例所做的更改，这些更改可能会影响任何定义的结果。

-   制定有关项目可交付结果的优先级决策，并批准临时可交付结果。

-   确定、管理和缓解需要从委员会进行额外影响的差距、风险和问题。

-   从总经理发起人或管理层领导团队处收集支持，以了解需要升级的问题、排定利益干系人的业务部门的冲突并解决这些冲突。 

-   根据适用情况，为管理层领导、变更咨询委员会或其他业务和 IT 利益干系人提供正式反馈和建议。

## <a name="identify-members-of-the-steering-committee"></a>确定筹划指导委员会的成员

一个委员会的工作方式仅与其参与者的影响更有效。 筹划指导委员会需要维护来自业务和技术决策者的表示形式，以及参与解决方案的核心项目团队领导和合作伙伴团队。

每个筹划指导委员会都需要强大的领导能力和明确定义的角色和职责。 若要为该委员会的业绩及其成员的参与设置切合实际的期望值，您必须记录并同意分配的角色和职责。 您绝对必须在此委员会上有响应性的行政主管和业务领导，以便为整个组织提供有效的影响，并确保多样化的视角。

> [!NOTE]
> 根据组织和项目团队设计调整角色和责任。 在较小的组织中，这些角色可能会重叠。

| 标题 | 角色 | 联系人 | 电子邮件 |
|----------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|---------|-----------|
| 执行发起人 | 对项目的提倡，确保其方向与组织的愿景保持一致。 作为获得本委员会不同意的决策的最终授权机构。 | | |
| 业务决策人 | 对项目的提倡，确保其方向与组织的愿景保持一致 | | |
| 体系 | 确保技术决策能够实现，并与商定的业务成果协调一致。 | | |
| 核心项目团队 | 提供有关任何特定组件（技术或非技术性）的详细信息（如采纳和更改管理） | | |
| 质量拥护者 | 负责整理质量指标。 | | |
| 计划/项目经理 | 对要合并到项目中的反馈进行负责演示和收集。 | | |
| 合作伙伴客户团队 | 通过更广泛的视角和其他合作伙伴活动，提供来自合作伙伴的观点的反馈。 | | |

## <a name="schedule-steering-committee-meetings"></a>安排筹划指导委员会会议 

虽然每个组织和每个项目团队的结构略有不同，但筹划指导委员会决不应该只会成为另一个 project standup 会议。 考虑筹划指导委员会会议的目标：这是您可以通过从行政主管或管理层领导处收集支持和方向来保持项目顺利进行跟踪的机会。 确保每个人都参与此会议，以便在整个项目团队和执行发起人或执行官之间建立直接消息传递，并且在邮件中继时没有 miscommunication。

安排一种委员会会议时，请考虑项目和组织节奏。 请根据您的 "执行赞助商" 首选项和组织的步调，尝试安排每周或每隔一周。

> [!TIP]
> 如果你是一种快速移动的环境，则不要在会议之间花费过多的时间间隔，这可能会导致更频繁的会议可能会避免的问题。 但是，在进度较慢的环境中，每周召开会议可能会导致会议 fatigue，这可能会降低指导委员会的有效性。 使用你的体验和 judgement 确定合适的节奏，并随时记住你可以更改计划。 不要为会议而开会。

### <a name="prepare-for-the-meeting"></a>准备会议

为了最大程度地利用所有人的时间，您需要为会议做准备。 核心团队可以开始为指导委员会会议准备材料。 [筹划委员会模板-链接到示例 PPT 模板] 指导委员会模板可用作准备工作的起点。

应完成以下资源，并将其包含在会议的演示文稿幻灯片中：

-   跟进以前的筹划指导委员会操作项目

-   筹划指导委员会 PowerPoint，项目高级别状态报告

    -   从每月/每年/每年一次的评论查看项目。

    -   确定关键项目里程碑：

        -   讨论里程碑的重要性和影响（应在风险登记中捕获它们）。

        -   根据准备情况确定即将进行的迁移的 "转到"/"否"，特别是对于业务决策者。

    -   调用项目执行日期和高影响活动。

    -   报告对委员会很重要的 OKRs;请确保包括使用情况、用户情绪、质量和可靠性指标。

-   筹划指导委员会 PowerPoint，project 每周状态报告

    -   包含突出显示和 lowlights、关键问题、操作项以及后续步骤。

    -   包含更多的技术详细信息，重点关注日常问题，并降低总体项目状态。

-   问题日志和[风险注册](1-envision-define-my-success-cloud-voice.md#define-okrs-ksis-and-risks)

    -   共享该委员会跟踪的任何问题的状态。

        -   准备好提供任何 bug 和所有 bug 的更新。

    -   确定可能影响实现业务成果成功的任何新项目。

        -   确定其对项目的影响。

        -   标识受影响的个人或企业单位。

        -   确定后续步骤以及委员会成员可能需要的帮助。

-   质量概述：使用指南[为团队准备网络](prepare-network.md)。

    -   准备深入介绍这些质量报告的结果，并提供建议以补救所记录的任何问题。

    -   每周调整质量报告以关注以下方面：

        -   组织的整体质量趋势

        -   已知问题站点或出现问题的站点的状态，用于显示网站的标题和方向（理想情况下）从补救工作中报告正面结果

        -   基于项目进度的最近启用的网站的质量状态

下一步，安排通话以准备进行筹划指导委员会会议。 此通话应仅包括对核心项目团队至关重要的团队成员，这可能包括来自网络、桌面、操作或其他团队的资源，这些资源包含在委员会会议期间将寻址的项目。 核心项目团队成员永远不能通过您计划在筹划指导委员会通话时所涉及的任何材料来预防。

此准备通话是在筹划指导委员会之前同步的一种好方法。 还可以确保项目团队中任何打开的项目、disagreements 或以的时间不太轻。 这有助于促进开放通信，防止任何人在会议期间 unpleasantly 惊讶。

查看该周的报表与核心团队和合作伙伴利益干系人的报表，并解决任何疑问或问题。 在团队中工作，根据你的讨论调整、添加或删除任何内容，或者同意从筹划指导委员会获取进一步的澄清。 分配将显示每个部分的团队成员，以帮助确保流和连续性。

### <a name="hold-the-meeting"></a>召开会议 

会议本身应是核心团队已完成的准备工作的自然延续。 请记住，这些会议的目标是促进沟通和协作的气氛，以实现项目的愿景。 使用以下建议帮助确保会议成功：

1.  安排面对面会议。 确定一个会议室，其中的表格足够大，可供每个人坐在身边。 确保会议室具有视频功能。 虽然你可以在聊天室中让每个人都是理想的，但并不总是这样，因此请确保会议邀请包括会议室和至少带有音频的联机会议信息，但最好是音频、视频和共享。 鼓励所有远程参与者在加入会议时使用视频。

2.  请尝试继续安排日程，尽管没有疑问干扰也会出现问题。 如果可能（取决于其重要性），请将这些项目添加到 "停车场" 部分，然后再将其解决。 如果该主题对延期太重要，请获取协议以继续讨论它，但请注意，你需要将一些其他议程项目移动到未来的会议中。 同意保留一段时间（5至10分钟），以解决需要立即关注的任何按下的项目。

3.  准备好讨论任何打开的支持案例。 尝试将讨论保留为此会议的高级别：票证数量、票证类型、主要打开项目。 你应该准备解决所有问题，即使它们很少，了解对业务的影响和对项目的影响

4.  委派笔记者。 在演示过程中或在更感兴趣的辩论期间做笔记非常困难。 请确保为在委员会中做出的任何决策记录详细信息。 准备好将这些决策和笔记作为后续电子邮件的一部分发送。

5.  始终尝试宣传有关人员的工作成果和用户反馈（如果可用）的积极消息。 告诉和销售故事。 如果反馈完全否定，请准备讨论它，并解释解决消极用户体验问题的操作。

    a.  有机会向单个团队成员提供 kudos 的工作。 了解团队成员是否正在完成出色的工作，尤其是在团队成员的领导团队中，尤其是认识到了重要工作。

6.  始终保存会议的结束时间，以同意任何后续步骤，并标识会议期间同意的待办事项的所有者。

7.  向与会者发送包含你创建的演示文稿的后续电子邮件，并附加所执行的任何笔记。 请确保包括任何未出席的人。

## <a name="next-steps"></a>后续步骤

查看指导委员会的概述后，现在就可以行动了。项目团队应开始执行工作，以将筹划指导委员会实施到项目的日程表，并准备定期的筹划指导委员会会议节奏。 理想情况下，将在项目开始之前形成委员会，并在其所有阶段提供指导，但如果不是这样，请将该委员会指引到项目团队的成功结果。

1.  确定指导委员会成员。 询问：

    -   我们是否有来自企业和领导层的正确表现？

    -   我们识别的成员是否能够在其他单元中影响其业务部门或对等？

2.  创建要发送给筹划指导委员会的每个成员的通信：

    -   包含 roles 表。

    -   销售委员会的重要性。

3.  设置第一个会议以讨论角色和职责、定义委员会的宗旨，并验证或定义 OKRs。 议程项目还可以包括：

    -   讨论会议节奏。

    -   验证章程和 OKRs 是否正在推动有意义的业务成果。

    -   验证策略是否与 OKRs 对齐。

4.  在每个对话中开始推动质量：

    -   清楚地表述有质量的拥护者的重要性。

    -   标识将承担质量拥护者角色的人员或团队。

幸运的是，指导委员会不会再成功。 选择合适的成员，获得最佳效果仍会产生不良结果。 为帮助推动筹划指导委员会的成功，您需要确保有以下几项：

-   项目的商定愿景和范围，该项目具有明确定义的、可衡量的业务成果，该委员会将 advocating。

-   一个整理针对性会议的条理完备的日程安排。

-   开放式通信的气氛。

    -   开放式通信和协作将使委员会更有能力。 创建一个可听到所有语音并遵守意见的环境。 独特的视角可能会产生最大的影响。

    -   当领导层促进成员协作和通信时，可以启用开放环境。 下面是一些指南：

        -   所有委员会成员都应严格负责向组织推出的变更。

        -   将委员会的大小限制为绝对需要的人员，以便提供准确的图片，以及可以在业务单元中做出或影响决策的人员。 了解组织的层次结构和每个成员的直接影响行将对阻止者显示为关键。

        -   让项目经理和团队领导参与委员会的讨论和 deliberations。 准备邀请其他核心团队成员，让特定区域中可能需要的深入讨论更清楚。

通过解决方案的生命周期，委员会的角色将会发生变化。 随着构想和加入到驱动器值阶段，委员会可能需要进行几项调整：

-   调整会议节奏。 查看组织的动态，确定会议是否需要更少的发生。 随着组织的更改开始 dwindle，可能会降低高级通信的频率。 调查将会议更改为每月或每季度会议节奏（如果适用）

-   调整会议中的代表。 当解决方案移动到 "驱动器价值" 阶段时，可能会将核心项目团队重新分配给新项目。 需要更新报告为支持的团队，并且操作将成为解决方案状态的潜在顾客。

-   针对操作 OKRs 的定义和报告。 关键结果和成功指示器将在解决方案的生命周期内更改。 应继续跟踪和报告所需的业务成果。 继续在推动业务成果的环境中报告语音解决方案的质量和可靠性。 其他指标和关键结果（如用户情绪、用户采纳和支持指标）将成为成功的最佳指标。

-   使用较大的指导委员会伞来涵盖多个解决方案的 "驱动器价值" 阶段。 寻找更高效地使用筹划指导委员会的机会。 使用委员会通过更大的解决方案球体提供指导。
