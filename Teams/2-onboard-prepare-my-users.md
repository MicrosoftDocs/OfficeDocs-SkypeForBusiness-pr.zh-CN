# <a name="prepare-my-users"></a>准备用户

本文概述了让用户准备好使用云语音服务的要求。 通过正确准备用户，你可以确保为组织提供最大的价值。

<!--ENDOFSECTION-->

## <a name="preparing-for-user-adoption"></a>为用户采用做准备

保护项目团队并定义项目的范围和目标后，下一步是确保组织和用户就绪性，这是一项与技术准备情况并行完成的活动。 若要从部署中实现价值，用户必须实际使用和采用它。 仅启用解决方案并不能保证实现目标。 此外，用户具有不同的用例和不同的学习风格，并且以不同的速度适应新技术。 好消息是，管理更改并不复杂，但确实需要集中努力。 下面包含的引导式讨论旨在帮助你了解用户群并准备适当的教育程度，以便促进和加速用户采用。

活动所有者：采用变更经理作为潜在顾客、Project主管、执行发起人、营销/培训/支持经理

> [!NOTE] 
> 下面提供的评估活动可适应组织的任何更改计划。 只需根据项目范围回答问题。 在这些活动的讨论中，"新解决方案"可应用于音频呼叫、音频会议或从 Skype for Business 升级到 Teams。

<!--ENDOFSECTION-->

## <a name="create-your-vision-to-value-statement"></a>创建愿景到价值声明

通过设置项目愿景，关键利益干系人可以看到全局。 这通常等同于成本节省、运营效率和工作效率提升等业务驱动因素，并与之前在项目中 &mdash; &mdash; 设定的目标保持一致。 对于用户，可能会认为同一项目会破坏熟悉的工具、生产力中断，以及管理层缺乏灵活性。 请花些时间将自己放在用户的脚上，将项目愿景转换为吸引他们的价值声明。 在通信和训练活动中使用此值声明，使用户能够查看Teams可如何工作。

### <a name="what-percent-of-users-fall-into-each-bucket"></a>哪些用户属于每个存储桶？

## <a name="assess-your-user-base"></a>评估用户群

通过人员级分析，可了解用户角色、队列和用例。 通过花时间了解用户是谁及其工作机制，你可以确保在实施更改时他们拥有成功所需的内容。 这可能包括定向消息传送、训练，甚至包括耳机和网络摄像机等设备。 例如，高层可能期望一对一培训，而出差现场代表可能会受益于他们空闲时可以观看的培训视频。 此外，人员级分析可以帮助确定谁现在应该开始使用新功能，或者谁在等待时可能获得最佳服务。

<tr><td><img src="media/audio_conferencing_image7.png" alt="An icon depicting decision points"/> <br/>决策点</td><td><ul><li>某些用户或队列是否依赖于此解决方案才能有效地执行其作业？</li><li>是否有可以通过使用此新解决方案促进的特定用例？</li></ul></td></tr>
<tr><td><img src="media/audio_conferencing_image9.png" alt="An icon depicting the next steps"/><br/>后续步骤</td><td><ul><li>映射用户角色。 首先，将用户群细分为人员/队列组 (主管、道路运输人员、协作工作者、零售等) 。 为易于管理，请列出不超过八个组。  (有关示例对话入门，请参阅下文。) 使用你的发现来进一步设计用户准备活动，并细分试点用户与可能受益于以后升级的用户。
</li><li>映射用例。 确定将受益于此新技术的核心用户组后，请考虑一般用户群体。 通过了解跨团队用例和方案，可以将使用此新技术的组织可视化。 此外，向通信和培训计划添加用例示例确实有助于使其与用户相关。</li></ul></td></tr>
</table>

对于每个人员组，与项目团队讨论以下内容：

-   此组的核心特征是什么 (，也就是说，典型的"生活一天"是什么) ？

-   此组如何受益于新解决方案？ 它们需要哪些特性和功能有效？

-   他们需要哪些设备 (例如耳机或扬声器) ，并且它们是否已经拥有？

-   若要在此组中创建开始使用新解决方案所需的其他价值消息传送（如果有）？

-   此组接收消息的最佳方法是什么：

    -   认知 (例如电子邮件、海报、部门经理) ？

    -   培训 (例如在线培训、用户指南、白) ？

    -   支持 (自助服务、支持人员、支持) ？  

-   此组是试点或早期采用者计划的良好候选者吗？ 如果是这样，请制定一个计划，让这些用户进行购买以参与试点。 有关实施成功的试点计划的信息，请参阅与 Teams[一起Skype for Business。](pilot-essentials.md)

![列出人员详细信息的值表](media/prepare-my-users-image2.png "列出角色及其常见角色以及通信和协作解决方案依赖项的表。突出显示了高层形象，并详细介绍了 (一天的个人资料) 功能、设备和培训高层需求。")

对于每个用例，请与您的项目团队讨论以下内容：

-   此用例/方案的核心特征是什么？

-   当前解决方案有什么好？

-   当前解决方案面临哪些挑战（如果有）？

-   新解决方案如何优化用户体验？

## <a name="optimize-your-readiness-plan"></a>优化准备计划
了解用户是谁及其如何在新解决方案中工作后，可以设计准备计划。 用户准备计划包含在试点、载入和操作新解决方案时如何与用户通信、培训和支持。 如本文前面的步骤所发现，就绪性并非一成多。 从广泛宣传的认知和培训计划开始，将此过程开始时定义的价值消息传递纳入其中。 然后，根据你定义的角色或队列来定制消息、培训和支持。 你的个性化计划将使用户能够更快速地与新解决方案关联，同时表明你了解他们的需求，这有助于促进其过渡。
