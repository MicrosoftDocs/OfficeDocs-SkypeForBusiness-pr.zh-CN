# <a name="prepare-my-users"></a>准备我的用户

这篇文章概括介绍了准备您的用户使用云语音服务的要求。 通过正确地准备您的用户，将确保您为您的组织提供最大的价值。

<!--ENDOFSECTION-->

## <a name="preparing-for-user-adoption"></a>准备用户采用率

您保护您的项目组和定义的范围和目标的项目之后下, 一步确保组织和用户准备、 活动完成与您的技术准备工作。 要认识到您的部署的值，用户必须实际使用和采用它。 只启用一种解决方案并不能保证您会实现目标。 此外，用户具有不同的使用情形和不同的学习方式，和他们适应新的技术，以不同的速度。 好消息是，管理变革并不那么复杂，但会占用多少精力。 包括下面的指导的讨论旨在帮助您了解您的用户群和准备适当级别的教育，以促进和加快获得用户的认可。

活动负责人： 采用更改管理员作为主管，项目主管，上级主管，市场营销/培训/支持经理

> [!NOTE] 
> 下面提供的评估活动是适应您的组织中的任何更改计划。 只是地址根据项目范围的问题。 在讨论这些活动的过程中，"新解决方案"可以从业务团队的 Skype 应用于调用音频、 音频会议或升级。

<!--ENDOFSECTION-->

### <a name="what-percent-of-users-fall-into-each-bucket"></a>满足用户的百分比分为每桶？

| **用户的类型**                                                             | **%** |
|------------------------------------------------------------------------------|-------|
| *早期采用者*的请求才可用的解决方案。                  |       |
| *告知用户*接受的解决方案，只要其值进行了演示。    |       |
| *滞后*拒绝解决方案，即使推入更改。                 |       |


> [!TIP]
> 早期采用者是试验性部署，还拥护者的计划非常适用。 如果您有大量的滞后，花时间专注于价值，以帮助它们在主板上进行此更改的用户和 handholding 消息。

### <a name="what-percent-of-users-fall-into-each-competency"></a>满足用户的百分比可以分成每个能力？

| **学员的类型**                                                        | **%** |
|----------------------------------------------------------------------------|-------|
| *自我激励*学员寻找资源，通过这样做来了解。                         |       |
| *团队队员*享受组和交互式培训;他们将与同事一起转。 |       |
| *Handholding* seekers 认为"白手套"或一对一的帮助。                   |       |

 
> [!TIP]
> 学习不是一成不变的。 通过了解您的用户如何最好地了解，可以创建一个可靠培训项目来帮助加快他们的学习。 如果许多用户会自我激励，链接到视频和用户参考线可能足够了。 对于那些需要特殊的关注，请务必安排一对一&mdash;或登记您的拥护者，来帮助这些用户在学习曲线上。

## <a name="assess-your-user-base"></a>评估您的用户群

人级别的分析使您了解您的用户角色、 群体，以及用例。 通过花时间了解了您的用户是谁以及它们如何工作，可以确保他们有需要会成功，因为实现更改。 这可以包括目标消息、 培训或甚至耳机和网络摄像机等设备。 例如，主管可能希望一对一的培训，而旅行现场销售代表可能受益于培训视频可以在他们方便的时候观看。 此外，人员级别分析有助于确定谁应开始使用新的功能，如今与那些可能最好由等待。

<table>
<tr><td>![](media/audio_conferencing_image7.png) <br/>决策点</td><td><ul><li>某些用户或群体是否核心依赖此解决方案有效地完成其作业？</li><li>是否有特定的用例，可以通过使用这个新的解决方案方便了？</li></ul></td></tr>
<tr><td>![](media/audio_conferencing_image9.png)<br/>后续步骤</td><td><ul><li>映射用户角色。 首先，将您的用户群划分角色/群体组 （高级管理人员、 旅途战士、 协作工作人员、 零售，等等）。 易管理性，列出不多于 8 个组。 （示例对话起动器，请参阅下文。）使用您的发现，进一步设计用户准备的促销活动，并引导用户从那些可能受益于在以后升级分段。
</li><li>将映射用例。 您已经标识了将受益于这种新技术的核心用户组之后，考虑一般用户填充。 通过了解跨团队的用例和方案中，可以将可视化组织使用这种新技术。 此外，添加到您的通讯和培训计划使用情形示例实际上帮助使您的用户相关。</li></ul></td></tr>
</table>

对于每个角色组，讨论了项目小组成员如下：

-   该组 （即什么是典型的"一天生活中"） 的核心特征是什么？

-   此组将如何受益于新的解决方案？ 哪些特性和功能将它们需要有效？

-   哪些设备它们将需要 （如耳机或扬声器），以及他们是否已有这些？

-   什么附加价值传递的讯息，如果有的话，是需要此组开始使用新的解决方案中创建愿望？

-   若要接收此组的最佳方式是什么：

    -   （例如，电子邮件、 海报、 部门经理） 的意识？

    -   培训 （例如，在线培训、 用户指南、 白色手套）？

    -   支持 （自助，帮助台冠军）？  

-   为此组试验或早期采用程序的一个很好的例子吗？ 如果是这样，请购买中获取来自下列用户参与试验计划。 有关实现成功的先导计划的详细信息，请参阅[旁边 Skype 业务试点项目团队](pilot-essentials.md)。

为每个用例，讨论了项目小组成员如下：

-   此使用案例/方案的核心特征是什么？

-   关于当前的解决方案很好是什么？

-   什么，如果有的话，您面临的挑战与您当前的解决方案？

-   新的解决方案如何优化用户体验？

用例中的示例，请访问[FastTrack 生产力库](https://fasttrack.microsoft.com/microsoft365/productivitylibrary)。
