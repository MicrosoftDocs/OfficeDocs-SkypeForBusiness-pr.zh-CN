## <a name="unclassified-streams"></a><span data-ttu-id="2aba5-244">未经分类的流</span><span class="sxs-lookup"><span data-stu-id="2aba5-244">Unclassified Streams</span></span>

<span data-ttu-id="2aba5-245">在 CQD 中，当交互式连接建立 (ICE) 连接失败或未报告计算流分类所需的所有指标时，流将标记为"未分类"。</span><span class="sxs-lookup"><span data-stu-id="2aba5-245">In CQD, a stream is marked _Unclassified_ when Interactive Connectivity Establishment (ICE) connectivity fails or when all the metrics required to compute the stream classification are not reported.</span></span>

<span data-ttu-id="2aba5-246">要检查 ICE 连接故障，请检查 "First Connectivity Ice" 和 "Second Connection Ice" 维度是否为 "FAILED" 值。</span><span class="sxs-lookup"><span data-stu-id="2aba5-246">To check for ICE connectivity failures, examine the dimensions "First Connectivity Ice" and "Second Connectivity Ice" for a "FAILED" value.</span></span> <span data-ttu-id="2aba5-247">如果任一值指示失败，流将标记为 _"未分类"。_</span><span class="sxs-lookup"><span data-stu-id="2aba5-247">If either value indicates a failure, the stream is marked as _Unclassified_.</span></span>

<span data-ttu-id="2aba5-248">如果未分类流的 ICE连接成功，则流可能被视为"未分类"，因为未报告密钥流指标。</span><span class="sxs-lookup"><span data-stu-id="2aba5-248">If ICE connectivity succeeded for an _Unclassified_ stream, the stream is likely considered _Unclassified_ because key stream metrics were not reported.</span></span> <span data-ttu-id="2aba5-249">这些指标可能不会被报告的原因有几个：</span><span class="sxs-lookup"><span data-stu-id="2aba5-249">There are a few reasons these metrics may not be reported:</span></span>

- <span data-ttu-id="2aba5-250">**未收到 QoE** 报告 - 用于分类的指标在通话结束时发送的 QoE 报告中报告。</span><span class="sxs-lookup"><span data-stu-id="2aba5-250">**QoE reports were not received** — The metrics used for classification are reported in a QoE report sent at the end of a call.</span></span> <span data-ttu-id="2aba5-251">如果未生成此报告 (例如，由于某些第三方终结点可能无法发送 QoE) 或无法发送 (例如，由于网络中断) ，CQD 无法对流进行分类。</span><span class="sxs-lookup"><span data-stu-id="2aba5-251">If this report is not produced (for example, because some third-party endpoints may not send QoE) or could not be sent (for example, because of a network outage), CQD is unable to classify the stream.</span></span>

  > [!TIP]

- <span data-ttu-id="2aba5-255">**短调用** - 短调用可能没有足够的媒体活动来计算密钥流指标。</span><span class="sxs-lookup"><span data-stu-id="2aba5-255">**Short calls** — Short calls may not have enough media activity to compute key stream metrics.</span></span> <span data-ttu-id="2aba5-256">如果没有这些指标，CQD 无法对流进行分类。</span><span class="sxs-lookup"><span data-stu-id="2aba5-256">Without these metrics, CQD is unable to classify the stream.</span></span>

  > [!TIP]

- <span data-ttu-id="2aba5-259">**数据包利用率** 低 - 与"短调用"方案一样，计算密钥流指标需要足够的数据包利用率。</span><span class="sxs-lookup"><span data-stu-id="2aba5-259">**Low packet utilization** — Like the "short call" scenario, sufficient packet utilization is required for computation of key stream metrics.</span></span> <span data-ttu-id="2aba5-260">如果没有这些指标，CQD 无法对流进行分类。</span><span class="sxs-lookup"><span data-stu-id="2aba5-260">Without these metrics, CQD is unable to classify the stream.</span></span>
