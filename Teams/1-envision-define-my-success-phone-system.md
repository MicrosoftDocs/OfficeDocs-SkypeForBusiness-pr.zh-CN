# <a name="define-my-success"></a>我的成功定义

## <a name="introduction"></a>简介

此内容概述了正确定义为您的组织的电话系统调用计划部署成功的要求。 通过正确定义您的成功，可以通过您的部署过程测量结果，并验证您获得的结果，是您想要的结果。

<!--ENDOFSECTION-->

## <a name="define-business-use-cases-for-phone-system-with-calling-plans"></a>定义具有通话套餐的电话系统的业务用例

组织可以使用调用计划与电话系统现代化其工作场所通过使用户能够从自己的计算机和移动设备进行业务无关的电话联络。

工作场所现代化可以是任意数量的方案的一部分 — — 基于活动的工作实现，主要办公移动，office 拟合出刷新，淘汰旧的专用分组交换机 (PBX) 解决方案，公用电话的结论网络 (PSTN) 服务提供商的合同，等等。

开头的核心项目利益干系人需要定义业务用例支持电话系统与调用计划的实施。

业务用例用于定义和记录预期的、 可衡量的业务成效，并包括以下：

-   当前的业务流程的描述

-   与现有的业务流程的挑战

-   技术如何帮助克服这些挑战

-   克服了这些挑战的情况下预期的可衡量业务成果

> [!TIP]
> 下面是完成的业务用例示例：
>|         |
>|---------|
>|**当前业务流程说明**<br>Contoso 的办公室工作区的标准配置包括每个桌面上一部桌面电话。 每个雇员有权直接内部拨号 (DID) 的电话号码。 桌面电话连接到 PBX 系统，并通过会话启动协议 (SIP) 干线连接到 PSTN。 员工可以使用分配给自己的桌面电话拨打和接听电话。|
>|**现有业务流程上的挑战**<br>桌面电话的使用率分析显示只有 10%的桌面电话积极使用，配置为将呼叫转移到手机或同时铃声到手机的其余部分。 维护现有的 PBX 系统和相关联的台式电话分配给 Contoso 的每月电话服务成本的 20%。|
>|**技术如何能够克服这些挑战**<br>与调用计划的电话系统将允许的用户个人计算机上接收和利用本机 Microsoft 小组应用程序在数据网络上放置电话联络。 这不再需要加以推广和维护桌面电话，并打开停用现有的 PBX 系统的机会，因为可以通过网络与传统的电话系统不依赖通过云发送电话服务。|
>|**预期的可衡量业务成果**<br>拆卸维护方面的要求和停止使用传统的 PBX 和桌面电话将提供减少 20%，在每月的电话服务费用。 采用具有通话套餐的电话系统后，将会简化办公室工作区，由此，Contoso 可以投入极低的前期电话成本来设立新办公室，进而扩大运营。|

-   **项目时间线：**该项目将运行特定时间线。

## <a name="define-okrs-ksis-and-risks"></a>定义 OKRs、 KSIs 和风险

与装配项目风险承担者，可以将业务用例、 测试组织范围和项目时间线转换为目标和关键结果 (OKRs)，并可以作为关键的成功指标 (KSIs) 的列表定义项目成功的度量值。

项目利益相关者在定义 OKRs 和 KSIs 的完全参与至关重要，有助于确保他们感觉到一种所有权并对齐这些措施成功与否与组织的业务需求。

OKRs 包含您的项目，开始在设定的目标，并每季度定义可衡量的关键结果。 检查关键结果每月跟踪整个项目的状态和 — — 根据进度 — — 根据需要调整每季度计划。

> [!TIP]
> 与具有通话套餐的电话系统实施相关的 OKR 示例参考如下：
><br>
>
>**愿景：通过最大限度地利用 Office 365 投资来提高工作效率**
>|目标  |关键结果  |待办事项  |
>|---------|---------|---------|
>|到 2018 财年结束时在欧洲分支机构部署具有通话套餐的电话系统|FY18Q3：在伦敦办公地点部署具有通话套餐的电话系统|展望<ul><li>制定成功计划</li><li>制定详细的技术实施计划</li></ul><p>上线<ul><li>执行成功计划</li><li>执行技术实施计划</li></ul>|
>|到 2018 财年结束时在伦敦办公地点停用旧的 PBX|FY18Q4：在伦敦办公地点停用旧的 PBX|推动价值<ul><li>激励用户参与并推动采用</li><li>管理和准备变更</li><li>衡量、分享成功以及重复进行</li>|

KSIs 测量质量以及成功与否的关键结果，并通过详细报告好和/或坏结果补充二进制 OKRs （实现或未实现） 的性质。

在定义 KSIs 时，我们建议您使用"具体、 可衡量、 赋、 逼真，与时间相关"(SMART) 的条件：

-   特定： 目标改进特定区域

-   可衡量的： 量化，或至少表示进度的指标

-   谁将它赋： 指定

-   写实： 状态结果可以真实地实现给定的可用资源

-   与时间有关： 指定当可以得到的结果

## <a name="establish-a-steering-committee"></a>建立一个指导委员会

筹划指导委员会是主要利益相关者以及项目主管人员已投入管理组在一起，指导项目群向其定义的业务成果。 指导委员会并不直接负责*如何*交付项目的但而*哪些*项目实现的业务。

每个项目需要商定的构想和章程。 提供从项目所需的结果，设想必须清楚地定义，并且它需要监视和维护。 这一点的指导委员会的责任： 驱动器的决定、 建议、 战略监督作为拥护者对项目的计划，组织和 — — 在必要的时候 — — 删除广告拦截器。

组织应对大量缜密筹划指导委员会成立。 该委员会必须确保项目达到业务目标，对于推动整个企业，满足定期来讨论该项目，当前脉冲的更改已定义并帮助解除阻止沿遇到任何障碍方式。

该委员会应定义其宗旨包括一些关键的目标：

-   让项目团队和上级主管或管理层领导力之间的强对齐。

-   对上级主管或管理层领导力提供洞察项目的状态。

-   允许的上级主管或行政领导团队，提供指导并输入到该项目并确保它对齐与总体业务目标，通过调整项目计划、 目标关键结果 (OKRs) 和其他项目活动。

以周期性间隔的整个生存期内的项目，以确保组织的领导层和项目团队之间的对齐方式可以满足筹划指导委员会。 此关键会议确保项目的方向具有领导地位的全面支持和集成了领导到驱动器成功项目所提供的任何反馈。 该委员会使用这些会议深入了解项目状态和到：

-   在对齐到业务用例，并确保项目将朝这些成果的交付的业务成果达成一致。

-   检查和审核的准确性和法规遵从性的业务案例的项目。

-   复查并验证到业务用例可能会影响任何已定义的结果所做的更改。

-   使项目可交付结果的优先顺序有关的战略决策和批准中期可交付结果。

-   识别、 管理和缓解的间隙、 风险和其他影响所在需要从该委员会的问题。

-   从上级主管或行政领导团队的问题，需要升级，优先级并解决任何冲突的利益相关者业务部门之间争取支持。 

-   提供正式的反馈和建议管理层领导力、 更改顾问委员会，或其他业务和 IT 部门，（如果适用）。
