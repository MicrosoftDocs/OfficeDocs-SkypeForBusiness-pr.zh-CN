# <a name="document-my-success-plan"></a>文档我成功的计划

这篇文章概述用于正确记录云语音部署要求。 通过定义和记录所有决策点及后续步骤规划云语音部署时，您可以确保所有利益干系人和项目工作组成员在提供成功产出对齐。 

## <a name="execution-planning"></a>执行规划 

定义如何您将音频会议或电话系统与调用规划解决方案中实现您的组织之后，您需要规划实施项目的执行。

如果您的组织具有一个或两个网站，您可能不需要完成本文中提供的所有详细信息，但您应阅读通过它可指导您方法。

<!--ENDOFSECTION-->

## <a name="deployment-model"></a>部署模型 

转换方式任何技术实现与人员在您的组织中工作，选择将要部署的正确方法会显著影响云语音实现成功。

潜在部署模型包括：

-   **每个网站：** 此模型是合适的情况下，地理上分散您的组织，并且分支有大量的员工。 但是，此部署模型可以可能会中断通信内部门员工，分布在多个位置的部门。

-   **每个部分**： 此模型通常的中型公司更好的选择，并确保所涉及的部门具有相同的体验。

-   **整个公司同时：** 此模型通常是小公司了最佳选项，其中所有雇员都获得部署的第一天的相同的体验。

## <a name="finalize-my-success-plan"></a>完成我成功的计划

成功计划已在构想阶段创建的文档的摘要。

成功计划提供项目工作组 — 其中可以包括 FastTrack 或部署合作伙伴 — 足够的信息来实现与调用计划服务中实现的音频会议或电话系统与您的组织目标。

一般情况下，一个成功计划将包含以下主要章节，许多哪些将过通过构想阶段：

-   业务案例

-   服务就绪

-   服务决策

-   执行计划

-   采用计划

-   运营计划

### <a name="business-case"></a>业务案例

业务用例的利益干系人、 OKRs 和 KSIs、 风险 register 和项目时间表列表通常组成批量业务案例所需的信息。 您应记录这些成功计划的一部分。

### <a name="service-readiness"></a>服务就绪

您的环境评估提供确定贵组织的技术就绪性实现音频会议和/或电话系统与调用规划所需的初始信息。

这里是您服务的准备情况评估和计划对需要通过环境评估发现的补救的地址区域。

### <a name="service-decisions"></a>服务决策

文档如何为您的组织将音频会议或电话系统计划与调用规划服务技术实现。

### <a name="execution-plan"></a>执行计划

记录计划要实现整个组织中的解决方案的项目执行的方式。

### <a name="adoption-plan"></a>采用计划

执行您采用准备情况评估后，项目团队需要以拿一组全面的通信计划、 培训计划和计划预备，启动，并后启动采用活动。

确定用于支持采用活动，如传单、 欢迎电子邮件和培训资料，以及您需要满足组织的要求的任何自定义资源。

从[Microsoft 团队客户成功工具包](https://www.microsoft.com/download/details.aspx?id=54244)下载应用活动的模板。

### <a name="operational-plan"></a>运营计划

映射操作角色的练习将建立角色和职责，以及分配给每个操作的角色，您需要支持音频会议的实现的团队。

你需要完成此项，并在成功计划中包括运营计划以确保完成解决方案的运营就绪。
