# <a name="upgrade-journey-and-coexistence-of-skype-for-business-and-microsoft-teams"></a>升级历程和共存的 Skype 业务和 Microsoft 小组

作为业务客户现有 Skype，完成过渡团队可能需要一些时间。 但是，您可以开始允许您的用户使用业务团队一起 Skype 今天，实现团队的价值。 考虑到有一些重叠的功能之间的两个应用程序，我们建议您查看可用的升级模式以帮助您确定哪种方法最适合您的组织。 例如，您可以选择启用互操作性而这两种解决方案的所有工作。 或者，您可能决定进行管理的用户体验，逐渐引入团队功能或目标组的用户选择功能，直至您的组织已准备好升级到团队的每个人。

与任何部署中，我们极力建议您到[试验预期的计划](pilot-essentials.md)到一组选定的用户之前推出到更广泛组织团队。 请记住，引入新的技术可以为用户中断。 需要时间来评估用户准备情况和实施通信和培训计划，以使用户了解并加快其接受的团队。 

> [!IMPORTANT]
> 业务团队快速发展升级历程为 Skype。 这篇文章将帮助您了解即将到来的团队升级，但它不包括端到端的升级说明。 从 Skype 业务成功的升级将包括技术和用户准备工作为重点的活动。 我们正在从事现在，本指南，我们将会发布它很快。
 
## <a name="upgrade-and-coexistence-modes-defined"></a>升级和共存模式定义
为了帮助指导您的决策过程，熟悉各种模式、 概念和术语相关的从 Skype 业务升级到团队。

### <a name="skype-for-business-with-teams-collaborationndashonly-mode"></a>与团队协作业务的 Skype&ndash;模式

在此模式下，团队被配置以支持团队和基于通道的对话，与私人聊天，电话和禁用的会议。

这种模式是引入团队环境中同时继续利用 Skype 业务在现有投资的好方法。 

### <a name="skype-for-business-with-teams-collaboration-and-meetings-mode"></a>Skype 业务与团队协作和会议模式
使用团队的团队，在此模式下基于通道的对话，以及用户开始使用团队安排和主持其会议。 私人聊天室和语音和视频通话，留在 Skype 的业务上。

如果您的组织需要团队会议，提供的功能，但业务需求要求您在 Skype 业务保持企业语音功能，您可以配置团队能够提供业务的 Skype 与团队协作和会议为您的整个组织 （或它的某些部分） 的功能。

### <a name="islands-mode"></a>孤岛模式
这是默认模式。 您可以使用它来部署独立于业务的 Skype 的团队。 

在此模式下，团队用户可以直接使用团队与所有功能与团队的其他用户，而其他用户可以继续使用 Skype 业务并保持对如何使用 Skype 的业务目前没有影响工作效率。

> [!NOTE]
> 孤岛模式附带允许聊天互操作性从团队到 Skype 的业务，在满足以下条件时，单向临时的互操作性：
> - 工作组用户穴，并且启用，Skype 的在线业务。
> - 业务用户的 Skype 从未使用过小组 （或团队不许可）。
>
> 当您希望用户主要使用团队仍在使用 Skype 的业务只在组织中其他用户的连接时，此模式非常有用。
>
> 以前所用的业务只是 Skype 的用户开始使用小组后，小组将到达他们来自其他团队的用户的所有聊天。 这会将这些用户切换到群岛模式。 从那一刻开始，他们必须继续运行团队，以确保它们保持连接与团队组织中的其他用户。

### <a name="teams-only-mode"></a>仅小组模式
在已升级的模式中，Skype 业务客户端不再提供基本的即时消息、 状态指示器、 聊天、 电话或会议功能。 在已升级的模式下工作的用户必须使用团队作为其主要的通信和协作工具。

准备使用作为其主要的通信和协作工具的团队的用户可以为团队只读用户升级。

已升级的用户只可以使用 Skype 业务客户端加入现有 Skype 业务会议或 Skype 业务按非升级用户或外部各方组织的会议。 已升级的用户可以继续与组织中仍在使用 Skype 业务为通过使用业务团队和 Skype 之间的互操作性功能的其他用户进行通信。

### <a name="skype-for-businessndashonly-mode"></a>Skype 业务&ndash;模式
用户需要在 Skype 业务可以配置为 Skype 业务&ndash;只有用户。 在此模式下配置的用户将无法使用团队，将能够通过使用业务团队和 Skype 之间的互操作性功能与团队专用用户通信。

## <a name="upgrade-and-coexistence-building-blocks"></a>升级和共存的构建基块
若要正式准备组织到团队行程，需要开始为升级方案，最终使您完全接纳作为唯一的通讯和协作解决方案的团队的组织计划。

准备就绪后，您的部分用户使用仅团队他们日常的通信和协作需求，您可以开始为它们启用仅团队模式这些用户升级到团队。

如果不是适用于整个组织移到团队，可以通过完全采用团队作为组的协作解决方案，首次同时为您的组织的统一的通信解决方案业务的 Skype 启动。 另一种方法是开始转向团队除了引入团队组协作的功能，同时保持统一的通信功能的其余部分在 Skype 的业务会议。

## <a name="upgrade-journeys"></a>升级的到达
您可以采取多种方法从 Skype 业务，可以在线升级到或内部，对团队：
- 简单的升级过程中，在首次部署在 Skype 业务团队与团队协作&ndash;仅作为评估和早期采用的模式，然后最终淘汰的业务从 Skype 的目标的实现仅小组模式组织中所有用户的的环境。
- 逐步升级历程为一组特定的用户 （也称为*群体*），这取决于他们的通信和协作要求提供特定升级模式。 随着时间的推移整个组织可以汇聚成只使用团队并最终取代 Skype 业务。 但是，如果您的组织有令人信服的业务原因保留业务的 Skype — — 如依赖基于托管 API UCMA 统一通信的解决方案集成的业务线应用程序或一堵隔离墙的解决方案当前可用于业务仅为 Skype — — 同时保留适用于业务用户 Skype 的一部分的用户群的业务模式之一的 Skype 升级的用户大部分仅团队模式。

> [!IMPORTANT]
> 对于这两种类型的升级历程，如果您的组织当前 Skype 业务内部仅针对部署，您需要开始规划到仅团队模式升级您的用户之前为业务混合实现 Skype。 这还将有助于促进团队与互操作性。
> 使用[MyAdvisor](https://myadvisor.fasttrack.microsoft.com/)可以指导您 Skype 业务混合实现的。

> [!NOTE]
> 只有团队模式要求的业务联机，被穴在 Skype 的用户群体的且需要便于升级您 Skype 业务内部部署与您的在线业务的租户 Skype 之间的混合关系和为业务 Skype 与团队之间的互操作性。 用户是群体的一部分，他们要到仅团队模式升级前，必须完成转向 Skype 的在线业务。 Skype 的业务服务器 2019，以及为业务服务器 2015，Skype 未来累积更新计划简化管理到 Skype 的在线业务的迁移和升级到只有团队的用户升级到团队的内部用户的机制在一个步骤中的模式。

选择，任何一个旅程 Microsoft 在此为您完成升级过程提供支持。 从用户准备工作资源的技术要求，我们将介绍指南和最佳实践，帮助确保成功过渡从 Skype 的业务团队。 详细的升级指南将很快会准备好，因此一定要重新检查的详细信息。
