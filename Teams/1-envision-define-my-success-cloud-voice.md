# <a name="define-my-success"></a>定义成功

本文概述了为组织定义成功部署音频会议、电话系统呼叫计划或电话系统直接路由的要求。 通过正确定义成功外观，可以在部署进行时衡量结果，并验证所实现的结果是否达到预期效果。

<!--ENDOFSECTION-->

音频会议通过允许会议参与者通过使用传统座机、专用交换机 (PBX) 或移动电话拨入，通过公用电话交换电话网络 (PSTN) 加入任何会议 (即席或计划)  (，为组织提供了额外的入口点。 当组织者或参与者不在计算机前面，或者数据连接不可用或太不可靠，无法支持语音通信时，例如，在移动数据覆盖度低的远程区域中，或者连接到带宽有限的免费公共 Wi-Fi 服务，或者当会议参与者希望使用电话终结点拨入会议时，这非常有用。

**电话系统呼叫计划 ("呼叫计划")** 使组织能够从计算机和移动设备进行业务相关的电话呼叫，为组织提供了一种现代化工作区的方法。 工作场所现代化可以是任何数量的方案的一部分-基于活动的工作实现、重大办公室移动、办公室调整刷新、停用旧 PBX 解决方案、结束 PSTN 服务提供商合同，等等。 借助呼叫计划，Microsoft 促进与 PSTN 的连接。

**电话系统直接路由 ("直接路由")** 为组织提供上面列出的呼叫计划的相同权益，但 PSTN 连接由第三方提供商而不是 Microsoft 提供。 这允许部署在呼叫计划不可用的国家/地区，或者需要维护现有 PSTN 服务提供商合同或需要与某些本地系统互操作性的部署中。 考虑直接路由的另外一个方案是电话系统互操作性。 当用户正在转换到在 Teams 呼叫时，某些用户可能仍使用旧版 PBX。 直接路由允许这两种用例共存。 旧系统上的用户与用户之间的呼叫Teams用户位于组织内部。

<!--ENDOFSECTION-->

## <a name="define-business-use-cases-for-audio-conferencing-calling-plans-or-direct-routing"></a>定义音频会议、呼叫计划或直接路由的业务用例

首先，核心项目利益干系人需要定义支持音频会议、呼叫计划或直接路由实现的业务用例。

业务用例旨在定义和记录预期和可度量的业务成果，并包括以下内容：

-   当前业务流程

-   现有应用程序的挑战业务流程

-   技术如何帮助克服这些挑战

-   克服了这些挑战的情况下预期的可衡量业务成果

## <a name="define-okrs-ksis-and-risks"></a>定义 OKR、KSIS 和风险

通过组合项目利益干系人，可以将业务用例、组织范围和项目日程表转换为目标和关键结果 (OKR) ，项目成功的衡量指标可定义为关键成功指标列表 (KSIs) 。

项目利益干系人全面参与定义 OKR 和 KSIS 至关重要，有助于确保他们感到有所有权感，并且这些成功衡量指标与组织业务要求保持一致。

OKR 包含你在项目开始时设置的目标，并且你每季度定义可度量的关键结果。 每月查看关键结果，跟踪整个项目的状态，并根据需要调整季度计划（基于进度）。

KSIs 通过详细说明好结果和/或坏结果来测量关键结果的质量与成功，并补充 (或未实现的 OKR) 二进制性质。

定义 KSIS 时，我们建议在 SMART) 条件中，使用"特定、可测量、可分配、现实、与时间相关的 (：

-   特定：针对特定领域进行改进

-   可度量：量化或至少建议进度指标

-   可分配：指定谁将执行该任务

-   现实：在给定可用资源后，说明可以实际实现哪些结果

-   时间相关：指定何时可以实现结果

## <a name="establish-a-steering-committee"></a>建立指导委员会

指导委员会是一组关键利益干系人与项目负责人，这些利益干系人与项目负责人一起共同指导项目或计划实现定义的业务成果。 指导委员会不直接负责项目的交付方式，而负责项目向业务交付什么。 

每个项目都需要达成一致的愿景和规定。 若要实现项目所需的成果，必须明确定义愿景，并且需要监视和维护该愿景。 这将成为指导委员会的责任：推动决策、提出建议、提供战略监督、为组织宣传项目计划，并在必要时删除阻止者。

组织应大量考虑组建指导委员会。 委员会必须确保项目实现你为推动整个组织的变化而定义的业务目标，定期开会讨论项目的当前脉动，并帮助消除在过程中遇到的任何障碍。

委员会应定义其规定，以包括一些关键目标：

-   在项目团队与执行发起人或高层领导之间保持强大的一致性。

-   向执行发起人或高层领导提供有关项目状态的见解。

-   通过调整项目计划、目标关键结果 (OKR) 和其他项目活动，允许执行发起人或高层领导团队为项目提供方向和意见，并确保它与总体业务目标保持一致。

指导委员会在整个项目生命周期内定期开会，以确保组织领导与项目团队保持一致。 此关键会议可确保项目方向获得领导层的完全支持，并将领导层提供的任何反馈纳入到项目中，以推动成功。 委员会使用这些会议来深入了解项目状态，并：

-   就符合业务案例的业务成果达成一致，并确保项目推动这些成果的交付。

-   检查和批准项目，确保项目的准确性和合规性符合业务案例。

-   查看并验证对业务案例所做的更改，这些更改可能会影响任何已定义的结果。

-   就项目可交付结果的优先级做出战略决策，并审批临时可交付结果。

-   确定、管理和缓解需要委员会施加其他影响的差距、风险和问题。

-   针对需要升级、确定优先级和解决利益干系人业务部门之间任何冲突的问题，从执行发起人或执行领导团队获得支持。 

-   向管理层、变更顾问委员会或其他业务和 IT 利益干系人（如果适用）提供正式反馈和建议。
