# <a name="private-channels-in-microsoft-teams"></a>Microsoft 团队中的专用频道

Microsoft 团队中的专用频道为团队内的协作创建了重点空间。 只有拥有专用频道的所有者或成员的团队用户才可以访问频道。 任何人（包括来宾）都可以添加为专用频道的成员，只要他们已经是团队的成员。

如果你想要将协作限制到需要知道的人员或希望在分配给特定项目的一组人员之间进行通信，而无需创建其他要管理的团队，你可能需要使用专用通道。

例如，在以下情况下，专用信道非常有用：

- 团队中的一组人员希望有重点的空间进行协作，而无需创建单独的团队。
- 团队中的人员的子集希望使用专用通道来讨论敏感信息，如预算、预算、战略定位等。

"锁定" 图标表示专用频道。 只有专用信道的成员可以查看和参与添加到的专用通道。

![团队中专用频道的屏幕截图](media/private-channels-in-teams.png)

## <a name="what-you-need-to-know-about-private-channels"></a>您需要了解的有关专用频道的哪些信息

目前，专用通道支持连接器和选项卡（流、Planner 和窗体除外）。 我们正在为专用频道（包括消息扩展和机器人）提供完整的应用支持。

每个团队最多可以有30个专用信道，每个专用通道最多可以有250个成员。 30个专用频道限制是每个团队200标准频道限制的补充。

> [!NOTE]
> 我们将不断向专用频道添加功能，以便查看有关应用、频道会议和大型团队的专用频道的最新信息。

## <a name="private-channel-creation-and-membership"></a>专用频道创建和成员身份

### <a name="who-can-create-private-channels"></a>哪些人可以创建专用频道？

默认情况下，任何团队所有者或团队成员都可以创建专用通道。 来宾无法创建。 创建专用频道的功能可以在团队级别和组织级别进行管理：

- 在团队的 "**设置**" 选项卡上，团队所有者可以关闭或打开成员创建专用频道的功能。
- 作为管理员，你可以使用[策略](teams-policies.md)控制允许组织中的哪些用户创建专用频道。

创建专用频道的人员是专用频道所有者，只有专用频道所有者可以直接在其中添加或删除人员。 专用频道所有者可以将任何团队成员添加到他们创建的专用频道中，包括来宾。 专用通道的成员具有安全的对话空间，当添加新成员时，他们可以在该专用频道中看到所有对话（甚至是旧对话）。

### <a name="what-happens-when-a-team-member-leaves-or-is-removed-from-a-team"></a>当团队成员离开或已从团队中删除时，会发生什么情况？

如果团队成员离开或已从团队中删除，则该用户也将离开或从团队中的所有专用频道中删除。 如果用户已添加回团队，则必须将其添加回团队中的专用频道。

### <a name="what-happens-when-a-private-channel-owner-is-removed-from-a-private-channel"></a>从专用频道删除专用频道所有者时会发生什么情况？

如果专用通道所有者是一个或多个专用频道的最后一个所有者，则不能通过团队客户端将其删除。

如果专用频道所有者离开你的组织，或者从与团队关联的 Office 365 组中删除了它们，则专用通道的成员将自动提升为专用通道所有者。

### <a name="what-can-team-owners-and-team-members-see-in-a-private-channel"></a>团队所有者和团队成员可以在专用频道中看到哪些内容？

团队所有者可以查看其团队中所有专用通道的名称，也可以删除团队中的任何专用通道。 （删除后，可以在30天内恢复已删除的专用频道）。 团队所有者看不到专用频道中的文件或专用频道的对话和成员列表，除非它们是该专用频道的成员。

下表显示了哪些人可以查看专用频道中的内容。

|项目  |团队所有者可以查看 |团队成员可以查看 |
|---------|---------|---------|
|名称和说明    |团队中的所有专用通道         |仅将其添加到的专用通道         |
|对话和选项卡     |仅在添加到专用频道时         |仅在添加到专用频道时         |
|文件和内容    |仅在添加到专用频道时        |仅在添加到专用频道时         |
|专用频道所有者    |团队中的所有专用通道        |仅在添加到专用频道时         |
|上次活动时间戳  |团队中的所有专用通道       |仅在添加到专用频道时         |

### <a name="manage-private-channel-membership-and-settings"></a>管理专用频道成员身份和设置

每个专用频道都有其自己的设置，包括为整个频道添加和删除成员、添加选项卡和 @mentioning 的功能。 这些设置独立于父团队设置。 创建专用通道后，它将继承父团队的设置，在此期间，其设置可以独立于父团队设置进行更改。

专用频道所有者可以单击 "**管理频道**"，然后使用 "**成员**" 和 "**设置**" 选项卡添加或删除成员和编辑设置。

## <a name="private-channel-sharepoint-sites"></a>专用频道 SharePoint 网站

每个专用频道都有自己的 SharePoint 网站集，针对文件共享和快速资源调配进行了优化。 单独的网站集是确保仅限于专用频道的成员与团队所有者有权访问网站集内的所有资产的团队网站的成员。 默认情况下，这些网站集是使用文档库创建的，并且可以通过[网站管理界面](https://support.office.com/article/Enable-or-disable-site-collection-features-A2F2A5C2-093D-4897-8B7F-37F86D83DF04)轻松地增强到功能齐全的网站集。 每个网站集在与父团队的网站集相同的地理区域中创建。 这些轻量级网站具有自定义模板 ID "TEAMCHANNEL # 0"，以便通过 PowerShell 和 Graph API 更轻松地进行管理。

为了容纳每个租户的更多网站集，该限制已从500000增加到2000000。 专用频道网站集同步数据分类，并从父团队的网站集继承来宾访问权限。  网站集所有者和成员组的成员身份与团队中的专用频道的成员身份保持同步。 对 SharePoint Online 中的所有者或成员组的成员身份所做的任何更改都将在四个小时内自动还原到专用通道成员身份。 在某些用户需要访问文档而无需访问专用频道消息的情况下，将它们添加到网站上的 "访问者" 组或与 "所有者" 和 "成员" 不同的新组中。

团队管理专用通道 SharePoint 网站集的生命周期。 如果在团队外删除网站集，后台作业将在四个小时内恢复网站，只要专用通道仍处于活动状态。 如果网站已删除并已硬删除，则为专用频道设置新的网站集。

如果已还原专用频道或包含专用频道的团队，则网站集将随之还原。 如果专用通道网站集已还原，并且超出专用通道的30天软删除窗口，则网站集将作为独立的网站集进行操作。

## <a name="considerations-around-access-in-private-channels"></a>有关专用通道中的访问权限的注意事项

在专用频道中创建新的 OneNote 笔记本时，其他用户仍然可以访问笔记本，因为该行为与用户共享专用通道 SharePoint 网站中的任何其他项目的访问权限相同。

如果通过 SharePoint 通过 SharePoint 向用户授予对笔记本的访问权限，则从团队或专用频道中删除用户不会删除用户对笔记本的访问权限。

如果现有笔记本作为选项卡添加到专用频道，则无法更改专用通道的访问权限。 这意味着以下内容：

- 默认情况下，私人频道中并非每个人都有权访问该笔记本。 这是因为他们可能无法访问托管笔记本的位置，例如另一个团队的 SharePoint 网站。
- 不是专用频道成员的用户可以查看笔记本。
