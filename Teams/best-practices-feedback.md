# <a name="best-practices-for-user-feedback-methods-in-microsoft-teams"></a>Microsoft Teams 中用户反馈方法的最佳做法

获得新的协作体验就是要改变你的用户的行为。 人的变化需要培训、鼓励和正面的示例。 在过渡期间，人们也可以听到这一点，这一点也非常重要。 如果你以前在贵组织中建立了成功的反馈方法，请考虑将其扩展以应用于你的 Teams 实现。 如果以前尚未这样做，请考虑以下方法之一：

- 在团队内Microsoft Teams任何用户都可以加入的公共团队，以提供有关其体验的反馈 (请记住，公共组限制为 5000 名) 。

- 使用 Yammer 为体验提供最佳做法和支持的开放社区。

社区驱动型反馈方法要求支持者和服务所有者参与。 我们建议创建一个监控计划，在这些成员之间轮换执行。 一个提示是，在 Microsoft Teams内创建一个单独的团队，作为这些冠军在与更广泛的团队共享这些资源和最佳做法之前，可以在这里进行协作、共享资源和最佳做法。 支持者向采用新服务的用户执行咨询角色，提供友好的人对更改的面容，并与部署团队的其他成员协作来提出和解决问题。

## <a name="showcases"></a>突出表现

随着你继续这一旅程，你将在你自己的员工环境中找到展示，这些员工的创造力Microsoft Teams带来惊喜和愉悦。 广泛共享这些展示。 这将鼓励其他人创新、培训采用曲线更早的用户，并奖励那些已花时间思考如何最好地使用Microsoft Teams应用的用户。 为了保持你的采用推进力，每个季度设定一个用于发现一定数量的突出表现的可衡量目标很有用。
