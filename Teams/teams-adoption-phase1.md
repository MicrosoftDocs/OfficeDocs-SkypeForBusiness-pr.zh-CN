# <a name="microsoft-teams-adoption-phase-1---start"></a>Microsoft Teams 采用第 1 阶段 - 开始

你将通过以下活动开始采用 Teams：

- [验证采用先决条件](#validate-adoption-prerequisites)。
- [组建你的团队](#assemble-your-team)。
- [了解团队和频道](teams-adoption-understand-teams-and-channels.md)。
- [在 Teams 上设置第一个团队](teams-adoption-your-first-teams.md) 以建立技术知识。
- [评估组织的更改准备情况](teams-adoption-assess-readiness.md)。

此阶段的主要目标是确保做好试验阶段的准备。 你还将确定你是否需要 Microsoft 或我们经验丰富的合作伙伴网络的协助。  

## <a name="validate-adoption-prerequisites"></a>验证采用先决条件

查看 [我们的采用先决条件](teams-adoption-get-started.md#adoption-prerequisites) ，并在必要时与技术团队合作以满足这些先决条件。 我们将在每个采用方案中解决更深入的技术要求。

## <a name="assemble-your-team"></a>组建团队

下表列出了项目团队的核心角色。 在较小的组织中，其中一些角色可能由同一个人执行。

| 角色 | 职责 | 部门 |
| ---- | ---------------- | ---------- |
| 执行发起<sup>人 1</sup> | 传达高级愿景。 将 Teams 实施连接到公司的核心任务和优先计划。 | 高层领导 |
| 成功所有者<sup>1</sup> | 确保通过 Teams 推出实现业务目标。 | 任何部门 |
| 项目经理<sup>1</sup> | 监督整个 Teams 启动执行和推出过程。 | IT |
| 得<sup>主 1</sup> | 帮助推广 Teams、支持技能构建，并提供使用情况见解。 | 多个部门 |
| 培训主管 | 管理和传达有关 Teams 的培训内容。 | IT 或其他 |
| 部门主管 (利益干系人) <sup>2</sup> | 确定特定部门如何使用 Teams 并鼓励参与。 | 任何部门 (管理)  |
| IT 专家<sup>1</sup> | 监督实施的所有技术层面，包括集成。 | IT |
| 通信主管<sup>2</sup> | 监督公司范围内有关 Teams 的通信。 | 企业通信、IT 或其他 |
| 社区管理员 | 管理日常 Yammer 网络活动。 提供指导和最佳做法。 | 多个部门 |

<sup>1</sup>这些角色对于项目的成功至关重要。</br>
<sup>2</sup>这些角色对于总体计划的成功非常重要。

在大多数情况下，此初始团队应是一小组对 Teams 感兴趣的、也可能需要对其实施负责的志同道合的人。 随着采用计划进入其他阶段，团队中将包含更多人员。 我们还建议你利用 Microsoft 资源来帮助你。
