# <a name="microsoft-teams-adoption-phase-1---start"></a>Microsoft Teams 采用第 1 阶段 - 开始

你将通过以下活动开始采用 Teams：

- [验证采用先决条件](#validate-adoption-prerequisites)。
- [组建团队](#assemble-your-team)。
- [了解团队和频道](teams-adoption-understand-teams-and-channels.md)。
- [在 Teams 上设置第一个团队](teams-adoption-your-first-teams.md) 以生成技术知识。
- [评估组织的更改准备情况](teams-adoption-assess-readiness.md)。

此阶段的主要目标是确保做好试验阶段的准备。 你还将确定是否需要 Microsoft 的帮助或来自我们熟练合作伙伴网络的帮助。  

## <a name="validate-adoption-prerequisites"></a>验证采用先决条件

查看我们的 [采用先决条件，](teams-adoption-get-started.md#adoption-prerequisites) 并与技术团队合作，在必要时满足这些先决条件。 我们将在每个采用方案中解决更深层次的技术要求。

## <a name="assemble-your-team"></a>组建团队

下表列出了项目团队的核心角色。 在较小的组织中，其中一些角色可能由同一个人执行。

| 角色 | 责任 | 部门 |
| ---- | ---------------- | ---------- |
| 执行发起人<sup>1</sup> | 传达高级视觉。 将 Teams 实现连接到公司的核心任务和优先级计划。 | 执行领导 |
| 成功所有者<sup>1</sup> | 确保通过 Teams 推出实现业务目标。 | 任何部门 |
| Program Manager<sup>1</sup> | 监督整个 Teams 启动执行和推出过程。 | IT |
| 冠军<sup>1</sup> | 帮助传福音 Teams、支持技能构建，并提供使用情况见解。 | 多个部门 |
| 训练主管 | 管理和传达有关 Teams 的培训内容。 | IT 或其他 |
| 部门主管 (利益干系人) <sup>2</sup> | 确定特定部门将如何使用 Teams 并鼓励参与。 | 任何部门 (管理)  |
| IT 专家<sup>1</sup> | 监督实现的所有技术方面，包括集成。 | IT |
| 通信线索<sup>2</sup> | 监督公司范围内有关 Teams 的通信。 | 公司通信、IT 或其他 |
| 社区管理器 | 管理日常 Yammer 网络活动。 提供指导和最佳做法。 | 多个部门 |

<sup>1</sup>这些角色对于项目的成功至关重要。</br>
<sup>2</sup>这些角色对于整个计划的成功非常重要。

在大多数情况下，这个初始团队应该是一小群志同道合的人，他们对 Teams 感兴趣，并可能对其实施负责。 随着采用计划进入其他阶段，团队中将包含更多人员。 我们还建议你利用可用于帮助的 Microsoft 资源。
