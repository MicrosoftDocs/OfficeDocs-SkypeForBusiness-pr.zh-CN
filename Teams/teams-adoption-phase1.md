# <a name="microsoft-teams-adoption-phase-1---start"></a>Microsoft Teams 采用第 1 阶段 - 开始

您将通过以下活动开始团队的采纳：

- [验证采纳先决条件](#validate-adoption-prerequisites)。
- [召集团队](#assemble-your-team)。
- [了解团队和频道](teams-adoption-understand-teams-and-channels.md)。
- [在团队中设置你的第一个团队](teams-adoption-your-first-teams.md)以构建你的技术知识。
- [评估您的组织的准备情况以进行更改](teams-adoption-assess-readiness.md)。

此阶段的主要目标是确保实验阶段的准备就绪。 您还将确定您是否需要 Microsoft 或来自训练有素合作伙伴网络的帮助。  

## <a name="validate-adoption-prerequisites"></a>验证采纳先决条件

查看我们的[采纳先决条件](teams-adoption-get-started.md#adoption-prerequisites)，并与技术团队协作，如有必要，请与他们见面。 我们将在每个采纳方案中解决更深入的技术要求。

## <a name="assemble-your-team"></a>召集团队

下表列出了项目工作组的核心角色。 在小型组织中，其中一些角色可能由同一个人执行。

| 角色 | 交给 | 部门 |
| ---- | ---------------- | ---------- |
| 总经理主办方<sup>1</sup> | 传达高级别愿景。 将团队实施与公司中的核心任务和优先级计划联系起来。 | 管理层领导 |
| 成功所有者<sup>1</sup> | 确保从团队推出中实现业务目标。 | 任何部门 |
| 计划经理<sup>1</sup> | 监视整个团队启动执行和推出过程。 | 更加 |
| 拥护机<sup>1</sup> | 帮助布道团队、支持技能构建和提供使用见解。 | 多个部门 |
| 培训负责人 | 管理和传达有关团队的培训内容。 | IT 或其他 |
| 部门领导（利益干系人）<sup>2</sup> | 确定特定部门将如何使用团队和鼓励参与。 | 任何部门（管理） |
| IT 专家<sup>1</sup> | 监督实施的所有技术方面，包括集成。 | 更加 |
| 通信组长<sup>2</sup> | 监督公司范围内有关团队的沟通。 | 公司沟通、IT 或其他 |
| 社区经理 | 管理日常 Yammer 网络活动。 提供指南和最佳做法。 | 多个部门 |

<sup>1</sup>这些角色对项目的成功至关重要。</br>
<sup>2</sup>这些角色对于整个计划的成功非常重要。

在大多数情况下，此初始团队应该是一小组对团队感兴趣的志同道合人员，也可能对其实施负责。 随着你的采纳计划移动到其他阶段，你的团队中将包含更多人员。 我们还建议您利用可帮助您解决的 Microsoft 资源。
