# <a name="plan-for-clients-and-devices"></a>规划客户端和设备
 
**摘要：**对 Skype 业务支持客户端和应用程序的审查。

## <a name="ways-to-deploy-the-client-to-your-users"></a>向用户部署客户端的方式
<a name="BK_User_Deploy"> </a>

客户端安装程序可供 MSI 和即点即用型的安装程序使用。 业务客户端 sustainment 战略为 Skype 可能影响您的选择，因此您应该了解以下几点： 
  
- 一般情况下 Skype 的业务不会添加新功能对以前发布的客户端
    
- 一般情况下 Skype 业务尚不打算在发布新功能在 Skype 业务 MSI 的初始发布后。 各版本之间的 MSI 改进在本质上主要是质量/安全方面的改进。 
    
- 业务客户端体验的最新和最好的 Skype 节在 Skype 业务 2016年单击以运行安装程序。 
    
您可以执行[业务服务器 2015年的 Skype 在自定义 Windows 客户端安装](../../deploy/deploy-clients/customize-windows-client-installation.md)中所述的客户端的自定义的部署。 在[部署客户端的业务服务器 2015年的 Skype](../../deploy/deploy-clients/deploy-clients.md)的更详细地介绍了安装方法
  
### <a name="click-to-run"></a>即点即用

单击以运行是 Microsoft 流和虚拟化技术，可用于安装和更新 Office 产品包括 Skype 业务。 这些流式和虚拟化功能以 Microsoft Application Virtualization (App-V) 中的技术为基础。 即点即用具有以下优势： 
  
- Office 套件的流式安装可缩短安装时间
    
- 流式处理更新和修补程序
    
- 占用的磁盘空间更少
    
- 基于用户的许可：每个用户安装 5 次
    
- 可通过用于安装独立程序的 XML 编辑器进行自定义
    
您可能需要为这种类型的安装程序使用[Office 2016 部署工具](https://www.microsoft.com/en-us/download/details.aspx?id=49117)。
  
同时基本的和完整的客户端版本 （32 位和 64 位版本的选项） 都可以使用单击以运行安装程序，您的用户可以在[业务分散到所有设备下载 Skype](https://products.office.com/en-us/skype-for-business/download-app?tab=tabs-3)下载。 
  
### <a name="msi"></a>MSI

MSI 是更传统的安装方法。 它允许您手动安装更新和修补程序、 使用批量许可和激活，并通过[Office 自定义工具](http://www.microsoft.com/en-us/download/details.aspx?id=49030)可自定义。 可通过应用组策略、使用 System Center Configuration Manager 或使用第三方工具来分发客户端。
