# <a name="plan-for-location-based-routing-in-skype-for-business-2015"></a>在 Skype for Business 2015 中规划基于位置的路由
 
Skype 在基于位置的路由业务企业语音，包括交互与并发响铃和委派，并支持基于位置的路由方案的规划。
  
基于位置的路由，使得可能限制电话 VoIP 终结点和 PSTN 基于位置的调用方的端点之间的路由。 基于位置的路由是控制呼叫业务服务器的路由通过 Skype 呼叫管理功能。 它强制上是否可调用传送给基于业务呼叫者的地理位置为 Skype 的 PBX 或 PSTN 终结点的调用授权规则。
  
基于位置的路由引入了一组新规则，它们可修改国内和国际 PSTN 呼叫的路由以防止收费绕路情形。使用基于位置的路由可灵活地将这些规则限定为仅特定区域、特定网关或特定用户集。
  
以下方案说明了主要类型的基于位置的路由可强制实施某些的限制：
  
- 出站呼叫的基于位置的路由可以强制执行拨出电话要出口到位于同一区域为调用方旨在防止 PSTN 免费，这样可以防止出口到位于与调用方不同区域中的 PSTN 网关呼叫 PSTN 网关。
    
- 入局呼叫-基于位置的路由可以阻止传入的 PSTN 呼叫响铃如果路由传入呼叫 PSTN 网关不在同一区域为业务用户调用 Skype Skype 业务的终结点。
    
- 未知的区域的基于位置的路由限制传入和传出 PSTN 呼叫与位于未确定的位置 （如远程用户从 Internet 连接或未知地区位于） 的用户。
    
- 国际区域-强制实施基于位置的路由，路由的传出通过国际 PSTN 网关呼叫，如果找不到通往本地用户的位置。
    
## <a name="guidance-for-where-to-apply-location-based-routing"></a>应用基于位置的路由位置指南

用户的终结点网络站点位置或 PSTN 网关的网络站点位置，可根据具体情况的基于位置的路由。 本主题提供了如何基于位置的路由的指导应用。
  
### <a name="applying-location-based-routing-at-the-users-location"></a>应用基于位置的路由用户的位置

基于位置的路由的利用相同网络的区域、 站点和子网定义在 Skype 的企业服务器使用的 E9-1-1，CAC 和绕过媒体应用呼叫路由限制以防止 PSTN 电话绕过。 用户的位置取决于业务终结点从连接的用户的 Skype 的 IP 子网。 每个 IP 子网都与一个网络站点相关联，它们将聚合到由管理员确定的网络区域中。 基于位置的路由会强制实施基于用户的网络站点。
  
基于位置的路由规则应用于每个网络站点为基础，即，一组给定的规则将应用到启用基于位置的路由位于同一网络站点内的所有端点。 管理员可以将基于位置的路由应用于需要它的网络站点。
  
可以在每个网络站点上定义语音路由策略，以定义应由网络站点内的所有用户用于呼叫 PSTN 号码的特殊 PSTN 网关。 这种语音路由策略将优先于由用户的语音策略，当用户位于网络站点启用基于位置的路由，并且它将会阻止通过为启用其他 PSTN 网关呼叫路由的路由基于位置的路由。 为业务用户 Skype 发出 PSTN 呼叫，当用户的语音策略确定是否可以授权用户来打电话。 如果用户的语音策略允许用户打电话，基于位置的路由确定哪些的 PSTN 网关呼叫应从出口。 基于位置的路由作出这一判断基于用户的位置。
  
用户位置可以通过以下方式进行分类：
  
- 用户位于已知的网络站点启用基于位置的路由，他 （向内直拨） DID 号码终止上放置在同一个网络站点 (如 office) 的 PSTN 网关。 出站呼叫通过用户所在的网络站点的语音路由策略进行路由。 发往用户的传入 PSTN 呼叫将路由到与 PSTN 网关位于相同网络站点中的终结点。
    
- 用户所在的已知网络站点不同于 PSTN 网关所在的网络站点。（例如，用户前往另一公司办事处出差。）出站呼叫使用用户所在的网络站点的语音路由策略进行路由。发往用户的传入 PSTN 呼叫不会路由到所在站点与 PSTN 网关不同的终结点，从而防止 PSTN 收费绕路情形。
    
- 当用户位于未知的业务服务器部署 Skype 的网络站点时，出站呼叫路由将基于分配给与 PSTN 网关不能绑定到基于位置的路由限制用户的语音策略。 传入 PSTN 呼叫不会路由到位于未知网络站点内的终结点，从而防止 PSTN 收费绕路情形。 
    
### <a name="applying-location-based-routing-at-the-pstn-gateways-location"></a>应用基于位置的路由在 PSTN 网关的位置

通过 PSTN 网关呼叫路由和 Pbx 可能需要基于位置的路由限制，具体取决于此类系统的位置。 在级别在每个干线的基础上，可以启用基于位置的路由。
  
基于位置的路由引入了下面的一组规则在干线上启用：
  
- 在每个干线的基础上启用基于位置的路由时，规则将定义上的干线将只应用于通过该中继路由呼叫。
    
- 若要阻止在其中调用源自于不同的网络站点，PSTN 工具绕过 PSTN 网关所在的网络站点，基于位置的路由引入到给定主干网络站点关联。 这定义了允许呼叫路由到给定中继的网络站点。
    
可以为以下两种方式基于位置的路由启用中继：
  
- 为中继定义将呼叫发出到 PSTN 的 PSTN 网关。此类型的中继路由的传入呼叫只能路由到位于与中继相同的网络站点中的终结点。
    
- 主干被定义为对不出口求助电话与传统电话静态位置 （即 PBX 电话） 中的 PSTN 和服务用户的中介服务器等。 对于此特殊配置，此类型的中继路由的传入呼叫将视为来自与中继相同的网络站点。 来自 PBX 用户的呼叫将拥有位于同一主干网络站点中的业务用户的同一个基于位置的路由强制为 Skype。 如果两个 PBX 系统位于不同的网络站点连接通过 Skype 业务服务器，基于位置的路由将允许从一个网络站点中的一个 PBX 端点路由到另一个网络站点的另一个 PBX 终结点。 基于位置的路由不会阻止这种情况。 除了这种情况下，以类似的方式在同一位置的业务用户的 Skype 为终结点连接到此配置中介服务器的对等机器能够发出和接收呼叫与其他中介服务器的对等方未路由呼叫 to PSTN （即终结点连接到不同的 PBX） 无论中介服务器的对等是相关联的网络站点。 所有的入站的调用出站呼叫，呼叫转移并调用的转发均应涉及 PSTN 终结点位置基于路由使用仅 PSTN 网关定义为本地的这类中介服务器的对等。

### <a name="delegation"></a>委派

Skype 的业务委派功能受基于位置的路由方式如下：
  
- 启用代理的基于位置的路由位置代表经理的调用，使用委托的语音策略授权呼叫和代理的网站语音路由策略用于将电话转给
    
- 对于打给经理的传入 PSTN 呼叫，适用于呼叫转接或同时响铃的相同规则将按照呼叫转接和同时响铃主题所述进行应用。
    
- 委托将 PSTN 终结点设置为同时振铃目标，给经理，请来电时声音传入干线与相关联的站点的路由策略将用于路由到委托的 PSTN 终结点的调用。
    
- 为委派，它已建议通常位于同一个网络站点管理器以及其关联的委托。
    
## <a name="other-planning-considerations"></a>其他规划考虑事项

在规划基于位置的路由时，应考虑对以下情况的影响。
  
### <a name="disaster-recovery"></a>灾难恢复

从主池故障转移到备份池期间当到主池恢复正常运营，在所有基于位置的路由保持现行时间在灾难和恢复过程。
  
### <a name="survivable-branch-appliance"></a>Survivable Branch Appliance

配置基于位置的路由会影响您部署网关与高存活力分支装置相关联的计划。 与您 SBA 的网关必须位于同一个网络站点为您高存活力的分支装置;否则，用户驻留在您高存活力的分支装置则不允许进行出站呼叫，如果配置基于位置的路由。 按下您高存活力的分支装置和中心站点之间的 WAN 连接时，基于位置的路由限制仍然实施了参照完整性。

## <a name="capabilities-not-supported-by-location-based-routing"></a>基于位置的路由不支持的功能

基于位置的路由不适用于下列类型的交互。 基于位置的路由未强制执行业务终结点的 Skype 时与 PSTN 终结点使用这些功能进行交互。
  
- PSTN 电话拨入式会议
    
- 通过响应组的传入和传出 PSTN 呼叫
    
- 通过呼叫寄存对 PSTN 呼叫进行呼叫寄存或检索
    
- 到公告服务的传入 PSTN 呼叫
    
- 通过组内呼叫应答检索的传入 PSTN 呼叫
    
要实施的以下列表中的交互组件类型的基于位置的路由规则，则必须为会议启用基于位置的路由：
  
- PSTN 电话拨出式会议
    
- 从点对点音频对话升级到涉及 PSTN 终结点的音频会议 
    
- 涉及 PSTN 终结点的咨询转接
    
若要为会议启用基于位置的路由，请参见[会议的基于位置的路由](http://technet.microsoft.com/library/e1acb1ba-0ed2-4abf-8a7b-1ca3049e95e3.aspx)。
