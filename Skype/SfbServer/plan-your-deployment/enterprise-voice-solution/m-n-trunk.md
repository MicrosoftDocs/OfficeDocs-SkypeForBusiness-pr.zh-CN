# <a name="mn-trunk-in-skype-for-business-server-2015"></a>Skype for Business Server 2015 中的 M:N 中继
 
Skype 的业务服务器企业语音支持中介服务器和组件，如 PSTN 网关、 会话边框控制器和 IP PBX 之间的 M:N 中继。
  
Skype 业务服务器的呼叫路由而从以前版本的主干定义中支持更大的灵活性。 主干是网关并侦听的端口号与中介服务器的侦听端口号之间的逻辑关联。 这意味着几件事： 中介服务器可以有多个中继到相同的网关;中介服务器可以有多个中继到其他网关;相反一个网关可以有多个不同的中介服务器的中继。
  
您仍然需要创建根中继线只要向 adde 使用拓扑生成器拓扑的网关。 可以处理给定的中介服务器的网关的数目在繁忙高峰取决于服务器的处理能力。 如果[业务服务器 2015年的 Skype 的服务要求](../../plan-your-deployment/requirements-for-your-environment/server-requirements.md)，然后估计多少活动不可回避呼叫所述部署中介服务器超出了 Skype 业务服务器的最低硬件要求的硬件上独立的中介服务器可以处理约 1000年调用。 预计这些规范，中介服务器执行转换代码，但仍然能将多个网关呼叫路由即使不支持网关的硬件会议上部署时绕过媒体。
  
在定义的呼叫路由时，您指定与该路由关联的中继，但未指定与该路由关联的中介服务器。 相反，您可以使用拓扑生成器将中继与中介服务器相关联。 换句话说，路由确定要用于调用的干线，随后，该干线与关联的中介服务器发送该调用的信号。
  
中介服务器可以部署为一个池;该池可以搭配与前端池，或者它可以作为一个独立的池部署。 当中介服务器搭配与前端池时，池的大小最多只能包含 12 （注册池的大小限制）。 总之，这些新功能提高的可靠性和的部署灵活性对于中介服务器，但它们需要下面的对等实体中的相关的功能：
  
- **PSTN 网关。** Skype 业务服务器限定网关必须实现 DNS 负载平衡，从而可以跨池充当中介服务器的从而为负载平衡调用一个池的负载平衡器的限定公共交换的电话网络 (PSTN) 网关.
    
- **会话边界控制器。** 对于 SIP 中继，对等实体是 Internet 电话服务提供商的会话边界控制器 (SBC)。 对 SBC 的方向从中介服务器池，SBC 可以接收来自池中任何中介服务器的连接。 从 SBC 池的方向，流量可以发给池中任何中介服务器。 实现此功能的一个方法是通过 DNS 负载平衡（如果服务提供商和 SBC 支持）。 另一种方法是为池中的所有中介服务器的 IP 地址的服务提供商和服务提供商将其 SBC 在作为单独的 SIP 中继为配置每个中介服务器。 然后，服务提供商将处理自己的服务器的负载平衡。 并非所有服务提供商或 SBC 都可以支持这些功能。 此外，服务提供商可能会对此功能收取额外费用。 通常，每个到 SBC 的 SIP 中继按月收费。
    
- **IP-PBX。** 在 SIP IP PBX 中止从中介服务器池方向，IP PBX 可以接收来自池中任何中介服务器的连接。 从 IP PBX 池的方向，流量可以发给池中任何中介服务器。 由于大多数 IP Pbx 不支持 DNS 负载平衡，因此建议个别直接 SIP 连接从 IP PBX 定义为池中的每个中介服务器。 然后，IP-PBX 将通过向中继组分发流量处理自己的负载平衡。 假定中继组在 IP-PBX 中具有一组一致的路由规则。 是否某个特定 IP PBX 支持此中继组概念以及如何与 IP-PBX 的自己冗余相交，它和群集体系结构需要确定之前，您可以决定是否中介服务器群集可以与 IP PBX 正确交互。
    
中介服务器池必须具有与它交互的对等网关的统一视图。 这意味着池中的所有成员都能从配置存储访问对等网关的同一定义，并且都可能针对传出呼叫与其进行交互。 因此，是没有办法分段池，以便与只拨出呼叫特定网关等的一些中介服务器通信。 如果需要此类分段，则必须使用单独的中介服务器池。 例如，如果与池进行交互的 PSTN 网关、SIP 中继或 IP-PBX 中的相关功能（如本主题中前面所述）不存在，则此情况适用。
  
特定的 PSTN 网关，IP PBX 或 SIP 中继等可路由到多个中介服务器或中继。 网关，可以控制特定池的中介服务器的数目取决于使用媒体回避的调用次数。 如果大量的调用使用媒体回避，池中中介服务器可以处理许多发出更多调用，因为只有信号传输层处理是必要的。
