# <a name="plan-your-enterprise-voice-solution-in-skype-for-business-server-2015"></a>规划业务服务器 2015 Skype 在企业语音解决方案
 
**摘要：**了解有关规划业务服务器 2015 Skype 在一个统一的声音和通信解决方案的选项。
  
Skype 业务服务器提供内部部署两个企业语音选项 — — 企业语音，调用通过工作 — — 为您与您的业务服务器部署 Skype 集成您的组织的电话系统，使真正的统一通信解决方案。 这两个选项允许用户使用其 Skype 业务客户端来与其他用户，您的组织内外启动语音呼叫。
  
您还可以利用 Microsoft 云 PBX 服务，包括通过连接到由 Skype 的在线业务提供服务的内部电话基础架构的 PBX 服务和 PSTN 呼叫。 有关详细信息，请参阅[规划 Office 365 (云 PBX) 解决方案中的电话系统](../../skype-for-business-hybrid-solutions/plan-your-phone-system-cloud-pbx-solution/plan-your-phone-system-cloud-pbx-solution.md)。
  
您还可以使用企业语音和调用通过工作在一起。
  
- 企业语音是最完整的语音产品。 它是一套基于软件的 IP 语音 (VoIP) 解决方案，提供了丰富的功能集，包括与 Outlook 和 Exchange 的丰富集成以及许多强大的功能，比如响应组、呼叫寄存、团队呼叫、组内呼叫应答和增强型紧急 E9-1-1 支持。 通过企业语音，用户使用音频设备，例如使用他们的计算机，或启用 VoIP 的电话，而不是传统的 PSTN 或 PBX 电话耳机。
    
- 调用通过工作提供了可与您现有的 PBX 电话系统集成您的 Skype 业务解决方案的方法。 启用呼叫通过工作的用户可以单击中业务调用另一个用户，或者在您的部署或外部用户的 Skype。 呼叫使用用户的 PBX 电话完成。 此外，这些用户可以单击 Skype 为参加会议的业务中。
    
    部署的工作通过调用也是用户仍有 PBX 电话，但都被转换为企业语音的优势因为它使这些用户正在使用其 Skype 业务客户端来控制他们的电话联络。
    
     调用通过工作提供一组更有限的语音功能 thanEnterprise 的声音。 例如，它不支持委派、 团队调用、 响应组或增强 E9-1-1。
    
当然，有一些您在其他用户正使用 PBX 电话时启用企业语音的用户。 此外，虽然过渡到完全的企业语音解决方案时，您可以启用仍具有企业语音 PBX 电话的用户。 这些用户可以使用 PBX 电话在他们的办公桌前时，并使用企业语音来发送或接收传呼使用在其他位置的 VoIP 设备。 如果调用通过工作启用了这些用户，他们可以使用其 Skype 业务客户端来控制他们的 PBX 电话时在他们的办公桌前。
  
> [!NOTE]
> 远程呼叫控制了 Lync 服务器，从而使用户能够使用业务客户端为其 Skype 拨打和接听电话的 PBX 电话上的早期版本中提供的功能。 控件不支持用户的远程调用业务服务器服务器驻留在 Skype 上，但支持用户使用 Skype 业务客户端仍驻留在服务器运行 Lync Server 2013。
