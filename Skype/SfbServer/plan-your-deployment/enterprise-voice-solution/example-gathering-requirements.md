# <a name="example-gathering-requirements-for-call-admission-control-in-skype-for-business-server-2015"></a>示例：在 Skype for Business Server 2015 中收集呼叫允许控制要求
 
提供详细的示例，在 Skype 呼叫许可控制规划的业务服务器企业语音，包括收集有关您的网络站点、 地区，以及带宽的信息。
  
此示例显示如何规划和实现呼叫允许控制 (CAC)。总体来说，包括以下活动：
  
1. 确定所有您的网络集线器和主干线 （也称为网络区域）。
    
2. 标识业务服务器将每个网络区域管理 CAC 的中心站点的 Skype。
    
3. 识别和定义连接到每个网络区域的网络站点。
    
4. 每个网络站点到 WAN 的连接是带宽受限，描述带宽容量的 WAN 连接和带宽限制的网络管理员已经为 Skype 设置对于媒体业务服务器的通信，有可能的话。 不需要包括与 WAN 的连接不受带宽限制的站点。
    
5. 将网络中的每个子网与一个网络站点相关联。
    
6. 映射网络区域之间的链路。 对于每个链接，描述了其带宽容量和网络管理员已放置在 Skype 媒体通信业务服务器的任何限制。
    
7. 定义每对网络区域之间的路由。
    
## <a name="gather-the-required-information"></a>收集所需信息

要准备呼叫允许控制，请收集以下步骤中描述的信息：
  
1. 标识网络区域。网络区域代表网络中枢或网络中心。 
    
    网络中枢或网络中心是计算机网络基础结构的一部分，它将网络的各个部分相互连接起来，提供在不同 LAN 或子网之间交换信息的路径。网络中枢可以将各种网络关联在一起，适用范围从较小区域到广阔的地理区域。网络中枢的容量通常大于与其连接的网络的容量。
    
    示例拓扑具有三个网络区域：北美、EMEA 和 APAC。网络区域包含网络站点的集合。与网络管理员合作以定义企业的网络区域。
    
2. 标识每个网络区域相关联的中心站点。 中心站点包含至少一个前端服务器，将所有通过网络区域的 WAN 连接的媒体通信管理 CAC 的业务服务器部署 Skype。
    
   **示例企业网络划分为三个网络区域**

     ![包含 3 个网络区域的网络拓扑的示例](../../media/Plan_CS_VoiceCAC_example3networkregions.jpg)
  
    > [!NOTE]
    > 多协议标签交换 (MPLS) 网络应代表一个网络区域，在该网络区域中，每个物理位置都具有一个相应的网络站点。 有关详细信息，请参阅规划文档中的"[组件和拓扑结构的用于调用中业务 2015年的 Skype 的许可控制](components-and-topologies.md)"主题。 
  
    在前面的示例网络拓扑中，有三个网络区域，每个都有 Skype 业务服务器管理 CAC 的中心站点。 按地理距离选择网络区域相应的中央站点。 由于网络区域内的媒体流量最多，按地理距离选择中央站点使网络区域能够独立运行，因此即使其他中央站点不可用，网络区域也可以继续正常工作。 
    
    在此示例中，名为芝加哥的业务部署 Skype 是北美地区的中心站点。
    
    所有在北美的业务用户的 Skype 被驻留在芝加哥部署中的服务器上。 下表显示了所有三个网络区域的中央站点。
    
    **网络区域和其相关的中心站点**

    |北美  <br/> |芝加哥  <br/> |
    |EMEA  <br/> |伦敦  <br/> |

    > [!NOTE]
    > 这取决于您的业务服务器拓扑结构的 Skype，同一个中心站点可以分配给网络中的多个区域。 
  
3. 对于每个网络区域，标识其 WAN 连接不受带宽限制的所有网络站点（办公室或位置）。由于这些站点不受带宽限制，因此无需对其应用 CAC 带宽策略。
    
    在下表显示的示例中，有三个网络站点没有受带宽限制的 WAN 链路：纽约、芝加哥和底特律。
    
   **不通过 WAN 带宽有限的网络站点**

   |**网络站点**|**网络区域**|
   |:-----|:-----|
   |纽约  <br/> |北美  <br/> |
   |芝加哥  <br/> |北美  <br/> |
   |底特律  <br/> |北美  <br/> |
   
4. 对于每个网络区域，标识所有通过受带宽限制的 WAN 链路连接到网络区域的网络站点。
    
    为了帮助确保音频和视频质量，我们建议这些受带宽限制的网络站点监控其 WAN，并拥有限制流入和流出网络区域的媒体（音频或视频）流量的 CAC 带宽策略。
    
    在下表显示的示例中，有三个受 WAN 带宽限制的网络站点：波特兰、里诺和阿尔伯克基。

5. 对于每个受带宽限制的 WAN 链路，需确定以下事项：
    
  - 要为所有并发音频会话设置的总体带宽限制。 如果新的音频会话会导致超出此限制，Skype 业务服务器不允许启动该会话。
    
  - 要为每个单独的音频会话设置的带宽限制。默认 CAC 带宽限制是 175 kbps，但是管理员可修改该值。
    
  - 要为所有并发视频会话设置的总体带宽限制。 如果新的视频会话会导致超出此限制，Skype 业务服务器不允许启动该会话。
    
  - 要为每个单独的视频会话设置的带宽限制。默认 CAC 带宽限制是 700 kbps，但是管理员可修改该值。

6. 对于网络中的每个子网，指定其关联的网络站点。
    
    > [!IMPORTANT]
    > 即使网络站点不受带宽限制，网络中的每个子网也必须与一个网络站点相关联。这是因为呼叫允许控制使用子网信息来确定终结点所在的网络站点。确定会话双方的位置后，呼叫允许控制可以确定是否有足够的带宽来建立呼叫。当通过没有带宽限制的链路建立会话时，会生成警报。 
  
    > [!IMPORTANT]
    > 如果部署音频/视频边缘服务器，则每台边缘服务器的公共 IP 地址都必须与部署边缘服务器的网络站点关联。 必须将 A/V 边缘服务器的每个公共 IP 地址作为子网掩码为 32 的子网添加到网络配置设置中。 例如，如果在芝加哥部署 A/V 边缘服务器，则为这些服务器的每个外部 IP 地址创建一个子网掩码为 32 的子网，并将网络站点芝加哥与这些子网相关联。 为公用 IP 地址的详细信息，请参阅[规划业务 2015年的 Skype 的网络需求](../../plan-your-deployment/network-requirements/network-requirements.md)。

    **原因**：网络配置设置中缺少相应 IP 地址的子网，或子网未与网络站点相关联。 
    
    **解决方案**：将与上述 IP 地址列表对应的子网添加到网络配置设置中，并将每个子网关联到网络站点。

### <a name="next-steps"></a>后续步骤

收集到所需的信息后，您可以执行由使用 Skype 的业务服务器管理外壳或 Skype 业务服务器控件面板的 CAC 部署。
  
> [!NOTE]
> 虽然通过 Skype 业务服务器的控制面板，您可以执行大多数网络配置任务，创建子网和站点间的链接，您必须使用 Skype 业务服务器管理程序。 有关详细信息，请参阅[新建 CsNetworkSubnet](https://docs.microsoft.com/powershell/module/skype/new-csnetworksubnet?view=skype-ps)和[新建 CsNetworkInterSitePolicy](https://docs.microsoft.com/powershell/module/skype/new-csnetworkintersitepolicy?view=skype-ps)。
