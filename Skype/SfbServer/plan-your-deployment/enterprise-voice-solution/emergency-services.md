# <a name="plan-for-emergency-services-in-skype-for-business-server-2015"></a>紧急服务 Skype 业务服务器 2015年计划
 
了解有关在 Skype 的增强 9-1-1 (E9-1-1) 服务业务企业语音，包括位置购买并进行呼叫路由。
  
Skype 业务服务器支持美国境内企业语音部署过程中增强 9-1-1 (E9-1-1) 服务。 E9-1-1 是一项紧急派发功能，对于来自办公楼和其他多租户设施的呼叫，该功能可将 9-1-1 呼叫与紧急响应位置 (ERL) 相关联，紧急响应位置包含市政（即街道）地址和其他更具体的位置信息，例如楼层号。 通过使用提供的 ERL，公共安全应答点 (PSAP) 可以立即向需要帮助的呼叫者派遣第一批响应者，从而降低因疏忽而将响应者指向不正确或模糊不清的位置的风险。
  
> [!NOTE]
> Skype 业务服务器现在支持的客户端配置的多个紧急号码。 详细信息请参阅[规划业务服务器 2015年的 Skype 在多个紧急号码](multiple-emergency-numbers.md)。 
  
> [!NOTE]
> Skype 业务服务器有三个高级的企业语音特征： 呼叫许可控制、 紧急服务 (E9-1-1)，以及媒体回避。 规划对这些功能的所有三个通用的信息的概述，请参阅[高级的企业语音功能，在业务服务器 2015年的 Skype 的网络设置](network-settings-for-advanced-features.md)。 
  
Skype 业务服务器支持增强 9-1-1 (E9-1-1) 调用从 Skype 业务客户和 Lync 电话版设备。 当您将 Skype 业务服务器配置 E9-1-1、 紧急呼叫从 Skype 业务放或 Lync 电话版包括紧急响应的位置 (ERL) 位置信息服务数据库中的信息。 ERL 包含市政（即街道）地址，以及有助于确定在办公楼和其他多租户设施中的更精确位置的其他信息。 当用户发出紧急呼叫时，Skype 业务服务器的路由呼叫音频，以及位置和回调的信息，通过中介服务器到 E9-1-1 的服务提供商。 E9-1-1 服务提供商会使用呼叫者的市政地址，将呼叫路由到为呼叫者所在的位置提供服务的公共安全应答点 (PSAP)，并沿着 PSAP 用来查找呼叫者 ERL 的紧急服务查询键 (ESQK) 进行发送。 
  
Skype 业务服务器支持路由的紧急呼叫，E9-1-1 的服务提供商的两种方法：
  
- 到合格的 E9-1-1 服务提供商的会话初始协议 (SIP) 中继连接
    
- 到基于公用电话交换网 (PSTN) 的 E9-1-1 服务提供商的紧急位置标识号 (ELIN) 网关
    
当您使用一个 SIP 中继 E9-1-1 服务提供商时，向位置信息服务数据库，添加 ERLs，然后验证针对维护 E9-1-1 的服务提供商的主街道地址指南 (MSAG) 的位置。 如果 E9-1-1 的服务提供商联系接收的调用中没有位置信息，或者有针对 MSAG 尚未经过验证的位置，E9-1-1 的服务提供商发送国家/地区紧急调用响应中心 (ECRC)，这是对的调用配备专门培训人员口头获取调用方的位置，如果可能，并手动将电话转给适当 PSAP 到。 （某些 SIP 中继 E9-1-1 服务提供商还提供与 PSTN 的客户直接向内拨号 （未） 号为 ECRC，它提供了路由 9-1-1 的替代方法调用时，如果由于某种原因失败的 SIP 中继。
  
与时分复用 (TDM) 和基于 IP 的专用分组交换机 (PBX) 的电话，其中有固定位置，Skype 业务终结点可以是很灵活。 部署 E9-1-1 功能时，可以帮助确保，无论调用方位于何处，可以为紧急呼叫路由到提供呼叫者的位置 PSAP Skype 业务服务器。 例如，如果用户的总部位于华盛顿州雷蒙德市，但该用户将从计算机紧急调用放在位于 Wichita，堪萨斯，SIP 中继的分支办公室或基于 PSTN 的 E9-1-1 的服务提供商将路由到在 Wichita PSAP 呼叫在雷蒙德 PSAP 不适。
  
ELIN 网关使用时，还将 ERLs 添加到位置信息服务数据库，但您还包括 ELIN 编号，为每个位置。 ELIN 号码会在紧急呼叫过程中变为紧急呼叫号码。 然后，你必须确保 PSTN 运营商将 ELIN 上载到自动位置标识 (ALI) 数据库。 
  
> [!NOTE]
> Skype 业务连接模拟设备不能接收来自该位置信息服务位置信息或传输到 E9-1-1 的服务提供商的位置。 
  
 如果使用 SIP 中继 E9-1-1 服务提供商选项并需要通过模拟电话支持 E9-1-1，则有以下两个选项：
  
- **传统的 PS 阿里选项**如果您在每个站点中，模拟电话进行部署，每个模拟电话必须 DID 有本地的 PSTN 网关，可以设置专用开关/自动位置标识 （PS-阿里） 服务提供商直接模拟设备的位置。 在这种情况下，您配置特制的 Skype 业务语音策略并将它们分配到模拟设备的联系人对象，以便从这些电话 E9-1-1 调用路由直接通过本地网关到服务站点 （而是 PSTN 供应商将呼叫传送至 E9-1-1 的服务提供者的 SIP 中继)。 发出紧急呼叫后，与 PSTN 中继关联的 PS-ALI 提供商的数据库会将每个模拟电话的 DID 映射到一个物理位置并将此位置提供给 PSAP。 每当将电话移至不同的 ERL 时，必须通过 PS-ALI 服务提供商更新这些记录。
    
- **E9-1-1 的服务提供程序选项**您可以模拟电话 DIDs 和其相应 ERLs E9-1-1 的服务提供程序，如果注册 E9-1-1 的服务提供程序支持此功能。 如果提供程序接收调用从 Skype 业务服务器不包含 PIDF 的 LO 数据，提供程序可以看到是否在调用方的 DID 号码匹配的数据库。 通过使用自动检索其数据库，提供程序可以从 ERL 路由到正确的 PSAP 和 PSAP 紧急呼叫将收到 DID 的模拟设备和 ESQK 记录，允许调度程序，以便查找调用方的位置。
    
如果您使用 ELIN 网关选项并需要通过模拟电话支持 E9-1-1，则可以直接向 PS-ALI 服务提供商提供模拟设备的位置，如上面第一个选项中所述。
  
从业务服务器的角度为 Skype，E9-1-1 的过程可以分为两个阶段：
  
- 阶段 1：获取位置
    
- 阶段 2：将紧急呼叫路由至 E9-1-1 服务提供商
    
本节介绍这些阶段的工作原理。
  
如果计划将基础结构配置为自动检测客户端位置，则首先需要确定将使用哪些网络元素将呼叫者映射到不同位置。 可能的选项的详细信息，请参阅[的定义网络元素用于确定在业务服务器 2015年的 Skype 的位置](network-location.md)。 
  
## <a name="acquiring-a-location"></a>获取位置

业务服务器 E9-1-1 部署 Skype，在业务或 Lync 电话版本客户端的每个内部连接 Skype 积极将获得自己的位置。 SIP 注册后客户端让它知道有关本身它位置请求中的位置信息服务，这是由复制的 SQL Server 数据库支持的 web 服务的所有网络连接信息。 中心站点的每个池都有一个位置信息服务，它使用的网络信息查询匹配位置的记录。 如果没有匹配项，位置信息服务将返回到客户端的位置。 如果没有匹配项，则可能会提示用户手动输入位置（取决于位置策略设置）。 位置数据将以 Internet 工程任务组 (IETF) 标准化 XML 格式（称为状态信息数据格式位置对象 (PIDF-LO)）传回客户端。
  
业务客户端的 Skype 的紧急呼叫的一部分包括 PIDF-LO 数据和 E9-1-1 的服务提供商使用这些数据来确定适当的 PSAP 并路由到正确的 ESQK，以及该 PSAP 呼叫允许到 PSAP 调度程序获取调用方的位置。
  
下图显示了 Skype 业务客户机如何获得 （除第三方客户端 MAC 地址基于位置方法） 的位置：
  
![客户端获取位置的方式的图示](../../media/Plan_LyncServer_E911_LocationAcquisition.jpg)
  
对于获取位置的客户端，必须执行下列步骤：
  
1. 管理员使用填充位置信息服务数据库与网络 wiremap （将各种类型的网络地址映射到相应的紧急响应位置 (ERLs) 的表）。
    
2. 如果使用 SIP 中继 E9-1-1 服务提供商，则管理员将针对 E9-1-1 服务提供商维护的主街道地址指南 (MSAG) 数据库验证 ERL 的市政地址部分。如果使用 ELIN 网关，则管理员将确保 PSTN 运营商会将 ELIN 上载到自动位置标识 (ALI) 数据库。
    
3. 在注册过程中或当出现网络更改，内部连接的客户端发送一个包含客户端的位置请求的发现到的位置信息服务的网络地址。
    
4. 位置信息服务查询位置，其发布的记录，并如果找到匹配项，则返回 ERL PIDF-LO 格式中的客户端。
    
## <a name="routing-e9-1-1-calls-using-a-sip-trunk"></a>使用 SIP 中继路由 E9-1-1 呼叫

使用 SIP 中继连接至限定的 E9-1-1 服务提供商是可用于部署 E9-1-1 的一种方式。 有关使用 ELIN 网关连接到公用交换的电话网 (PSTN) 的详细信息-基于 E9-1-1 服务提供商，请参阅[使用 ELIN 网关路由 E9-1-1 的呼叫](http://technet.microsoft.com/library/5a3997e3-898d-49cb-922a-4184c3373350.aspx)。
  
下图显示如何紧急呼叫被路由从 Skype 业务服务器到公共安全应答点 (PSAP) 当您使用一个 SIP 中继和限定的 E9-1-1 的服务提供商。
  
**通过 SIP 中继路由 E9-1-1 调用**

![从 Lync Server 路由到 PSAP 的紧急呼叫](../../media/Plan_LyncServer_E911_CallRouting.jpg)
  
当紧急呼叫从兼容 Skype 业务服务器的客户机：
  
1. SIP 邀请包含位置、 调用方的回拨号码和 （可选） 通知 URL 和大会回拨号码被路由到 Skype 业务服务器。
    
2. Skype 业务服务器的紧急号相匹配，并通过 SIP 中继到 E9-1-1 的服务提供商到中介服务器，并从那里，路由调用 （基于**PSTN 使用**该值适用位置策略中定义）。
    
3. E9-1-1 服务提供商根据呼叫提供的位置将紧急呼叫路由至正确的 PSAP。如果客户端随紧急呼叫提供了已验证的紧急响应位置 (ERL)，则提供商会自动将呼叫路由至相应的 PSAP。如果位置是由用户手动输入的，则在将紧急呼叫路由至 PSAP 之前，紧急呼叫响应中心 (ECRC) 将首先口头与呼叫者核实位置的准确性。
    
4. 如果配置为通知的位置策略时，一个或多个组织的安全官员将发送特殊 Skype 业务紧急通知和即时消息。 此消息始终安全官员屏幕上弹出并包含调用方的名称、 电话号码、 时间和位置，使安全人员能够快速响应紧急调用方通过使用即时消息或语音。
    
5. 如果您配置了用于会议的位置策略并且 E9-1-1 服务提供商支持该策略，则国际安全服务台会通过单向音频或双向音频作为与会者加入呼叫。
    
6. 如果过早中断呼叫，则 PSAP 使用回拨号码直接与呼叫者联系。
    
## <a name="routing-e9-1-1-calls-by-using-an-elin-gateway"></a>使用 ELIN 网关路由 E9-1-1 呼叫

统一通信开放互操作性计划中的某些合作伙伴提供了合格的支持紧急位置标识号 (ELIN) 的网关，该网关可充当合格 E9-1-1 服务提供商的 SIP 中继连接的备选项。 ELIN 网关支持 ISDN 或与基于公用电话交换网 (PSTN) 的 E9-1-1 服务的集中式自动通话记帐 (CAMA) 连接。 有关合作伙伴提供 ELIN 网关和它们的文档链接的详细信息，请参阅[Microsoft Lync 的合格的基础结构](https://go.microsoft.com/fwlink/p/?LinkId=248425)和[业务的 Skype 的电话基础架构](https://technet.microsoft.com/en-us/office/dn947483)。 
  
与 E9-1-1 服务提供商的 SIP 中继连接相似，ELIN 网关还提供了将紧急呼叫路由到呼叫者的最合适的公共安全应答点 (PSAP) 的方法，但这些网关将 ELIN 用作位置标识符。 在您的组织定义 ELINs 每个紧急响应的位置 (ERL) （有关详细信息，请参阅[管理业务服务器 2015年的 Skype 在 ELIN 网关的位置](elin-gateways.md)）。 
  
ELIN 网关使用的紧急电话时，可以使用同一个 Skype 业务服务器 E9-1-1 基础结构时所使用的 SIP 中继连接。 这就是说，位置信息服务数据库位置为 Skype 业务为客户提供，而位置策略启用功能并定义的路由。 与 ELIN 网关，但是，您需要将 ELINs 添加到位置信息服务数据库，并让您的 PSTN 运营商将它们上载到自动位置标识 （阿里） 数据库。
  
当 Skype 业务客户端从位置信息服务中获取其位置时，该位置将包括 ELIN。 在紧急呼叫期间，ELIN 将包含在发送到 ELIN 网关的位置中。 ELIN 网关会将此呼叫标识为紧急呼叫并将呼叫者的号码与 ELIN 交换。 ELIN 网关随后会将呼叫路由到带有作为主叫号码的 ELIN 的 PSTN。 PSTN E9-1-1 提供商将在 ALI 数据库中查找 ELIN，该数据库是主街道地址指南 (MSAG) 数据库的附带数据库。 PSTN 随后会根据 ALI 查找将呼叫发送到最合适的 PSAP，而 PSAP 会根据 ALI 查找将第一个响应者发送到呼叫者的位置。 主叫号码将在 ELIN 网关上缓存一段预定时间以供回拨。 在回拨期间，PSAP 将到达 ELIN 网关，后者会将 ELIN 与呼叫者的直拨分机 (DID) 号码交换。
  
ELIN 网关仅支持来自您组织的网络中的紧急呼叫。它们不支持从您的网络外发出的紧急呼叫。
  
> [!NOTE]
> 有关使用紧急呼叫的 SIP 中继连接的详细信息，请参阅[使用 SIP 中继路由 E9-1-1 的呼叫](http://technet.microsoft.com/library/157753c3-fe74-4e2c-81da-ee06911d4cc2.aspx)。 
  
下图显示如何紧急呼叫被路由从 Skype 业务服务器到 PSAP 时使用 ELIN 网关。
  
**与 ELIN 网关的路由 E9-1-1 调用**

![显示紧急服务呼叫如何经由中介服务器，随后传输到紧急服务提供商。在此之后，可以选择通过 IM 发送给现场安全部门和/或回拨原始呼叫方。](../../media/Plan_LyncServer_E911_ELINCallRouting.jpg)
  
1. SIP 邀请包含位置、 调用方的回拨号码和 （可选） 通知 URL 和大会回拨号码被路由到 Skype 业务服务器。
    
2. Skype 业务服务器的紧急号相匹配，然后将 （基于**PSTN 使用**值适用位置策略中定义） 的调用路由到中介服务器，并从那里到 ELIN 网关。
    
3. ELIN 网关通过 ISDN 或 CAMA 中继将呼叫路由到 PSTN。
    
4. PSTN 将呼叫标识为紧急呼叫并将其路由到网路中的 E9-1-1 选择性路由器。E9-1-1 选择性路由器在 ALI 数据库中查找呼叫者的号码以获取地理位置。E9-1-1 选择性路由器根据从 ALI 数据库中检索到的位置信息将呼叫发送到最合适的 PSAP。  
    
5. 如果配置为通知的位置策略时，一个或多个组织的安全官员将发送特殊 Skype 业务紧急通知和即时消息。 此消息始终安全官员屏幕上弹出并包含调用方的名称、 电话号码、 时间和位置，使安全人员能够快速响应紧急调用方通过使用即时消息或语音。
    
6. 如果呼叫被永久性中断，PSAP 将使用 ELIN 直接联系呼叫者。ELIN 网关会将 ELIN 与呼叫者的 DID 交换。
