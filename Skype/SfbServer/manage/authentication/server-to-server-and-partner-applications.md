# <a name="manage-server-to-server-authentication-oauth-and-partner-applications-in-skype-for-business-server"></a><span data-ttu-id="98a39-103">在 Skype for Business Server (OAuth) 和合作伙伴应用程序管理服务器到服务器身份验证</span><span class="sxs-lookup"><span data-stu-id="98a39-103">Manage server-to-server authentication (OAuth) and partner applications in Skype for Business Server</span></span>
 
<span data-ttu-id="98a39-104">**摘要：** 在 Skype for Business Server 中管理 OAuth 和合作伙伴应用程序。</span><span class="sxs-lookup"><span data-stu-id="98a39-104">**Summary:** Manage OAuth and partner applications in Skype for Business Server.</span></span>
  
<span data-ttu-id="98a39-105">Skype for Business Server 必须能够安全且无缝地与其他应用程序和服务器产品进行通信。</span><span class="sxs-lookup"><span data-stu-id="98a39-105">Skype for Business Server must be able to securely, and seamlessly, communicate with other applications and server products.</span></span> <span data-ttu-id="98a39-106">例如，可以配置 Skype for Business Server，以便联系人数据和/或存档数据存储在 Microsoft Exchange Server 2013 中;但是，只有当 Skype for Business Server 和 Exchange 能够安全地相互通信时，才能完成此操作。</span><span class="sxs-lookup"><span data-stu-id="98a39-106">For example, you can configure Skype for Business Server so that contact data and/or archiving data is stored in Microsoft Exchange Server 2013; however, this can only be done if Skype for Business Server and Exchange are able to securely communicate with one another.</span></span> <span data-ttu-id="98a39-107">同样，可以从 Office Web Apps Server 中安排 Skype for Business Server 会议;同样，只有在 SharePoint 和 Skype for Business Server (服务器相互信任) 才能完成此操作。</span><span class="sxs-lookup"><span data-stu-id="98a39-107">Likewise, you can schedule a Skype for Business Server conference from within Office Web Apps Server; again, this can only be done if the two servers (SharePoint and Skype for Business Server) trust one another.</span></span> <span data-ttu-id="98a39-108">虽然可以使用一种身份验证机制在 Skype for Business Server 和 Exchange 之间通信，但使用一个单独的机制用于 Skype for Business Server 和 SharePoint 通信，但更好、更有效的方法是使用标准化方法进行所有服务器到服务器身份验证和授权。</span><span class="sxs-lookup"><span data-stu-id="98a39-108">Although it's possible to use one authentication mechanism for communication between Skype for Business Server and Exchange but a separate mechanism for Skype for Business Server and SharePoint communication, a better and more efficient approach is to use a standardized method for all server-to-server authentication and authorization.</span></span>
  
<span data-ttu-id="98a39-109">使用单一的标准化方法进行服务器到服务器身份验证是 Skype for Business Server 采用的方法。</span><span class="sxs-lookup"><span data-stu-id="98a39-109">Using a single, standardized method for server-to-server authentication is the approach taken by Skype for Business Server.</span></span> <span data-ttu-id="98a39-110">从 Office Server 2013 版本开始，Skype for Business Server (以及其他 Microsoft Server 产品（包括 Exchange Server 和 SharePoint Server) ）支持用于服务器到服务器身份验证和授权的 OAuth (Open Authorization) 协议。</span><span class="sxs-lookup"><span data-stu-id="98a39-110">Started with Office Servers 2013 release, Skype for Business Server (as well as other Microsoft Server products, including Exchange Server and SharePoint Server) supported the OAuth (Open Authorization) protocol for server-to-server authentication and authorization.</span></span> <span data-ttu-id="98a39-111">使用 OAuth，这是一种由许多主要网站使用的标准授权协议，用户凭据和密码不会从一台计算机传递到另一台计算机。</span><span class="sxs-lookup"><span data-stu-id="98a39-111">With OAuth, a standard authorization protocol used by a number of major websites, user credentials and passwords are not passed from one computer to another.</span></span> <span data-ttu-id="98a39-112">但是，身份验证和授权是基于安全令牌的交换；这些令牌会将访问权限授予特定时间量的一组特定资源。</span><span class="sxs-lookup"><span data-stu-id="98a39-112">Instead, authentication and authorization is based on the exchange of security tokens; these tokens grant access to a specific set of resources for a specific amount of time.</span></span>
  
<span data-ttu-id="98a39-113">OAuth 身份验证通常涉及到三方：一台授权服务器和两个需要相互通信的领域。</span><span class="sxs-lookup"><span data-stu-id="98a39-113">OAuth authentication typically involves three parties: a single authorization server and the two realms that need to communicate with one another.</span></span> <span data-ttu-id="98a39-114"> (您还可以使用授权服务器执行服务器到服务器身份验证，本文档稍后将讨论此过程。) 安全令牌由授权服务器 (也称为安全令牌服务器) 颁发给需要通信的两个领域;这些令牌验证源自一个领域的通信应受另一领域信任。</span><span class="sxs-lookup"><span data-stu-id="98a39-114">(You can also do server-to-server authentication without using an authorization server, a process that will be discussed later in this document.) > [!NOTE]
> <span data-ttu-id="98a39-116">领域只是一个安全容器。</span><span class="sxs-lookup"><span data-stu-id="98a39-116">A realm is simply a security container.</span></span> <span data-ttu-id="98a39-117">默认情况下，Skype for Business Server 将默认 SIP 域用作其 OAuth 领域。</span><span class="sxs-lookup"><span data-stu-id="98a39-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="98a39-118">其他 SIP 命名空间将添加到 OAuth 证书中的"主题备用名称"列表中。</span><span class="sxs-lookup"><span data-stu-id="98a39-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="98a39-119">Skype for Business Server 支持三种服务器到服务器身份验证方案。</span><span class="sxs-lookup"><span data-stu-id="98a39-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="98a39-120">使用 Skype for Business Server，你可以：</span><span class="sxs-lookup"><span data-stu-id="98a39-120">With Skype for Business Server, you can:</span></span>
  
- <span data-ttu-id="98a39-121">在 Skype for Business Server 本地安装和 Exchange 和/或 SharePoint Server 本地安装之间配置服务器到服务器身份验证。</span><span class="sxs-lookup"><span data-stu-id="98a39-121">Configure server-to-server authentication between an on-premises installation of Skype for Business Server and an on-premises installation of Exchange and/or SharePoint Server.</span></span>
    
- <span data-ttu-id="98a39-122">在一对 Microsoft 365 或 Office 365 组件之间配置服务器到服务器身份验证 (例如，Microsoft Exchange Server 和 Skype for Business Server 之间，或者 Skype for Business Server 和 SharePoint) 。</span><span class="sxs-lookup"><span data-stu-id="98a39-122">Configure server-to-server authentication between a pair of Microsoft 365 or Office 365 components (for example, between Microsoft Exchange Server and Skype for Business Server, or between Skype for Business Server and SharePoint).</span></span>
    
- <span data-ttu-id="98a39-123">在跨界环境中配置服务器到服务器身份验证 (即本地服务器与 Microsoft 365 或 Office 365 组件之间的服务器到服务器身份验证) 。</span><span class="sxs-lookup"><span data-stu-id="98a39-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and a Microsoft 365 or Office 365 component).</span></span>
    
<span data-ttu-id="98a39-124">请注意，此时仅 Exchange 2013、SharePoint Server、Lync Server 2013、Skype for Business Server 2015 和 Skype for Business 2019 支持服务器到服务器身份验证;如果未运行其中一个服务器，将无法完全实现 OAuth 身份验证。</span><span class="sxs-lookup"><span data-stu-id="98a39-124">Note that, at this point in time, only Exchange 2013, SharePoint Server, Lync Server 2013, Skype for Business Server 2015, and Skype for Business 2019 support server-to-server authentication; if you are not running one of these servers, you will not be able to fully implement OAuth authentication.</span></span>
  
<span data-ttu-id="98a39-125">还应指出，服务器到服务器身份验证是可选的：如果 Skype for Business Server 不需要与其他服务器 (如 Exchange) ，可以完全跳过服务器到服务器身份验证。</span><span class="sxs-lookup"><span data-stu-id="98a39-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="98a39-126">如果已经为 Lync Server 2013 和其他应用程序配置了服务器到服务器身份验证，则无需为 Skype for Business Server 重新进行配置。</span><span class="sxs-lookup"><span data-stu-id="98a39-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span> 
  
<span data-ttu-id="98a39-127">但是，如果要使用 Skype for Business Server 中的某些功能（如"统一联系人存储"），需要服务器到服务器身份验证。</span><span class="sxs-lookup"><span data-stu-id="98a39-127">However, server-to-server authentication is required if you want to use some of the features in Skype for Business Server, such as the "unified contact store."</span></span> <span data-ttu-id="98a39-128">使用统一的联系人存储，Skype for Business Server 联系人信息存储在 Exchange 中，而不是存储在 Skype for Business Server 中;这使用户能够拥有一组可从 Skype for Business、Outlook 或 Outlook Web Access 中轻松访问的联系人。</span><span class="sxs-lookup"><span data-stu-id="98a39-128">With unified contact store, Skype for Business Server contact information is stored in Exchange instead of in Skype for Business Server; this enables users to have a single set of contacts that is readily accessible from within Skype for Business, Outlook, or Outlook Web Access.</span></span> <span data-ttu-id="98a39-129">由于统一联系人存储要求 Skype for Business Server 与 Exchange 共享信息，因此必须使用服务器到服务器身份验证才能部署该功能。</span><span class="sxs-lookup"><span data-stu-id="98a39-129">Because the unified contact store requires Skype for Business Server to share information with Exchange, you must use server-to-server authentication in order to deploy the feature.</span></span> <span data-ttu-id="98a39-130">如果选择使用 Exchange 存档，还需要服务器到服务器身份验证，在这种情况下，即时消息会话的脚本将另存为 Exchange 电子邮件，而不是单个数据库记录。</span><span class="sxs-lookup"><span data-stu-id="98a39-130">Server-to-server authentication is also required if you choose to use Exchange archiving, in which the transcripts of instant messaging sessions are saved as Exchange emails rather than as individual database records.</span></span>
  
<span data-ttu-id="98a39-131">若要让 Microsoft 365 或 Office 365 版本的 Skype for Business Server 与对应的 Exchange 进行通信，Skype for Business Server 必须先从授权服务器获取安全令牌。</span><span class="sxs-lookup"><span data-stu-id="98a39-131">For the Microsoft 365 or Office 365 version of Skype for Business Server to communicate with its Exchange counterpart, Skype for Business Server must first obtain a security token from the authorization server.</span></span> <span data-ttu-id="98a39-132">然后，Skype for Business Server 使用该安全令牌向 Exchange 标识自身。</span><span class="sxs-lookup"><span data-stu-id="98a39-132">Skype for Business Server then uses that security token to identify itself to Exchange.</span></span> <span data-ttu-id="98a39-133">Exchange 的 Microsoft 365 或 Office 365 版本必须经过相同的过程才能与 Skype for Business Server 进行通信。</span><span class="sxs-lookup"><span data-stu-id="98a39-133">The Microsoft 365 or Office 365 versions of Exchange must go through the same process in order to communicate with Skype for Business Server.</span></span>
  
<span data-ttu-id="98a39-134">但是，对于两台 Microsoft 服务器之间的本地服务器到服务器身份验证，不需要使用第三方令牌服务器。</span><span class="sxs-lookup"><span data-stu-id="98a39-134">However, for on-premises server-to-server authentication between two Microsoft servers there is no need to use a third-party token server.</span></span> <span data-ttu-id="98a39-135">服务器产品（如 Skype for Business Server 和 Exchange）具有一个内置令牌服务器，可用于与支持服务器到服务器身份验证的其他 Microsoft 服务器 (例如 SharePoint Server) 进行身份验证。</span><span class="sxs-lookup"><span data-stu-id="98a39-135">Server products such as Skype for Business Server and Exchange have a built-in token server that can be used for authentication purposes with other Microsoft servers (such as SharePoint Server) that support server-to-server authentication.</span></span> <span data-ttu-id="98a39-136">例如，Skype for Business Server 可以自行颁发和签署安全令牌，然后使用该令牌与 Exchange 进行通信。</span><span class="sxs-lookup"><span data-stu-id="98a39-136">For example, Skype for Business Server can issue and sign a security token by itself, then use that token to communicate with Exchange.</span></span> <span data-ttu-id="98a39-137">在此类情况下，不需要第三方令牌服务器。</span><span class="sxs-lookup"><span data-stu-id="98a39-137">In a case like this, there is no need for a third-party token server.</span></span>
  
<span data-ttu-id="98a39-138">若要为 Skype for Business Server 本地实现配置服务器到服务器身份验证，必须执行两项操作：</span><span class="sxs-lookup"><span data-stu-id="98a39-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
