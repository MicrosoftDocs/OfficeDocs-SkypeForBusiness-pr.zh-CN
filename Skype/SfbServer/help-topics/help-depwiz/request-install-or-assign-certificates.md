# <a name="request-install-or-assign-certificates"></a>请求、安装或分配证书
 
 单击“**运行**”后，“**步骤 3：请求、安装或分配证书**”会启动证书向导。 使用向导配置的证书基于业务服务器 2015年拓扑配置并发布拓扑生成器通过中央管理存储到 Skype 的定义。 要成功运行组织中联机证书颁发机构 (CA) 的证书向导，必须以计算机本地 Administrators 组成员的用户身份登录计算机。 您还必须是计算机和 CA 所在的域中经过身份验证的域用户。 证书向导提供了可以指定备用凭据以访问您的组织的 CA。
  
> [!NOTE]
> 您还可以使用证书向导请求和处理发送到公共 CA 或其他脱机公钥基础结构 (PKI) 的脱机证书请求。要生成脱机请求，除登录计算机所需的组成员身份外，没有其他特定的要求。要处理公共 CA 响应并将证书分配给计算机和角色，必须以本地 Administrators 组成员或等效身份登录。
