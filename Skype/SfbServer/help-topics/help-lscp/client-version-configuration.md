# <a name="client-version-configuration"></a>客户端版本配置
 
除了指定您想要在您的环境支持的客户端的版本，也可以没有已定义的版本策略的客户端中指定默认操作。 这使您能够限制您可以帮助您控制与支持多个客户端版本相关的成本的环境中使用的客户端版本。
  
## <a name="tasks-you-can-perform"></a>可执行的任务

在**客户端版本配置**页，可以执行以下任务：
  
- 编辑默认值 (**全局**) 客户端版本配置。
    
- 创建特定站点的客户端版本配置。
    
- 启用和禁用现有的客户端版本配置。
    
> [!NOTE]
> 因为匿名用户不与某一站点相关联，则匿名用户会受到全局级别的策略。 
  
## <a name="ui-reference"></a>用户界面参考

下表介绍了该页上的菜单、命令、字段和属性。
  
- **新**您可以创建客户端版本配置为特定网站。
    
- **编辑**您可以更改任何客户端版本策略的选项。 使用此选项，可以执行以下操作：
    
  - **显示详细信息**此选项将打开一个对话框，您可以在其中更改客户端版本配置的选项。
    
  - **选择全部**此选项在列表中选择所有的客户端版本配置。
    
  - **删除**此选项将删除所有选定的客户端版本配置。
    
- **刷新**您可以刷新客户端版本配置列表来验证所有的客户端版本配置的选项的状态。
    
在客户端和客户端版本之间的互操作性的详细信息，请参阅规划文档中的[Lync 2013 预览中的客户端互操作性](http://technet.microsoft.com/library/0f126571-91a2-45d5-855c-1e4ddb45fc04.aspx)。 有关使用客户端版本配置的详细信息，请参阅操作文档中[修改客户端未显式支持或受限默认动作](http://technet.microsoft.com/library/548dd0f5-62fe-4c3f-8952-2b9fd4c5fff3.aspx)。
