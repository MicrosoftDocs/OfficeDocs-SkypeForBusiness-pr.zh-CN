# <a name="response-groups-create-new-or-edit-existing-agent-group"></a>响应组：创建新的或编辑现有的代理组
 
代理组定义可应答针对响应组（称为代理）的呼叫的人员以及应用于组中所有代理的设置。
  
## <a name="ui-reference"></a>用户界面参考

下表介绍了该页上的各个字段。
  
- **名称**每个代理组需要唯一的名称。 使用标识组的函数的描述性名称。 例如，技术支持。
    
- **说明**此字段是可选的。 使用它来提供有关组的其他详细信息。
    
- **参与策略**指定要登录到响应组代理的方式：
    
  - 选择“**非正式**”指定组中的代理无需登录和注销。非正式代理登录后，将自动登录。默认设置为“**非正式**”。
    
  - 选择**正式**指定组中的代理必须登录和注销。时选择此选项，单击代理客户端打开浏览器并显示网页的登录和注销控制台中的菜单项。
    
- **警报时间 （秒）** 指定拨打代理电话提供调用下一个空闲的代理之前的秒数。 值必须是至少 10 秒钟和小于： 180 秒。 默认为 20 秒。
    
- **路由方法**选择确定代理接收呼叫的顺序的方法：
    
  - 选择“**最长空闲时间**”可将新呼叫优先路由至空闲（处于“**空闲**”或“**非活动**”的状态）时间最长的代理。
    
  - 选择“**并行**”可将新呼叫同时路由至所有空闲的代理。呼叫将发往第一个接受该呼叫的代理。
    
  - 选择“**循环**”可将新呼叫轮流路由至每个代理。
    
  - 选择“**串行**”将新呼叫始终按照“**代理**”列表中代理的排列顺序路由至代理。
    
  - 选择**助理**以将新呼叫路由至所有代理登录和响应组应用程序同时，无论其当前状态。 助理和客户端用户被配置为代理都可以查看所有呼叫处于等待和可以应答正在等待按任意顺序的呼叫。 呼叫将发往第一个接受该呼叫的代理，其他助理和用户不会再看到此呼叫。
    
- **代理**选择的用户将成为响应组代理通过以下方式之一：
    
  - 选择**使用现有的电子邮件通讯组列表**使用 Exchange 通讯组列表。 在“**通讯组列表地址**”中键入该通讯组列表的电子邮件地址。
    
    > [!NOTE]
    > 只能为每个代理组选择一个通讯组列表。如果通讯组列表包括嵌套的通讯组列表，则该嵌套的通讯组列表将不会被包括在代理组中。 
  
    > [!NOTE]
    > 通讯组列表中代理排列的顺序将影响代理接收呼叫以进行循环和串行路由的顺序。 
  
    > [!NOTE]
    > 隐藏成员身份或隐藏的列表可能会变为可见到响应组管理员或用户。 有关详细信息，请参阅[创建或修改代理组中的业务的 Skype](../../../deploy/deploy-enterprise-voice/create-or-modify-an-agent-group.md)。 
  
  - 选择“**定义自定义代理组**”可选择您要分配为响应组代理的用户。单击“**选择**”可将代理添加到列表中。单击“**删除**”可从列表中删除所选代理。
    
    向上和向下箭头可在代理列表中向上和向下移动所选代理。列表中代理的排列顺序将影响代理接收呼叫以进行循环和串行路由的顺序。
    
有关响应组特性和功能的详细信息，请参阅[规划响应组应用程序中的业务服务器 Skype](../../../plan-your-deployment/enterprise-voice-solution/response-group.md)。 有关使用代理组的详细信息，请参阅操作文档中的[Managing Agent Groups](http://technet.microsoft.com/library/36084cdc-38f1-4c45-922f-f81c7e86210c.aspx) 。
