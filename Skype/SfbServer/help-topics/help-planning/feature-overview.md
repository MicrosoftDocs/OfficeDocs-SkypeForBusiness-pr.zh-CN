# <a name="feature-overview-planning-tool"></a>功能概述 （规划工具）
 
Skype for Business Server 2015 规划工具
  
可以使用规划工具的**中央网站**页面设计业务服务器部署 Skype。 可以创建两个集中或分布式部署。 集中的部署只有一个中心站点，对象为所有有关组织中的业务用户的 Skype。 分布式部署有多个中央站点。 如果您为多个中心站点的业务服务器部署 Skype，然后将在规划工具中输入每个中心站点的用户数。
  
为了完成中央站点的定义，您首先需要提供以下信息：
  
- **站点名** 输入中央站点的名称。
    
- **用户数** 输入用户数，包括驻留在中央站点的分支站点用户。
    
- **云托管用户**托管的用户数从输入到中心站点 Skype 的在线业务。
    
## <a name="ui-elements"></a>UI 元素

其余元素要么已经由你在“**入门**”向导中提供的问题答案填充，要么由规划工具自动填充（如果你跳过了该向导）。
  
### <a name="online-collaboration"></a>联机协作

 “**联机协作**”包含以下选项：
  
- **IM 和状态**
    
    即时消息 (IM) 使用户能够使用基于文本的消息在其计算机上实时相互通信。 支持两方和多方 IM 会话。 状态向用户提供网络上其他人的状态信息。 用户的状态提供信息来帮助确定用户是否处于联机状态，以及如何最好地与用户联系的其他人。 例如，最好通过电子邮件与正在开会的用户联系。
    
- **音频和视频会议**
    
    音频/视频 (A/V) 会议实现实时音频和视频会议。
    
- **电话拨入式会议**
    
    拨入式会议使用户能够从 PSTN 上的电话加入 A/V 会议。拨入式会议需要您部署会议助理和会议公告服务应用程序。
    
- **Web 会议**
    
    Web 会议使防火墙内部和外部的企业用户能够创建和加入在内部服务器上召开的实时会议。
    
- **持久聊天**
    
    通过持久聊天，多个用户可以参与可在其中发布和访问有关特定主题的内容（包括文本、链接和文件）的对话。尽管在会话过程中用户可以实时通信，但每个会话的内容都是持久存在的，这意味着在会话结束后，内容仍然可用。
    
### <a name="users"></a>用户

 “**用户**”包含以下选项：
  
- **内部组织**
    
- **与其他组织的联盟**
    
- **与以前版本的联盟**
    
- **与公共 IM 服务提供商的联盟** 允许组织中的用户与公共即时消息服务提供商（如 MSN、Yahoo! 和 AOL）建立通信。与公共即时消息网络建立联盟需要单独的许可证。
    
- **与基于 XMPP 的服务提供商的联盟**
    
    Skype 业务服务器 2015年的介绍 （在边缘服务器上部署） 的完全集成的 XMPP 代理和您前端服务器上部署的 XMPP 网关。 添加和配置 XMPP 代理服务器可以部署并 XMPP 网关允许业务服务器 2015年用户您 Skype 要从基于 XMPP 的即时消息 (IM) 的伙伴和状态添加联系人。
    
- **移动性**
    
    在部署 Skype 业务服务器 2015年移动服务时，用户可以使用支持的苹果 iOS、 Android、 Windows Phone 或诺基亚移动设备执行为发送和接收即时消息、 查看联系人，查看存在此类活动。
    
- **W15 Exchange 邮箱**
    
    Skype 的业务服务器 2015年使您能够将语音邮件消息存储在 Exchange 统一消息 (UM);然后，这些语音邮件消息将显示为用户的收件箱中的电子邮件。
    
### <a name="voice"></a>语音

 “**语音**”包含以下选项：
  
- **企业语音**
    
    企业语音是 Microsft 的软件驱动的 VoIP 解决方案。 企业语音使用户能够使用 Skype 业务要发出电话呼叫，从他们的计算机。
    
- **Exchange 统一消息**
    
    Exchange 统一消息 (UM) 结合语音邮件和电子邮件到一个消息基础结构。 商业服务器 2015年的 Skype 使用 UM Exchange 提供呼叫应答、 订阅者访问、 电话通知和自动助理服务。 如果您使用这些服务，您需要集成业务共享的 Active Directory 拓扑中的服务器的 Exchange UM 和 Skype。
    
### <a name="additional-deployment-options"></a>其他部署选项

 “**其他部署选项**”包含以下选项：
  
- **高可用性**
    
    高可用性启用备用服务器以支持故障转移。
    
- **灾难恢复**
    
    灾难恢复措施将使您对前端池位于两个数据中心。
    
- **监控**
    
    监控功能捕获与通信会话相关的呼叫详细记录。 它还从参与者终结点处的音频和视频会话收集指标。 监视服务器提供使用情况统计信息、 趋势和介质质量统计。
    
- **存档**
    
    存档功能存储即时消息对话和会议。
    
- **Exchange 存档集成**
    
    如果您有用户驻留在 Exchange 2013 和其邮箱已放在原地保留，可以选择将 Skype 集成业务服务器 2015年存储与 Exchange 存储有关的选项。
    
- **IPv4**
    
    IPv4 地址是 32 位地址，允许计算机通过 Internet 进行通信。由于世界范围内设备数量的不断增加，可用的 IPv4 地址已经用完。因此，许多新设备开始使用 IPv6 地址。
    
- **IPv6**
    
    IPv6 地址执行与 IPv4 地址相同的功能（并另外增加了一些功能），只不过 IPv6 地址不是使用 32 位，而是使用 128 位。这不仅可以提供一组新地址，而且地址数量也大大增加。
    
- **设备更新 Web 服务**
    
    设备更新 Web 服务提供了自动化的方式来更新所有设备，如 Skype 业务为 Windows Phone，对于部署在您的组织之外。
    
### <a name="server-applications"></a>服务器应用程序

 “**服务器应用程序**”包含以下选项：
  
- **响应组**
    
    响应组应用程序自动回答和分布对可用的帮助台代理调用。
    
- **公告**
    
    如果您计划同时部署企业语音，可能希望能够配置电话拨叫的号码是否有效，但不是分配给用户的公共区域的处理方式。 这样，这些调用将转移到预先确定的目标 （电话号码或 SIP URI） 或播放和 / 或音频通知，管理员可以配置发布服务。 使用发布服务可以避免这种情况在其中调用 misdials，听到占线音或 SIP 客户端收到一条错误消息。 发布服务功能是一个典型的 PBX 功能。 
    
- **呼叫寄存**
    
    调用公园应用程序启用企业语音用户呼叫置于保持从一个电话、 并再从另一部电话接收呼叫，而不必占用资源上接收到呼叫电话。 当用户需要转接呼叫，但特定收件人未知时，调用公园应用程序非常有用。 
    
- **会议助理**
    
    会议助理应用程序而无需第三方音频会议提供商的服务电话用户提供音频会议功能。
    
- **会议公告**
    
    会议发布应用程序会生成用户输入或在静音或 unmuted 时向电话用户离开会议，以及通知时发出信号的声音。
    
- **呼叫允许控制**
    
    呼叫允许控制 (CAC) 也称为 WAN 带宽管理，它根据可用带宽确定是否允许建立新的实时通信会话，以帮助防止拥塞网络上出现较差的用户体验质量。 
    
    > [!NOTE]
    > CAC 只控制实时流量，不影响数据流量。 
  
    如果新的语音或视频会话超出您在 WAN 上分配的带宽限制，会话将会被阻止（仅限电话呼叫）或重新路由到 PSTN。
