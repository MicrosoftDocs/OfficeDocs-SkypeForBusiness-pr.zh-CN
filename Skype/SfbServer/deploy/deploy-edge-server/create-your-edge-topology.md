# <a name="create-your-edge-topology-for-skype-for-business-server"></a>为 Business Server Skype 创建边缘拓扑
 
**摘要：** 了解如何构建、 发布和业务服务器导出边缘服务器拓扑中 Skype。
  
拓扑生成器是用于生成边缘服务器拓扑，就像使用任何拓扑组件的 Skype 业务服务器所需的工具。 执行以下步骤之前，您将需要已设置至少一个前端池或 Standard Edition server。
  
我们将在本文中介绍以下主题：
  
- 构建边缘服务器拓扑
    
- 发布边缘服务器拓扑
    
- 验证边缘服务器拓扑
    
  > [!NOTE]
  > 为了遵循下面的步骤，需要登录到下面提及的域服务器，且用来登录的用户应该是以下域组的成员： 
  
- RTCUniversalServerAdmins
    
- Domain Admins
    
## <a name="build-your-edge-server-topology"></a>构建边缘服务器拓扑

第一个部署步骤构建您 Skype 的业务 Server 边缘服务器拓扑，其中包括三个选项之一：
  
- 一台边缘服务器
    
- DNS 负载平衡的边缘池 （一个或多个服务器）
    
- 硬件负载平衡边缘池 （一个或多个服务器）
    
如果不确定自己需要哪种选项，那么在开始遵循这些步骤之前，最好先重温规划文档。否则，让我们开始。
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>为单台边缘服务器定义拓扑

1. 登录到您的业务 Server Standard Edition server 的 Skype 或业务 Server 前端池的 Skype。
    
2. 一次，打开**Skype 的业务 Server 拓扑生成器**。
    
3. 在控制台树中，展开要部署边缘服务器的站点。
    
4. 右键单击**边缘池**，然后单击**新建边缘池**。
    
5. 您将在**定义新边缘池**屏幕上以单击**下一步**。
    
6. 在**定义边缘池 FQDN**屏幕中，键入内部完全限定的域名 (FQDN) 的边缘服务器使用，并选择**单计算机池**，单击**下一步**完成。
    
7. 在“**选择功能**”屏幕上，可选择：
    
   - 要使用相同的 FQDN 和 IP 地址 SIP 访问服务、 业务 Server Web 会议服务，您 Skype 和 A / V 边缘服务。 如果是这样，需要选中“**使用一个 FQDN 和 IP 地址**”复选框（做到下面的步骤 9 时，请记得这一点）
    
   - 如果你计划启用联盟，请选中“**为此边缘池启用联盟(端口 5061)**”复选框。
    
8. 单击“**下一步**”后，你将进入“**IP 选项**”屏幕。选项如下：
    
   - 在内部接口上启用 IPv4
    
   - 在内部接口上启用 IPv6
    
   - 在外部接口上启用 IPv4
    
   - 在外部接口上启用 IPv6
    
   这些是无需解释，是否您使用的 IPv4 或 IPv6 地址，并且您将应用这些地址上边缘服务器内部或外部 （需要牢记这的步骤 10）。 您还可以使用网络地址转换 (NAT) 地址的外部 IP 地址配置边缘服务器或边缘池的选择。 选中“**此边缘池的外部 IP 地址是由 NAT 转换的**”复选框可以执行此配置。 完成后，单击“**下一步**”。
    
9. 在“外部 FQDN”屏幕上，你的选择取决于你在上面的步骤 7 中所作的选择。
    
   - 如果选中**使用单个 FQDN 和 IP 地址**复选框，您需要在**SIP 访问**框中输入您的单个外部 FQDN。 然后，您需要输入不同的端口号为每个边缘服务使所有独立连接。 我们建议 **SIP 访问**边缘服务使用 5061，**Web 会议**边缘服务使用 444，**A/V** 边缘服务使用 443。 完成后单击“**下一步**”。
    
   - 如果未选中**使用单个 FQDN 和 IP 地址**复选框，您现在需要输入三个外部 Fqdn 的**SIP 访问**边缘服务， **Web 会议**边缘服务，以及**A / V**边缘服务。 完成后，单击“**下一步**”。
    
10. 现在，您在**定义内部 IP 地址**屏幕上。 此处，您需要在**内部 IPv4 地址**和**内部 IPv6 地址**文本框中的根据您在步骤 8 中后所做的选择中, 键入边缘服务器的 IP 地址。 完成后单击“**下一步**”。
    
11. 在“**定义外部 IP 地址**”屏幕上，根据你前面的选择，有几个选项：
    
    - 你可能让所有服务使用一个 FQDN。 如果是这样，在**SIP 访问**文本框中，键入 （无论您使用） 您外部 IPv4 或 IPv6 地址，然后单击**下一步**。
    
    - 你可能选择让这些服务使用三个不同的 FQDN 和 IP 地址。 如果是这样，用于**SIP 访问**边缘服务， **Web 会议**边缘服务中，输入您的外部 IPv4 或 IPv6 地址和**A / V**边缘服务，，然后单击**下一步**。
    
    > [!NOTE]
    > 如果先前未选择启用和分配 IPv6 编址，则不会出现此对话框。这是正常现象，请直接进入下一步。 
  
12. 如果您选择在步骤 8 后使用 NAT，您现在将获取与**公共 IP 地址**文本框的屏幕。 您需要输入公共 IPv4 和/或 IPv6 地址已设置为 A / V 边缘服务，要转换的 nat。 然后单击“**下一步**”。
    
13. 接下来的屏幕是“**定义下一跃点**”。 在**下一跃点池**框中，选择您的内部池，这可能是前端池或独立池的名称。 如果您的环境中有控制器，则应选择控制器。 然后单击" **下一步**"。
    
14. 在**关联前端池**屏幕中，您需要指定一个或多个内部池，包括前端池和 Standard Edition server，与该边缘服务器相关联。 只需选择要使用此边缘服务器的受支持的外部用户进行通信的内部池的名称。 单击“**下一步**”。
    
    > [!NOTE]
    > 内容需要注意以下事项此处是，如果您的内部池或独立服务器已经在使用不同的 Skype 业务 Server 边缘服务器，不能具有多个关联。 如果您选择的内部池或独立服务器上是这种情况下，您将看到有关其他边缘服务器，告知您显示一条警告，您可以决定是否要继续与否。 如果继续这个新的关联，与另一台边缘服务器的连接将停止。 
  
15. 在下一个屏幕上单击“**完成**”。
    
16. 现在，您将能够发布此更新的技术，并按照[Skype 业务服务器中部署边缘服务器](deploy-edge-servers.md)中的说明从此处部署到边缘服务器。

## <a name="publish-your-edge-server-topology"></a>发布边缘服务器拓扑

在完成上述步骤后，就可以发布此新拓扑，还允许您将其导出到业务 Server 边缘服务器或边缘池您 Skype。 请按以下步骤执行：
  
1. 启动“**拓扑生成器**”（如果在前面的过程中还未启动）。
    
2. 在**拓扑生成器**，在控制台树中，右键单击**Skype 业务服务器**，然后单击**Skype 的业务 Server 拓扑生成器**。

## <a name="export-your-edge-server-topology"></a>验证边缘服务器拓扑

若要成功部署，业务 Server 部署向导的 Skype 需要对中央管理存储数据的访问。 对于域或林中的内部服务器，这通常很容易。 边缘服务器的外部域，并因此所需手动到边缘服务器上的位置，通常物理媒体导出拓扑文件。 这样的导出通过 PowerShell 执行：
  
1. 启动**Skype 的业务 Server 命令行管理程序**。
    
2. 在**Skype 的业务 Server Management Shell**中，运行以下命令：
    
   ```
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. 将导出的文件复制到外部介质 （例如，USB 驱动器或网络共享，可以从边缘服务器的位置达到）。
