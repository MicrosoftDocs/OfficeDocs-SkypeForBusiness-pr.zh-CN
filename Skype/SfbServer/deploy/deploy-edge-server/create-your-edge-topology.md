# <a name="create-your-edge-topology-for-skype-for-business-server"></a>创建边缘拓扑Skype for Business Server
 
**摘要：** 了解如何在 Skype for Business Server 中生成、发布和导出边缘服务器Skype for Business Server。
  
拓扑生成器是生成边缘服务器拓扑所需的工具，就像它用于任何拓扑组件Skype for Business Server。 在按照以下步骤操作之前，至少需要设置一个前端池或一Standard Edition服务器。
  
我们将介绍本文中的以下主题：
  
- 构建边缘服务器拓扑
    
- 发布边缘服务器拓扑
    
- 导出边缘服务器拓扑
    
  > [!NOTE]
  > 若要按照以下步骤操作，你需要以以下域组的成员用户登录下面提到的域服务器： 
  
- RTCUniversalServerAdmins
    
- Domain Admins
    
## <a name="build-your-edge-server-topology"></a>构建边缘服务器拓扑

第一个部署步骤是Skype for Business Server边缘服务器拓扑，其中包括以下三个选项之一：
  
- 单个边缘服务器
    
- DNS 负载平衡边缘池 (一台或多台服务器) 
    
- 硬件负载平衡边缘池 (一台或多台服务器) 
    
如果您不确定所需的内容，那么在开始执行这些步骤之前，应该先查阅规划文档。 否则，让我们开始吧。
  
### <a name="defining-the-topology-for-a-single-edge-server"></a>定义单个边缘服务器的拓扑

1. 登录到您的Skype for Business Server Standard Edition服务器或Skype for Business Server前端池。
    
2. 完成后，打开 **Skype for Business Server生成器"。**
    
3. 在控制台树中，展开要部署边缘服务器的站点。
    
4. 右键单击 **"边缘池"，** 然后单击"**新建边缘池"。**
    
5. 您将在"定义 **新的边缘****池"屏幕上单击"下一步**"。
    
6. 在"定义边缘池 **FQDN"** 屏幕上，键入边缘服务器将使用的内部完全限定域名 (FQDN) ，然后选择"单计算机池"，完成后单击"下一步"。 
    
7. 在 **"选择功能** "屏幕上，可以选择：
    
   - 您可能想要对 SIP 访问服务、Skype for Business Server Web 会议服务和 A/V 边缘服务使用相同的 FQDN 和 IP 地址。 如果是这样，你需要选中"使用单个 **FQDN 和 IP** 地址"复选框 (在下面的步骤 9 中请记住这一) 
    
   - 如果计划启用联盟，请选中"为此边缘池启用联盟 (**端口 5061**) 复选框。
    
8. 单击"下一步 **"** 后，你将在"IP 选项" **屏幕上找到** 自己。 选项如下所示：
    
   - 在内部接口上启用 IPv4
    
   - 在内部接口上启用 IPv6
    
   - 在外部接口上启用 IPv4
    
   - 在外部接口上启用 IPv6
    
   无论是使用 IPv4 地址还是 IPv6 地址，这些地址在边缘服务器内部或外部应用 (在步骤 10) 中都需要记住这一点。 还可以将边缘服务器或边缘池配置为使用外部 IP 地址的网络地址 (NAT) 地址。 为此，请选中"此边缘池的外部 IP 地址由 **NAT** 转换"复选框。 准备就绪 **后，** 单击"下一步"。
    
9. 在"外部 FQNS"屏幕上，您的选择取决于您在以上步骤 7 中所做的选择。
    
   - 如果选中了" **使用单个 FQDN 和 IP** 地址"复选框，则需要在 **"SIP** 访问"框中输入单个外部 FQDN。 然后，你需要为各个边缘服务输入不同的端口号，以允许它们相互独立连接。 建议使用 5061 作为 **SIP** 访问边缘服务，444 用于 **Web** 会议边缘服务，443 用于 **A/V** 边缘服务。 完成后，单击 **“下一步”**。
    
   - 如果未选中"使用单个 **FQDN** 和 IP 地址"复选框，则现在将需要为 **SIP** 访问边缘服务 **、Web** 会议边缘服务和 **A/V** 边缘服务输入三个外部 FQDN。 完成后，单击 **“下一步”**。
    
10. 现在，您位于" **定义内部 IP 地址"** 屏幕上。 在这里，您将在"内部 **IPv4** 地址"和"内部 **IPv6** 地址"文本框中键入边缘服务器的 IP 地址，具体取决于在步骤 8 中所做的选择。 准备就绪 **后，** 单击"下一步"。
    
11. 在 **"定义外部 IP 地址** "屏幕上，有一些选项，具体取决于以前的选择：
    
    - 您可以针对所有服务使用单个 FQDN。 如果是这样，请在 **"SIP** 访问"文本框中键入外部 IPv4 或 IPv6 (键入任何使用) 的地址，然后单击"下一 **步"。**
    
    - 您可能选择了对服务使用三个单独的 FQDN 和 IP 地址。 如果是这种情况，请输入 **SIP** 访问边缘服务 **、Web** 会议边缘服务和 **A/V** 边缘服务的外部 IPv4 或 IPv6 地址，然后单击"下一步 **"。**
    
    > [!NOTE]
    > 如果之前未选择启用和分配 IPv6 寻址，则看不到此对话框。 这很正常，只需转到下一步。 
  
12. 如果你选择在步骤 8 中重新使用 NAT，你现在将看到一个包含公用 **IP** 地址文本框的屏幕。 你需要输入为 A/V 边缘服务设置的公共 IPv4 和/或 IPv6 地址，以通过 NAT 进行转换。 然后单击 **下一步**。
    
13. 下一个屏幕是 **定义下一个跃点**。 在" **下一个** 跃点池"框中，选择内部池的名称，该名称可能是前端池或独立池。 如果您的环境中有控制器，则应该选择控制器。 然后单击 **下一步**。
    
14. 在 **"关联前端池**"屏幕上，需要指定一个或多个内部池（包括前端池和 Standard Edition 服务器）以与此边缘服务器关联。 只需选择希望使用此边缘服务器与受支持的外部用户进行通信的内部池的名称。 单击“**下一步**”。
    
    > [!NOTE]
    > 此处需要记住的一点就是，如果内部池或独立服务器已在使用与边缘服务器Skype for Business Server不同，则它们不能有多个关联。 如果您选择处于该情况的内部池或独立服务器，则会显示一条警告，告知您有关其他边缘服务器的信息，您可以决定是否要继续。 如果继续此新关联，将停止与其他边缘服务器的连接。 
  
15. 单击 **下一** 个屏幕上的"完成"。
    
16. 现在，您将能够发布此更新的技术，并按照部署 Skype for Business Server 中的边缘服务器中的[](deploy-edge-servers.md)说明从此处部署到边缘服务器。

## <a name="publish-your-edge-server-topology"></a>发布边缘服务器拓扑

完成上述步骤后，可以发布此新拓扑，这还允许您将其导出到 Skype for Business Server 边缘服务器或边缘池。 请按以下步骤操作：
  
1. 如果 **拓扑生成器** (上一过程尚未启动，请启动拓扑生成器) 。
    
2. 在 **拓扑生成器的** 控制台树中，右键单击 **"Skype for Business Server"，** 然后单击"Skype for Business Server **生成器"。**
    
3. 在向导的“欢迎”页上，单击“下一步”。
    
4. 在“创建其他数据库”页上，单击“下一步”。
    
5. 当状态指示数据库创建成功时，请执行下列操作：
    
    - 要查看日志，请单击“查看日志”。
    
    - 要关闭向导，请单击“完成”。
    
## <a name="export-your-edge-server-topology"></a>导出边缘服务器拓扑

若要成功部署，Skype for Business Server向导需要访问中央管理存储数据。 对于域或林中的内部服务器，这通常很简单。 边缘服务器位于域外部，因此有必要手动将拓扑文件导出到边缘服务器位置，通常是在物理媒体上。 此导出通过 PowerShell 完成：
  
1. 启动 **Skype for Business Server 命令行管理程序**。
    
2. 在 **"Skype for Business Server命令行管理程序**"中，运行以下命令：
    
   ```powershell
   Export-CsConfiguration -FileName <ConfigurationFilePath.zip>
   ```

3. 将导出的文件复制到外部媒体 (例如，可以从边缘服务器的位置访问的 USB 驱动器或网络) 。
