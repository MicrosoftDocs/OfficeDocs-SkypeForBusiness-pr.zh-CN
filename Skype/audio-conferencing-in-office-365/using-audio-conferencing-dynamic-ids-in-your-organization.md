# 在您的组织中使用音频会议动态 Id

> [!重要信息]
> 本文是由机器翻译的，请参阅[免责声明](e55e4bff-fb67-4389-8695-f03024baa9b6.md#MT_Footer)。请在 [此处](https://support.office.com/en-us/article/e55e4bff-fb67-4389-8695-f03024baa9b6) 中查找本文的英文版本以便参考。
  
正在更新的音频会议服务提供每个 Skype for Business 和 Microsoft 团队会议与其他会议 Id。动态会议 Id 通过静态会议 Id，并重大改进，因为它们提供：
  
- **增强的安全性** 会议 Id 是唯一的每个 Skype for Business 或 Microsoft 小组会议和安排了会议时，会生成。
    
- **提供了更好的背靠背和侧对侧会议体验** 系统会为一个组织者安排的每场会议提供特定的拨入信息，当多个会议的时间彼此邻近时，这些信息可防止一个会议的电话参与者与其他会议的参与者混淆在一起。
    
- **可以无缝过渡** 为组织启用动态会议 ID 后，组织中具有静态会议 ID 的所有已安排会议将继续有效。
    
> [!提示]
> 动态 Id 仅适用于启用了 **音频会议**的用户和 Microsoft 设置为其音频会议提供商。 可以为您的用户[将 Microsoft 作为音频会议提供商分配](assign-microsoft-as-the-audio-conferencing-provider.md)。 
  
## 我的组织中的用户将看到哪些更改？

为您的组织已启用动态会议 Id 后，将任何新 Skype for Business 或 Microsoft 小组会议的计划由您的组织中启用了音频会议将具有会议 Id 将不同的用户他们以前的静态会议 ID。组织者具备静态会议 Id 之前需要提醒加入他们他们现在会议邀请中使用新会议 ID，然后才可以加入它所需的会议的用户。
  
> [!注释]
> 组织启用的动态会议 Id 仍然能拥有的静态会议 Id，以便他们将继续安排会议，而不影响之前，具有静态会议 Id 的用户的计划的会议。
