# <a name="end-of-life-program-for-the-integration-of-skype-for-business-with-third-party-audio-conferencing-providers"></a>与第三方音频会议提供商 Skype 业务集成的生命程序结束 

Microsoft 发布生命程序与第三方音频会议提供商 (ACPs) 集成业务的 Skype 的结束的开始。 

在 4 月 1 日，2019年将得出的结论生命程序结束。 当该程序得出结论时，与第三方音频会议提供商 Skype 业务的集成将停止工作并将在这一天 (4 月 1 日，2019) 观察到以下更改： 

- 尝试加入任何拨入号码通过商务会议的 Skype 提供由第三方 ACP 服务将不能再连接到 Skype 业务会议的参与者。
 
- 用户启用第三方 ACP 服务将不再具有拨入信息自动包含在任何新的 Skype 业务会议的邀请。 

作为生命程序结束的开始发布的一部分，以下更改已经生效，并将继续生命程序结束的结论之前准备就绪： 

- 没有适用于业务用户配置为使用第三方 ACP 服务 Skype 的客户将无法配置任何用户使用第三方 ACP 服务。 

- 适用于业务用户配置为使用第三方 ACP 服务与 Skype 的现有客户将继续能够在生命周期结束的持续时间内添加新用户。 请注意，我们不建议设置其他 Skype 业务用户使用第三方 ACP 服务，作为在 4 月 1 日的 2019年会生效的更改也将应用于它们。 

## <a name="preparing-for-this-change"></a>准备此更改

要准备进行此更改，我们鼓励受影响的组织通知此计划在 4 月 1 日，2019年之前更新其启用的用户。 

在 2019，4 月 1 日之后, 用户可以继续对其在线会议; 没有中断业务使用 Skype但是，企业将需要启用及其用户如果它们需要拨入音频会议与 Skype 业务或 Microsoft 小组由 Microsoft 提供的音频会议的。 若要了解有关 Microsoft 音频会议的详细信息，请参阅[音频会议](https://products.office.com/en-us/skype-for-business/audio-conferencing)。 

根据组织的需要的最终状态，没有可遵循的三个路径：

- 将迁移到 Microsoft 音频会议。 
- 继续单独使用第三方音频会议提供商。 
- 停止使用拨入会议完全。

### <a name="path-1-migrate-to-microsoft-audio-conferencing"></a>路径 1： 迁移到 Microsoft 音频会议   

期间或该日期之后，决定迁移到 Microsoft 音频会议并完成 2019，4 月 1 日之前, 他们迁移的组织将不会遇到任何服务的影响。 在迁移到 Microsoft 音频会议将介绍组织以下更改： 

- 该服务将货款与所有其他 Office 365 提供服务。 

- 如果购买的标准订阅，收费电话拨入成本将包括在每个用户每月订购费的成本。 

- 将为每个组织和它的用户提供一组新的拨入电话号码。 若要查看 Microsoft 音频会议服务的地理覆盖范围，请参阅[音频会议和调用计划的国家和地区可用性](../country-and-region-availability-for-audio-conferencing-and-calling-plans/country-and-region-availability-for-audio-conferencing-and-calling-plans.md)。
 
- 会议已安排与第三方 ACP 已启用的用户将自动重新安排包括 Microsoft 音频会议拨入信息。
 
- 会议 Id 的每次会议将是动态的也就是说，每次会议都将有自己专用的会议 id。 动态的会议 Id 为背对背会议提供增强的安全性和改进的经验。

- 所有服务的使用率都受到音频会议服务使用的术语。 

迁移到 Microsoft 音频会议很简单，并且它获取服务的许可证后可以在几个简单的步骤。 要了解有关如何迁移到 Microsoft 音频会议信息，请参阅：

- [试用或购买 Office 365 中的音频会议](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)
 
**摘要：**

- 将迁移到 Microsoft 音频会议和 2019，4 月 1 日，将不会看到任何影响到他们的服务期间或之后的日期之前完成其迁移的企业。

- 若要了解有关迁移到 Microsoft 音频会议的详细信息，请参见[试用或购买 Office 365 中的音频会议](../audio-conferencing-in-office-365/try-or-purchase-audio-conferencing-in-office-365.md)。 

### <a name="path-2-continue-to-separately-use-a-third-party-audio-conferencing-provider"></a>路径 #2： 继续单独使用第三方音频会议提供商

组织决定继续使用第三方 ACP 上和 2019，4 月 1 日之后, 将会服务受到影响，因为第三方 ACP 拨入信息将不再能够被用来加入 Skype 业务会议的音频部分。 

若要防止在 Skype 的通过让一些参与者加入通过 VoIP 业务会议音频和其他通过第三方 ACP 的碎片，建议这些组织可以禁用其用户的会议上使用 VoIP。 这样一来，所有的参与者将需要加入使用第三方 ACP 会议的音频部分，（如聊天或屏幕共享） 的会议的所有其他工作负荷可以继续通过 Skype 的业务支持。 

- 若要禁用 VoIP 的给定组织的所有会议中，将他或她会议策略的 AllowIPAudio 参数通过设置 CsConferencingPolicy cmdlet 的假 有关其他信息，请参阅[设置 CsConferencingPolicy](https://docs.microsoft.com/en-us/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)。
 
根据计划，和自 2019，4 月 1 日，第三方 ACP 的拨入信息将不再自动包括在 Skype 业务会议邀请的。 用户将需要手动添加其 Skype 业务会议上的拨入信息邀请如果他们想要继续包括此信息作为其会议的一部分。 

请注意，在 2019，4 月 1 日，用户的现有会议将不能自动重新安排移去任何第三方 ACP 拨入信息。 决定保留为其用户的会议应考虑禁用第三方为其用户的 ACP 的集成和重新安排其会议使用的会议启用 VoIP 的组织迁移服务来删除第三方音频从其现有会议会议拨入信息和防止碎片上已安排会议的音频。 

- 若要禁用的给定组织的第三方音频会议服务的集成，使用删除 CsUserAcp cmdlet。 有关其他信息，请参阅[删除 CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps)。 

- 若要禁用第三方音频会议提供商集成后自动重排的用户参加的会议，请参阅"如何运行手动为用户会议迁移？" 在[设置了会议迁移服务 (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)。 

**摘要：**

- 决定继续使用第三方 ACP 上和 2019，4 月 1 日之后, 的公司将受到影响，因为第三方 ACP 不能用来参加商务会议 Skype 和新会议将不包括第三方 ACP 拨入信息。 

- 建议在 4 月 1 日，2019，以防止在参与者加入通过 VoIP 和通过第三方 ACP 的碎片化音频之前 VoIP 禁用对所有受影响的用户的所有会议。 

    - 若要禁用 VoIP 的给定组织的所有会议中，设置的 AllowIPAudio 的用户的会议策略通过设置 CsConferencingPolicy cmdlet 的假。 有关其他信息，请参阅[设置 CsConferencingPolicy](https://docs.microsoft.com/en-us/powershell/module/skype/set-csconferencingpolicy?view=skype-ps)。
 
- 如果组织不能禁用 VoIP 的所有会议，建议用户使用 Skype 与第三方 ACP 的在线业务集成被禁用并重新安排其会议删除第三方 ACP 拨入信息到防止碎片的音频。

    - 若要禁用的给定组织的第三方音频会议服务的集成，使用[删除 CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps) cmdlet。 

    - 以便自动重新计划会议，请参阅"如何运行手动为用户会议迁移？" 在[设置了会议迁移服务 (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)。

### <a name="path-3-stop-using-dial-in-conferencing-altogether"></a>路径 3： 停止完全使用拨入会议

决定停止完全使用拨入会议的组织 （既不提供由 Microsoft 也不受第三方 ACP） 可以完全依赖于 VoIP 业务会议支持 Skype 的音频部分。 

这些组织需要禁用其用户使用第三方音频会议提供商，并已使用会议迁移自动重新安排其会议服务中删除其拨入会议信息。 

- 若要禁用的给定组织的第三方音频会议服务的集成，使用删除 CsUserAcp cmdlet。 有关其他信息，请参阅[删除 CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps)。 

- 若要禁用第三方音频会议提供商集成后自动重排的用户参加的会议，请参阅"如何运行手动为用户会议迁移？" 在[设置了会议迁移服务 (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)。 

**摘要：** 

- 决定停止使用音频会议的组织完全在 2019，4 月 1 日之前不会受到影响。

- 若要禁用的给定组织的第三方音频会议服务的集成，使用删除 CsUserAcp cmdlet。 有关其他信息，请参阅[删除 CsUserAcp](https://docs.microsoft.com/en-us/powershell/module/skype/remove-csuseracp?view=skype-ps)。 

- 若要禁用第三方音频会议提供商与集成后自动重排的用户参加的会议，请参阅"如何运行手动为用户会议迁移？" 在[设置了会议迁移服务 (MMS)](../audio-conferencing-in-office-365/setting-up-the-meeting-migration-service-mms.md)。
