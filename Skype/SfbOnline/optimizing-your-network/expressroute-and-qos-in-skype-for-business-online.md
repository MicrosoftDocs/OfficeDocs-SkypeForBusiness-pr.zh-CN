### <a name="understanding-the-types-of-ip-network-services--basic-ip-and-mpls"></a><span data-ttu-id="0a8ff-736">了解 IP 网络服务的类型-基本 IP 和 MPLS</span><span class="sxs-lookup"><span data-stu-id="0a8ff-736">Understanding the types of IP Network Services- Basic IP and MPLS</span></span>

<span data-ttu-id="0a8ff-737">在 "最大努力" 原则上运营的原始 IP 数据包转发。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-737">The original IP packet forwarding operated on the principle of "best effort."</span></span> <span data-ttu-id="0a8ff-738">这意味着，转发这些 IP 数据包的路由器最能将它们传送到其目的地，但对他们的目的地或他们的目的地而言绝对不能保证。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-738">That meant that the routers forwarding those IP packets would do their best to deliver them to their destinations, but there was absolutely no guarantee with regard to when or if they would arrive at their destinations.</span></span> <span data-ttu-id="0a8ff-739">这就是如何使用基本的互联网服务，包括您的家庭互联网连接、今天的工作。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-739">That is how basic Internet services, including your home Internet connection, work today.</span></span> <span data-ttu-id="0a8ff-740">思路是，如果特定应用程序需要可靠性，它将在协议堆栈中更高级别提供。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-740">The idea was that if reliability was required for a particular application, it would be provided at a higher level in the protocol stack.</span></span> <span data-ttu-id="0a8ff-741">可靠的传递机制是传输控制协议（TCP）。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-741">The reliable delivery mechanism is the Transmission Control Protocol (TCP).</span></span> <span data-ttu-id="0a8ff-742">用于实时语音和视频的用户数据报协议（UDP）是不可靠的（即 "最大努力"）交付机制。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-742">The User Datagram Protocol (UDP), which is used for real-time voice and video, is the unreliable (i.e. "best effort") delivery mechanism.</span></span> 
  
<span data-ttu-id="0a8ff-743">多协议标签交换（MPLS）是为网络服务提供商提供的一种方法，为网络服务提供商提供了一种可保证延迟、抖动和数据包丢失性能的 IP 服务。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-743">Multi-Protocol Label Switching (MPLS) was developed as a means for network service providers to offer an IP service with performance guarantees for delay, jitter and packet loss.</span></span> <span data-ttu-id="0a8ff-744">为提供这些性能保证，MPLS 将从传统 IP 中提取一些 unpredictability。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-744">To deliver on those performance guarantees, MPLS takes some of the unpredictability out of traditional IP.</span></span> <span data-ttu-id="0a8ff-745">首先，不是让每个数据包都能找到其目标的路由器到路由器的方式（这可能是因为每个数据包都采用不同的路由来源于目标），MPLS 将 "虚拟线路" 连接上的所有数据包路由到目标固定路由称为标签交换路径（LSP）。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-745">First, rather than having each packet find its way router-to-router to its destination (the result of which could be that each packet takes a different route from the source to the destination), MPLS routes all of the packets on a "virtual circuit" connection with a fixed route called a Label Switched Path (LSP).</span></span> <span data-ttu-id="0a8ff-746">如果该路径中的其中一个链接失败，则使用该链接的所有 Lsp 都将快速重新路由。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-746">If one of the links in that path fails, all of the LSPs using that link are quickly rerouted.</span></span>
  
<span data-ttu-id="0a8ff-747">将数据包发送到 MPLS 网络时，网络服务提供商的边缘路由器会将一个附加报头追加到数据包中，该数据包包含用于将其转发到相应 LSP 的标签。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-747">When a packet is sent into the MPLS network the network service provider's edge router appends an additional header to the packet that includes a label that is used to forward it over the appropriate LSP.</span></span> <span data-ttu-id="0a8ff-748">该标签由 MPLS 网络另一端的边缘路由器去除。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-748">The label is stripped off by the edge router at the other end of the MPLS network.</span></span>
  
<span data-ttu-id="0a8ff-749">在简化转发过程的旁边，MPLS 提供的另一个优点是网络管理系统将知道在网络中的每个链接上传输了哪些连接。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-749">Beside simplifying the forwarding process, the other advantage MPLS provides is that the network management system will know what connections are being carried on every link in the network.</span></span> <span data-ttu-id="0a8ff-750">通过控制流量通过网络路由的方式，运营商可以保证每个路径将提供的 QoS。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-750">By controlling the way traffic is routed through the network, the operator can guarantee the QoS each path will provide.</span></span> <span data-ttu-id="0a8ff-751">因此，与传统或基本 IP 的最佳操作性能不同，MPLS 操作员可以提供具有可预测性能的 IP 服务。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-751">So unlike the best effort performance of traditional or basic IP, MPLS operators can provide an IP service with predictable performance.</span></span> <span data-ttu-id="0a8ff-752">这种 LSP 还使得 MPLS 比传统的 Internet 服务本身更安全。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-752">That LSP also makes MPLS inherently more secure than traditional Internet services.</span></span> <span data-ttu-id="0a8ff-753">因此，使用基本 IP 服务，我们可以希望网络能够以更好的方式提供优质语音，并使用 FEC 和更具弹性的语音编码来提高机会，但通过使用 MPLS，我们可以确定。</span><span class="sxs-lookup"><span data-stu-id="0a8ff-753">So with basic IP service we can hope that the network will perform well enough to provide good quality voice and use techniques like FEC and more resilient voice coding to improve the odds, but by using MPLS, we can be sure of it.</span></span>
