<span data-ttu-id="a468e-737">原始 IP 数据包转发以"尽力而为"的原则运行。</span><span class="sxs-lookup"><span data-stu-id="a468e-737">The original IP packet forwarding operated on the principle of "best effort."</span></span> <span data-ttu-id="a468e-738">这意味着转发这些 IP 数据包的路由器将尽力将其发送到其目标，但绝对无法保证它们何时到达或是否到达目的地。</span><span class="sxs-lookup"><span data-stu-id="a468e-738">That meant that the routers forwarding those IP packets would do their best to deliver them to their destinations, but there was absolutely no guarantee with regard to when or if they would arrive at their destinations.</span></span> <span data-ttu-id="a468e-739">这就是基本的 Internet 服务（包括家庭 Internet 连接）目前如何工作。</span><span class="sxs-lookup"><span data-stu-id="a468e-739">That is how basic Internet services, including your home Internet connection, work today.</span></span> <span data-ttu-id="a468e-740">其思路是，如果特定应用程序需要可靠性，则它将在协议堆栈中提供更高的级别。</span><span class="sxs-lookup"><span data-stu-id="a468e-740">The idea was that if reliability was required for a particular application, it would be provided at a higher level in the protocol stack.</span></span> <span data-ttu-id="a468e-741">可靠的传送机制是 TCP 传输 (协议) 。</span><span class="sxs-lookup"><span data-stu-id="a468e-741">The reliable delivery mechanism is the Transmission Control Protocol (TCP).</span></span> <span data-ttu-id="a468e-742">用户数据报协议 (UDP) （用于实时语音和视频）是不可靠的 (，即"尽力而为") 机制。</span><span class="sxs-lookup"><span data-stu-id="a468e-742">The User Datagram Protocol (UDP), which is used for real-time voice and video, is the unreliable (i.e. "best effort") delivery mechanism.</span></span> 
  
<span data-ttu-id="a468e-743">多协议标签 (MPLS) 是网络服务提供商提供 IP 服务，保证延迟、抖动和数据包丢失的一种途径。</span><span class="sxs-lookup"><span data-stu-id="a468e-743">Multi-Protocol Label Switching (MPLS) was developed as a means for network service providers to offer an IP service with performance guarantees for delay, jitter and packet loss.</span></span> <span data-ttu-id="a468e-744">为了提供这些性能保证，MPLS 从传统 IP 中排除一些不可预测性。</span><span class="sxs-lookup"><span data-stu-id="a468e-744">To deliver on those performance guarantees, MPLS takes some of the unpredictability out of traditional IP.</span></span> <span data-ttu-id="a468e-745">首先，MPLS 使用名为标签切换路径 (LSP) 的固定路由将"虚拟线路"连接上的所有数据包路由到"虚拟线路"连接上的所有数据包，而不是让每个数据包找到其路由器到路由器到目标 (的结果，其结果可能是每个数据包采用不同的路由到目标) 。</span><span class="sxs-lookup"><span data-stu-id="a468e-745">First, rather than having each packet find its way router-to-router to its destination (the result of which could be that each packet takes a different route from the source to the destination), MPLS routes all of the packets on a "virtual circuit" connection with a fixed route called a Label Switched Path (LSP).</span></span> <span data-ttu-id="a468e-746">如果该路径中的一个链接失败，使用该链接的所有 LSP 都快速重新路由。</span><span class="sxs-lookup"><span data-stu-id="a468e-746">If one of the links in that path fails, all of the LSPs using that link are quickly rerouted.</span></span>
  
<span data-ttu-id="a468e-747">将数据包发送到 MPLS 网络时，网络服务提供商的边缘路由器将附加一个标头追加到数据包，其中包括用于通过相应 LSP 转发该数据包的标签。</span><span class="sxs-lookup"><span data-stu-id="a468e-747">When a packet is sent into the MPLS network the network service provider's edge router appends an additional header to the packet that includes a label that is used to forward it over the appropriate LSP.</span></span> <span data-ttu-id="a468e-748">该标签被 MPLS 网络另一端的边缘路由器去除。</span><span class="sxs-lookup"><span data-stu-id="a468e-748">The label is stripped off by the edge router at the other end of the MPLS network.</span></span>
  
<span data-ttu-id="a468e-749">除了简化转发过程外，MPLS 提供的另一个优势是，网络管理系统将知道网络中每个链路上携带了哪些连接。</span><span class="sxs-lookup"><span data-stu-id="a468e-749">Beside simplifying the forwarding process, the other advantage MPLS provides is that the network management system will know what connections are being carried on every link in the network.</span></span> <span data-ttu-id="a468e-750">通过控制通过网络路由流量的方式，操作员可以保证每个路径将提供 QoS。</span><span class="sxs-lookup"><span data-stu-id="a468e-750">By controlling the way traffic is routed through the network, the operator can guarantee the QoS each path will provide.</span></span> <span data-ttu-id="a468e-751">因此，与传统或基本 IP 的最佳性能不同，MPLS 操作员可以提供具有可预测性能的 IP 服务。</span><span class="sxs-lookup"><span data-stu-id="a468e-751">So unlike the best effort performance of traditional or basic IP, MPLS operators can provide an IP service with predictable performance.</span></span> <span data-ttu-id="a468e-752">该 LSP 还使 MPLS 本身比传统 Internet 服务更安全。</span><span class="sxs-lookup"><span data-stu-id="a468e-752">That LSP also makes MPLS inherently more secure than traditional Internet services.</span></span> <span data-ttu-id="a468e-753">因此，对于基本 IP 服务，我们可以希望网络性能足以提供高质量的语音，并使用 FEC 等技术以及复原能力更强的语音编码来提高机率，但通过使用 MPLS，我们可以确信这一点。</span><span class="sxs-lookup"><span data-stu-id="a468e-753">So with basic IP service we can hope that the network will perform well enough to provide good quality voice and use techniques like FEC and more resilient voice coding to improve the odds, but by using MPLS, we can be sure of it.</span></span>
  
<span data-ttu-id="a468e-754">MPLS 提供商提供了多种服务渐变，遗憾的是，这些渐变使用不同的术语来标识它们。</span><span class="sxs-lookup"><span data-stu-id="a468e-754">MPLS providers offer several class of service gradients that unfortunately uses different terms to identify them.</span></span> <span data-ttu-id="a468e-755">您必须与提供商密切合作，确保他们了解[Lync 2010 和 2013](https://go.microsoft.com/fwlink/?LinkID=690282)带宽计算器的输出，以及适用于不同 Microsoft 365 或 Office 365 实时工作负荷应用程序的建议选项。</span><span class="sxs-lookup"><span data-stu-id="a468e-755">You will have to work closely with your provider to ensure they understand the outputs from the [Lync 2010 and 2013 Bandwidth Calculator](https://go.microsoft.com/fwlink/?LinkID=690282) and the recommended options for different Microsoft 365 or Office 365 Real Time Workloads applications.</span></span>
