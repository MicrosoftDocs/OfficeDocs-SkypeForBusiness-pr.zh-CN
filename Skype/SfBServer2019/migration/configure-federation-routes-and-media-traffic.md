# <a name="configure-federation-routes-and-media-traffic"></a>配置联盟路由和媒体流量

联盟是两个或多个 SIP 域，它允许不同组织跨网络边界进行通信中的用户之间的信任关系。 迁移到试点池后，您在需要转换到您的 Skype 的联盟路由的早期版本的边缘服务器联盟路由从业务 Server 2019 边缘服务器。
  
使用以下过程的联盟路由和媒体流量路由从早期版本的边缘服务器和控制器到您的 Skype 业务 Server 2019 边缘服务器，转换为单站点部署。
  
> [!IMPORTANT]
> 更改联盟路由和媒体流量路由需要业务服务器 2019年和早期版本边缘服务器的 Skype 安排维护停机时间。 此整个转换过程还意味着联盟的访问将不可用中断的持续时间内。 当您预期最少的用户活动的时间，应安排停机时间。 您还应向最终用户提供足够的通知。 此中断并设置适当的相应规划组织内的预期。 
  
> [!IMPORTANT]
> 如果旧版边缘服务器配置为使用相同的 FQDN 为访问边缘服务、 Web 会议边缘服务和 A / V 边缘服务，本节中的过程不受支持。 如果旧的边缘服务配置为使用相同的 FQDN，您必须首先迁移所有用户，然后在启用 Business Server 2019 边缘服务器上的 Skype 联合身份验证之前停用旧版边缘服务器。 
  
> [!IMPORTANT]
> 如果 XMPP 联盟路由通过 Skype 业务 Server 2019 边缘服务器，在早期版本的用户将不能与 XMPP 联盟伙伴进行通信，直到所有用户都移动到 Skype 的业务服务器 2019，XMPP 策略和已配置证书、 已进行了业务服务器 2019 Skype 上配置 XMPP 联盟的伙伴和最后，已更新 DNS 条目。 
  
## <a name="to-remove-the-legacy-federation-association-from-skype-for-business-server-2019-sites"></a>若要从 Skype 业务服务器 2019年网站中删除旧联盟关联

1. 业务服务器 2019年前端服务器的 Skype，在拓扑生成器中打开现有拓扑。 
    
2. 在左窗格中，导航到网站节点，位于**Skype 业务服务器**的正下方。
    
3. 右键单击站点，，，然后单击**编辑属性**。
    
4. 在左窗格中，选择**联盟路由**。 
    
5. 在**站点联盟路由分配**下，请清除**启用 SIP 联盟**复选框，若要禁用联盟路由通过旧环境。 
  
6. 单击**确定**以关闭编辑属性页。 
    
7. 从**拓扑生成器**中，选择顶层节点**Skype 业务服务器**。
    
8. 从**操作**菜单中，单击**发布拓扑**。
    
9. **下一步**完成发布过程中，单击，然后单击**完成**完成发布过程后。 
    
## <a name="to-configure-the-legacy-edge-server-as-a-non-federating-edge-server"></a>若要配置为非联盟边缘服务器的旧的边缘服务器

1. 在左窗格中，依次导航到旧安装节点和**边缘池**节点。 
    
2. 右键单击边缘服务器，然后单击**编辑属性**。
    
3. 在左窗格中，选择**常规**。 
    
4. 清除**为此边缘池 （端口 5061） 启用联盟**复选框，然后选择**确定**关闭页面。 
  
5. 从**操作**菜单中，选择**发布拓扑**，，然后单击**下一步**。
    
6. **发布向导**完成后，单击**完成**以关闭向导。 
    
7. 验证在拓扑生成器中已禁用旧边缘服务器的联盟。
  
## <a name="to-configure-certificates-on-the-legacy-edge-server"></a>在旧边缘服务器上配置证书

1. 导出外部访问代理证书及私钥，从旧的边缘服务器。 
    
2. 在业务 2019年边缘服务器和导入的 Skype 的访问代理外部证书上一步。
    
3. 将访问代理外部证书分配给边缘服务器外部接口业务服务器 2019 Skype。
    
4. 应从受信任的 CA 请求和分配 Skype 业务 Server 2019 边缘服务器的内部接口证书。 
    
## <a name="to-change-the-previous-versions-federation-route-to-use-skype-for-business-server-2019-edge-server"></a>早期版本的联盟路由更改为 Skype 用于业务 Server 2019 边缘服务器

1. 从拓扑生成器的左窗格中导航到**业务服务器 2019年的 Skype**节点和**边缘池**节点。 
    
2. 右键单击边缘服务器，然后单击**编辑属性**。
    
3. 在左窗格中，选择**常规**。 
    
4. 对于**为此边缘池 （端口 5061） 启用联盟**，请选中复选框，然后单击**确定**以关闭页。 
  
5. 从**操作**菜单中，选择**发布拓扑**，，然后单击**下一步**。
    
6. **发布向导**完成后，单击**完成**以关闭向导。 
    
7. 确认**联盟 （端口 5061）** 设置为**已启用**在拓扑生成器。
    
  
## <a name="to-update-skype-for-business-server-2019-edge-server-federation-next-hop"></a>若要更新 Skype 业务 Server 2019 边缘服务器联盟下一个跃点

1. 从拓扑生成器的左窗格中导航到**业务服务器 2019年的 Skype**节点和**边缘池**节点。 
    
2. 展开节点，右键单击列出，边缘服务器，然后单击**编辑属性**。 
    
3. 在**常规**页面，**下一个跃点选择**下，从列表中选择下拉业务服务器 2019年池 Skype。
  
4. 单击**确定**以关闭编辑属性页。 
    
5. 从**拓扑生成器**中，选择顶层节点**Skype 业务服务器**。 
    
6. 从**操作**菜单中，单击**发布拓扑**并完成向导。 
    
## <a name="to-configure-skype-for-business-server-2019-edge-server-outbound-media-path"></a>若要配置 Skype 业务 Server 2019 边缘服务器出站媒体路径

1. 从拓扑生成器的左窗格中导航到**业务服务器 2019年的 Skype**节点和**Standard Edition 前端服务器**或**Enterprise Edition 前端池**下方的池。
    
2. 右键单击池，，，然后单击**编辑属性**。
    
3. 在**关联**部分中，选择**关联边缘池 （用于媒体组件）** 复选框。 
  
4. 从下拉列表框中，选择 Skype 业务 Server 2019 边缘服务器。
    
5. 单击**确定**以关闭**编辑属性**页。 
    
## <a name="to-turn-on-skype-for-business-server-2019-edge-server-federation"></a>打开 Skype 业务 Server 2019 边缘服务器联盟

1. 从拓扑生成器的左窗格中导航到**业务服务器 2019年的 Skype**节点和**边缘池**节点。 
    
2. 展开节点，右键单击列出，边缘服务器，然后单击**编辑属性**。 
    
    > [!NOTE]
    > 仅可以为单个边缘池启用联盟。 如果您有多个边缘池，请选择一个要用作联盟的边缘池。 
  
3. 在**常规**页上，验证已选中**为此边缘池 (端口 5061) 启用联盟**复选框。 
  
4. 单击**确定**以关闭编辑属性页。 
    
5. 导航到站点节点。 
    
6. 右键单击站点，，，然后单击**编辑属性**。
    
7. 在左窗格中，单击**联盟路由**。
    
8. 在**站点联盟路由分配**下选择**启用 SIP 联盟**，，然后从列表选择列出的业务服务器 2019年边缘服务器的 Skype。 
  
9. 单击**确定**以关闭**编辑属性**页。 
    
     对于多站点部署，完成此过程在每个站点。 
    
## <a name="to-publish-edge-server-configuration-changes"></a>发布边缘服务器配置更改

1. 从**拓扑生成器**中，选择顶层节点**Skype 业务服务器**。 
    
2. 从**操作**菜单中，选择**发布拓扑**并完成向导。 
    
3. 等待在部署中的所有池间都执行 Active Directory 复制。
    
    > [!NOTE]
    > 您可能会看到以下消息：**警告： 拓扑包含多个联盟的边缘服务器。这可以对该产品的较新版本的迁移过程中出现。在这种情况下，只有一台边缘服务器将主动使用实现联合身份验证。验证外部 DNS SRV 记录指向正确的边缘服务器。如果您想要部署多个联合身份验证边缘服务器可同时活动 （即，不迁移方案中），验证所有联盟的伙伴的业务服务器使用 Skype。验证外部 DNS SRV 记录列出所有启用联盟的边缘服务器。** 该警告预期警告，可以放心地忽略。 
  
## <a name="to-configure-skype-for-business-server-2019-edge-server"></a>为业务 Server 2019 边缘服务器配置 Skype

1. 将所有 Skype 业务 Server 2019 边缘服务器联机。 
    
2. 更新外部防火墙的路由规则或硬件负载平衡器设置发送外部访问的 SIP 流量 （通常为端口 443） 和联合 （通常为端口 5061） 到业务服务器 2019年边缘服务器，而不是旧边缘服务器的 Skype。
    
    > [!NOTE]
    > 如果您没有硬件负载平衡器，您需要更新联合身份验证将解析为业务服务器访问边缘服务器的新 Skype 的 DNS A 记录。 实现这一点中断降至最低，减少 TLL 值的外部 Skype 业务服务器访问边缘 fqdn，以便在 DNS 更新为指向新的 Skype 业务访问边缘服务器的联盟和远程访问将更新快速。 
  
3. 停止从每台边缘服务器计算机**的业务 Server 访问边缘的 Skype** 。 
    
4. 从每台旧的边缘服务器计算机，打开**管理工具**中的**服务**小程序。
    
5. 在服务列表中，找到**Skype 的业务 Server 访问边缘**。
    
6. 右键单击服务名称，并选择**停止**以停止该服务。 
    
7. 设置为**已禁用**的启动类型。 
    
8. 单击**确定**以关闭**属性**窗口。
