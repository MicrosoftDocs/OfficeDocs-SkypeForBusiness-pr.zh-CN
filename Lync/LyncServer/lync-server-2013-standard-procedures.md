# <a name="standard-procedures-in-lync-server-2013"></a>Lync Server 2013 中的标准过程

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainBody">

<span> </span>

_**上次修改的主题：** 2014-08-18_

有几个资源可帮助您定义组织中所需的标准过程以及如何执行这些过程。 由于每个组织都是唯一的，因此您可能需要进一步自定义和调整这些资源以满足日常要求。标准操作过程更改，并且文档偶尔必须修订。 在进行更改后，更改管理过程在 Microsoft 操作框架的服务管理功能中定义，应确定每个更改可能对执行管理任务的方式和时间有何影响。 使用更改管理功能来更新和控制过程文档。我们建议将操作任务分为易于管理的工作负载，其中任务在每天、每周、每月和需要的基础上执行。 每日任务将重点放在对系统的正常运行非常重要的方面，并且每月任务的详细情况将重点放在确保系统长期运行的情况。 必须执行的任务可以分为以下几类：

  - [Lync Server 2013 中的日常任务](lync-server-2013-daily-tasks.md)

  - [Lync Server 2013 中的每周任务](lync-server-2013-weekly-tasks.md)

  - [Lync Server 2013 中的每月任务](lync-server-2013-monthly-tasks.md)

  - [Lync Server 2013 中的必需任务](lync-server-2013-as-needed-tasks.md)

在为操作管理准备文档时，请使用清单来帮助确保在适当的时间执行所需的任务。 有关如何准备操作清单的详细信息，请参阅位于操作清单中的示例检查表。

通常情况下，更改管理将接管系统管理完成的位置。 如果某个任务由标准过程覆盖，则它是系统管理功能的一部分。 如果没有针对任务的标准过程，则应使用更改管理功能来处理该任务。
