# <a name="documentation-in-lync-server-2013"></a>Lync Server 2013 中的文档

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**上次修改的主题：** 2015-05-15_

MOF 模型由许多服务管理功能组成。 有关如何以及何时执行任务的文档可以与同一个团队的成员或其他团队共享。 存储和共享文档的方法可能因函数类型而异。 例如，系统管理过程可能存储为 Word 文档，因为它们可能会经常打印和引用。 配置管理信息可能会自动生成并存储在数据库中，以方便搜索和编制索引。 有些文档可能是敏感文档，应加以限制。

<div>

## <a name="document-management-systems"></a>文档管理系统

文档管理系统充当文档的中央存储库，并帮助确保仅提供最新的文档修订。 您还可以考虑存档文档的旧版本以供参考。 Lync Server 提供了适用于此任务的功能。

</div>

<div>

## <a name="databases"></a>Databases

讨论了几个适用于使用数据库的工具和管理功能。 配置管理过程可能使用存储大量需要索引和搜索的数据的自动化过程。 支持人员在对新问题进行故障排除时，可以在数据库中搜索过去的问题和解决方法。

可能会有不同的数据库用于不同的用途。 决定是否应链接或组合这些数据库。 例如，如果服务中心标识了一个常见主题（例如新软件导致特定网络适配器出现问题）的多个问题，则支持人员可以查询配置数据库，以预测可能受影响的计算机数。
