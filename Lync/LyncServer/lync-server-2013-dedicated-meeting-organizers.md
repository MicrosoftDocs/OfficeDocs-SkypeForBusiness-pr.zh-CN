# 专用组织

 

_**上一次修改主题：** 2012-10-01_

若要尽可能降低大型会议池中的实时通信流量，不建议托管定期使用 Lync 客户端登录并参与即时消息 (IM)、状态、会议和语音会话的用户，而是建议您执行以下操作之一：

  - 仅为计划召开的大型会议创建一个或多个专用用户帐户，或

  - 托管负责在大型会议池中安排大型会议的员工的用户帐户。

在任一情况下，除了安排会议外，均不应使用托管在大型会议池中的用户帐户定期登录到 Lync。
