# <a name="clients-supported-for-response-group-in-lync-server-2013"></a>Lync Server 2013 中的响应组支持的客户端

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**上次修改的主题：** 2014-03-28_

响应组应用程序支持以下客户端：

  - Lync 2013 桌面客户端

  - Lync 2010 桌面客户端

  - Lync 2010 Attendant

  - Office Communications Server 2007 R2 Attendant

  - Lync Phone Edition

<div>


> [!NOTE]  
> Lync 移动客户端不支持响应组应用程序。



</div>

有关新功能的详细信息，请参阅入门文档中的 [Lync Server 2013 中的新响应组应用程序功能](lync-server-2013-new-response-group-application-features.md) 。

您可以使用的特定客户端取决于您所使用的响应组用户的类型：

  - **呼叫者**可使用前面列出的任何客户端和公用电话交换网 (PSTN) 上的标准电话来呼叫响应组。

  - 不登录到组并从中注销的**非正式代理** (代理，以接受呼叫) 可以使用助理、Lync 或 Lync Phone Edition 接受呼叫。 当使用这些客户端之一登录 Lync Server 2013 时，将自动登录到他们的组中的非正式代理。

  - 必须通过使用 Lync 2013 并从菜单项中访问代理控制台，或使用助理直接从 Internet Explorer 访问代理控制台的**代理 (代理，这些代理必须**登录到他们的组并从中接收呼叫) 可以接受呼叫。
