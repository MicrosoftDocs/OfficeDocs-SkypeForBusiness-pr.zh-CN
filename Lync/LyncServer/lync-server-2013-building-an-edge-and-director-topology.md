# <a name="building-an-edge-and-director-topology-in-lync-server-2013"></a>在 Lync Server 2013 中构建边缘和控制器拓扑

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**上次修改的主题：** 2012-09-08_

构建拓扑涉及以下规划和部署任务：

  - **规划**   您需要为您的组织定义适当的拓扑，并确定部署它所需的组件。 这些是规划过程中的标准步骤。 使用 Lync Server 2013 提供的 Microsoft Lync Server 2013 和规划工具，可以轻松地开始规划过程，也可以在要求和计划最终完成时轻松做出更改。

  - **部署**   使用拓扑生成器定义的拓扑对部署任何 Lync server 2013 服务器至关重要。 如果您未在规划工作过程中使用拓扑生成器来定义和发布拓扑，则必须先完成它并发布拓扑，然后再部署边缘服务器。

只有在部署了至少一个内部池之后，才能部署边缘服务器组件，并且必须安装拓扑生成器才能部署内部池。 本节不包含拓扑生成器安装，因为这是内部池的安装过程的一部分。

有关这些工具的详细信息，请参阅[Lync Server 2013 中的外部用户访问的部署清单](lync-server-2013-deployment-checklist-for-external-user-access.md)。

<div>


> [!NOTE]  
> 如果您以前使用拓扑生成器定义了完整的拓扑，包括边缘拓扑，则可以跳过在<A href="lync-server-2013-define-your-edge-topology.md">Lync server 2013 中定义边缘拓扑</A>并在此部分中的<A href="lync-server-2013-publish-your-topology.md">lync server 2013 任务中发布拓扑</A>，但您确实需要完成<A href="lync-server-2013-export-your-topology-and-copy-it-to-external-media-for-edge-installation.md">导出 Lync Server 2013 拓扑并将其复制到外部媒体以进行边缘安装</A>任务。
