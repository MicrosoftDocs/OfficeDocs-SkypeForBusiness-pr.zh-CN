# 在 Lync Server 2013 中定义组织的监控要求

 

_**上一次修改主题：** 2012-09-05_

简化 Microsoft Lync Server 2013 中监控的部署和安装也会简化定义组织的监控要求时涉及的过程。然而，在开始安装和配置 Lync Server 2013 之前，仍有几个关键问题应该解决：

**您想要监控哪种类型的数据？** Lync Server 2013 可监控两种不同类型的数据：呼叫详细信息记录 (CDR) 数据和用户体验质量 (QoE) 数据。呼叫详细信息记录提供了一种方法，用于跟踪 Lync Server 功能（如 IP 语音 (VoIP) 电话呼叫、即时消息 (IM)、文件传输、音频/视频 (A/V) 会议和应用程序共享会话）的使用情况。此信息帮助您了解哪些 Lync Server 功能正在被使用（以及哪些功能没有被使用），还提供关于这些功能何时被使用的信息。用户体验质量数据用于维护组织中发出的音频和视频呼叫的质量记录，包括网络数据包丢失数目、背景噪音、“抖动”量（数据包延迟的差异）等。

如果选择在 Lync Server 2013 中启用监控，则可以同时启用 CDR 监控和 QoE 监控，也可以选择启用一种监控，而禁用另一种监控。例如，假设用户仅使用即时消息和文件传输，不进行音频或视频呼叫。在这种情况下，可能没有理由启用 QoE 监控。同样，在部署监控后， Lync Server 可轻松启用和禁用监控。例如，您可能选择部署监控但最初禁用 QoE 监控。如果用户开始遇到音频或视频呼叫问题，您可以启用 QoE 监控并使用该数据帮助排查和解决这些问题

安装 Lync Server 的同时安装监控与安装 Lync Server 后再安装监控相比没有特别的优点（或缺点）。需要谨记的一点是，在安装监控之前，必须选择一台计算机来承载后端监控存储，并且必须在该计算机上安装和配置支持的 SQL Server 版本，之后才能将该计算机用于监控。如果已经在某台计算机上安装了 SQL Server，并且该计算机已准备就绪，则可以在安装 Lync Server 的同时安装监控。如果尚未准备好后端计算机，则可以继续安装 Lync Server 本身，然后在后端计算机就绪后安装监控。

**想要何时安装监控？** 可以在安装和配置 Lync Server 2013 的同时安装和配置监控； Lync Server 部署向导将为您提供在安装期间将前端池与监控数据库关联的机会。或者，可以在安装 Lync Server 本身后安装监控；完成此操作的方法是，使用拓扑生成器将前端池与含有监控数据库的服务器关联，然后发布修订后的拓扑。

**需要多少个后端监控数据库？** 一个监控数据库可以支持数万个用户（ 对于 Microsoft Lync Server 2010，估计一个同时用于监控和存档的并置数据库可支持 240,000 个用户）。此外，一个监控数据库可以被多个前端池使用；如果组织中有三个前端池，则可以将全部三个池与同一后端存储关联。

这就意味着，对于许多组织来说，数据库容量并不是确定所需后端监控数据库数量时的决定因素。相反，网络速度可能是更重要的考虑事项。假设您有三个前端池，但其中一个池位于慢速网络连接。在这种情况下，您可能想要使用两个监控数据库：一个数据库为网络连接良好的两个池提供服务，另一个数据库为网络连接较慢的池提供服务。

当规划监控基础结构时，还应该考虑到 Lync Server 2013 支持使用镜像数据库。“数据库镜像”提供了同时维护两个数据库副本，且每个数据库位于不同服务器上的方式。任何时候将数据写入主数据库，相同的数据也会写入镜像数据库。如果主数据库发生故障或不可用，可以使用一个简单的 Lync Server PowerShell 命令“故障转移”到镜像数据库。例如：

    Invoke-CsDatabaseFailover -PoolFqdn atl-cs-001.litwareinc.com -DatabaseType "Monitoring" -NewPrincipal "Mirror"

这对于规划来说很重要，仅仅因为镜像需要将所需数据库数量翻倍：除了各个主数据库，还需要第二个数据库作为镜像。

**您的 Lync Server 站点是否需要自己的自定义监控配置？** 当安装 Lync Server 2013 时，同时也安装了 CDR 和 QoE 配置设置的全局集合；通过这些全局集合可将相同的 CDR 和 QoE 设置应用到整个组织。这在许多情况下将足够：比如通常您会想要为所有用户启用 CDR 监控。

但是，有时您也可能想要对不同的站点应用不同的设置。例如，也许您想要在 Redmond 站点同时使用 CDR 和 QoE 监控，但在 Dublin 站点仅使用 CDR 监控。同样，您可能想要在 Redmond 站点将监控数据保留 60 天，而在 Dublin 站点只需要保留 30 天。 Lync Server 2013 允许您在站点作用域内创建单独的 CDR 和 QoE 配置设置集合；这样您可以采用不同方式管理各个站点。（其中包括启用和禁用监控，以及配置管理设置，如数据的保留时长。）

请注意，您既可以在部署监控前也可以在部署监控后做出此决定。例如，您可以先部署监控，然后使用全局设置管理整个组织。如果以后改变主意，例如可以为 Redmond 站点创建单独的设置集合，然后使用这些设置管理 Redmond 的监控。（在站点作用域内应用的设置始终优先于在全局作用域内应用的设置。）如果您再次改变主意，只需删除应用于 Redmond 站点的配置设置。删除站点设置集合后，全局设置集合将自动应用于该站点。
