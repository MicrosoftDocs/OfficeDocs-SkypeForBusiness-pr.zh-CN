# 基于位置的路由会议概述

 

_**上一次修改主题：** 2015-03-09_

基于位置的路由会议应用程序向 Lync 会议提供一种机制来防止 PSTN 收费绕路情形。该应用程序监视活动的会议，并基于参与的 Lync 用户的位置强制实施基于位置的路由限制。

基于位置的路由会议应用程序确定如果满足以下条件，是否对 Lync 会议强制实施基于位置的路由：

  - 会议组织者已启用基于位置的路由。基于位置的路由限制仅适用于由启用了基于位置的路由的用户组织的会议。

  - 至少一个会议参与者是 PSTN 终结点。基于位置的路由限制仅适用于包括 PSTN 终结点的会议。

  - 用于将会议桥接到 PSTN 的 PSTN 网关所在的网络站点以及组织者和参与者从中连接的网络站点。

基于位置的路由会议应用程序禁止来自不同网络站点的 Lync 用户和 PSTN 终结点参与相同会议。如果会议的组织者已启用基于位置的路由，则会议应用程序将强制实施以下限制：

  - 可以加入 Lync 会议的终结点取决于已经加入会议的终结点，此限制随着已加入的终结点离开而新终结点加入会议进行调整。如果组织者和参与者从相同网络站点加入 Lync 会议，则 PSTN 终结点、来自相同网络站点的另一参与者、来自不同网络站点的另一参与者或者来自未知网络站点的参与者都允许加入。

  - 如果组织者和参与者从不同的网络站点或未知网络站点加入会议，而 PSTN 呼叫从启用了基于位置的路由的 SIP 中继进入，则 PSTN 终结点不允许加入会议。

  - 如果组织者和参与者全部都从相同网站站点加入会议，并且有参与者从 PSTN 加入相同会议，那么来自不同网络站点的 Lync 终结点不允许加入会议。

以下是基于位置的路由会议应用程序的其他特点：

  - 如果按照给定的基于位置的路由限制，用户不允许加入会议，则用户对该会议发出的呼叫将被拒绝，其 Lync 客户端将报告该呼叫未完成或已结束。

  - 如果 PSTN 终结点加入强制实施了基于位置的路由的会议，并且是通过未启用基于位置的路由的中继加入的，那么不管其状态如何，它都不受限制，可加入会议。

  - 通过不会将呼叫发出到 PSTN 的 SIP 中继连接到中介服务器的 PBX 系统将与位于定义了 SIP 中继的相同网络站点中的 Lync 用户具有相同的强制实施。例如，如果 PBX 用户和 Lync 用户位于相同网站站点中，则 PSTN 终结点能够与他们同时加入会议；否则，如果 PBX 用户与 Lync 用户位于不同的网站站点中，PSTN 终结点不允许加入会议。
