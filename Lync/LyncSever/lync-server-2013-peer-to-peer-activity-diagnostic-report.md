# Lync Server 2013 中的点对点活动诊断报告

 

_**上一次修改主题：** 2015-03-09_

对等活动诊断报告提供有关成功和失败的对等通信会话的信息。请注意， Microsoft Lync Server 2013 区分不同类型的失败：

  - **预期失败** 。预期失败通常指纯技术意义上的失败。例如，假设您呼叫某人，但他或她不在办公室，因此无法接听电话。此呼叫未应答，因而在技术上视为失败。另一方面，这是一个预期失败：如果您不方便接听电话， Microsoft Lync Server 2013 并不会期望您接听电话。同样地，如果您尝试发送一条即时消息给某人，而对方处于脱机状态，或者对方只登录到不支持即时消息的电话时，也会发生预期失败。

  - **意外失败** 。意外错误顾名思义就是根据当时情形您没有料到会发生的错误。例如，假设您呼叫某人，对方也方便接听呼叫；但是，当 Lync Server 2013 尝试将您的呼叫路由到语音邮件时，呼叫却失败了，因为与 Exchange 统一消息的连接中断了。这是一个意外错误：您原以为呼叫总是能路由到语音邮件。一般说来，意外失败是真正的失败：不能通过用户培训或类似措施加以补救。

## 访问对等活动诊断报告

可以从“监控报告”主页访问对等诊断报告。可以单击下列任一度量值来访问[Lync Server 2013 中的故障分布报告](lync-server-2013-failure-distribution-report.md)：

  - 意外失败量

  - 预期失败量

## 充分利用对等活动诊断报告

虽然可以通过多种方法来筛选对等活动诊断报告，但默认情况下，那些筛选选项在视图中是隐藏的。若要查看您可以使用的筛选选项，请单击报告窗口右上角中的“显示/隐藏参数”按钮。然后，即可使用筛选选项。

## 筛选器

利用筛选器，您可以返回一组针对性更强的数据或通过不同的方式查看返回的数据。例如，对等活动诊断报告允许您根据会话形式（例如，即时消息、文件传输或应用程序共享）等内容进行筛选。您还可以选择数据的分组方式。在此示例中，将按小时、天、周或月对呼叫进行分组。
