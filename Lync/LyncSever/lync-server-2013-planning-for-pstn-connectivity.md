# 在 Lync Server 2013 中规划 PSTN 连接

 

_**上一次修改主题：** 2012-09-21_

企业级 VoIP 解决方案必须以始终如一的服务质量 (QoS) 提供来往于公用电话交换网 (PSTN) 的呼叫。发出和接收呼叫的用户应不必了解底层技术：从用户的角度来看， 企业语音基础结构与 PSTN 之间的呼叫应该就像是另一个电话呼叫。

Lync Server 2013 通过以下选项提供可靠且可扩展的 PSTN 连接：

  - 到 Internet 电话服务提供商 (ITSP) 的 **SIP 中继**

  - 到 PSTN 网关的 **直接 SIP 连接**

  - 到 PBX 的 **直接 SIP 连接**

根据企业大小、地理范围和现有语音基础结构，企业可能会在不同位置使用一个、两个甚至所有三个选项。
